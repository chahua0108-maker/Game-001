# Vampire Crawlers 前三小时核心体验分析

生成时间：2026-05-10 20:10 HKT  
输入证据：`13-first-3-hours-video-map.md`、三条长 VOD 的 metadata / 字幕 / 已完成短片段 / contact sheet。  
结论等级：比上一轮“只能理解 20 分钟战斗段”明显更强，但还不是逐分钟完整复盘。

## 当前判断

我现在可以更有把握地理解它前三小时的核心体验，不再只是宣传片和短 gameplay 的表层理解。证据显示它的前三小时不是单纯“幸存者 + 卡牌皮肤”，而是三条压力同时推进：

1. 地牢走廊中的即时压力：移动、怪潮、血量、boss 时间点持续逼迫玩家。
2. 卡组引擎压力：抽牌、费用、颜色、combo、宝石槽和卡牌升级会持续抢注意力。
3. 长线解锁压力：商店、地图推进、永久成长、牺牲卡牌解锁和 reroll 让失败也有下一局目标。

这说明它的爽感不只来自屏幕弹幕数量，而来自“快节奏战斗 + 短周期 build 决策 + 长周期解锁目标”三层叠加。

## 三小时体验曲线

| 阶段 | 体验功能 | 玩家主要在处理什么 | 设计意义 |
|---|---|---|---|
| 0-15 分钟 | 定义游戏身份 | “这是 Vampire Survivors if it was a card game”；理解走廊、卡牌、费用、地图、商店 | 开场必须非常快地说明差异化，不然会被当成换皮 |
| 15-45 分钟 | 第一轮压力建立 | 经验、升级、血量、抽牌、早期死亡风险 | 早期不能只给数值成长，要让玩家看到卡牌引擎能救命 |
| 45-90 分钟 | build 开始成形 | boss 节点、抽牌紧张、奖励选择、might/armor/卡牌升级 | 中段爽感来自“我刚才的选择开始连起来了” |
| 90-180 分钟 | 长线系统显形 | gem slots、牺牲正确颜色卡牌解锁、reroll、Reaper/后期压力 | 第三小时要让玩家看到 meta 结构，不只是重复跑同一局 |

## 关键体验拆解

### 1. 它的“伪 3D”不是主要深度，主要是空间压迫感

Unity 3D 的表现让走廊、敌潮、角色移动更像实体空间，但真正的系统深度不在 3D 本身，而在：

- 怪物从走廊压过来，减少玩家拖时间的空间。
- boss / Reaper 这类节点给 run 一个时间边界。
- 卡牌选择界面打断战斗节奏，形成“喘息但不安全”的决策窗口。

Web 原型如果用 Three.js 或轻量 2.5D 表现，可以先复刻空间压迫和 UI 节奏，不必一开始追求完整 Unity 级 3D 场景。

### 2. 卡牌不是奖励菜单，而是战斗引擎

VOD 证据里反复出现 draw card、mana、socket、gem slot、sacrifice correct color 等信息。也就是说，卡牌系统不是每次升级时弹一下的三选一，而是贯穿 run 的引擎：

- 抽牌决定短期可行动作。
- 费用和颜色决定能不能串 combo。
- socket/gem slot 决定长期 build 上限。
- sacrifice 解锁把“删卡/献祭”变成永久进度的一部分。

我们的原创原型不能只做“升级三选一”，至少要验证一套低成本卡牌引擎。

### 3. 爽感来自“将要崩盘但被 build 救回来”

字幕和片段里多次出现接近死亡、需要抽牌、boss 将至、资源不够、后期压力上升的表达。这个体验不是无限平推，而是：

1. 当前局势在崩。
2. 玩家赌下一张牌、下一次升级或下一次奖励。
3. 某个 combo 或升级把局面拉回来。
4. 玩家获得“我构筑对了”的反馈。

这对我们的原型很重要：第一版不应该只追求大量敌人和高伤害数字，而要有明确的“濒临崩盘 -> 组合翻盘”窗口。

## 对原创方向的影响

结合 `10-market-theme-research.md`，当前更推荐把题材从纯 `Organ Fleet / 星舰胃袋` 调整为更市场友好的：

`夜班收容：器官档案`

这个方向可以把前三小时体验映射成：

| Vampire Crawlers 体验 | 原创映射 |
|---|---|
| 地牢走廊压力 | 异常收容设施夜班走廊、封锁门、污染潮 |
| 卡牌/费用/颜色 | 档案卡、器官样本、禁忌配方、污染颜色 |
| gem slot/socket | 器官插槽、样本培养皿、收容柜改造 |
| sacrifice correct color | 献祭同色样本解锁异常档案 |
| boss/Reaper | 收容等级突破、巡夜警报、最终异常实体 |
| hub/shop | 夜班办公室、黑市实验台、档案柜 |

这个映射的好处是：市场上更容易理解“都市怪谈/异常收容”，同时仍能容纳器官链、卡牌构筑和伪 3D 走廊压迫。

## Web 原型必须验证的 5 件事

1. 2.5D 走廊压迫：敌潮必须让玩家感到被推挤，而不是平面刷怪。
2. 10 分钟内的卡牌引擎：抽牌、费用、颜色、combo 至少要跑通一条。
3. 濒临崩盘的翻盘窗口：第 5-8 分钟必须出现一次 build 救场。
4. 长线解锁预览：即使只做 demo，也要让玩家看到下一局能解锁什么。
5. 题材与机制绑定：卡牌名、插槽、献祭、boss 都要从“收容/器官/档案”世界观自然长出来。

## 当前可执行结论

我现在认为已经可以进入“正式 GDD 重写”阶段，不需要继续等完整逐帧视频证据。剩余视频缺口主要影响细节校准，不影响核心判断：

- 竞品前三小时核心不是性能或单纯视觉，而是三层压力循环。
- Web 原型要优先验证卡牌引擎和走廊压迫的组合。
- 题材方向应服务机制，不应只是换皮。
- 市场向方向建议以“都市怪谈/异常收容 + 器官炼金/档案卡牌”为主，而不是纯太空器官或纯传统仙侠。
