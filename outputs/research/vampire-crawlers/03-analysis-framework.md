# 高阶分析框架

## 分工原则

低档 AI 只负责：

- 找链接。
- 拉公开数据。
- 提取页面上的评分、日期、标题。
- 生成初步结构化表。

高阶 AI 负责：

- 判断证据可信度。
- 解释媒体与玩家意见为什么冲突。
- 区分 IP 红利和机制本身。
- 把负面评价转成突破机会。
- 把核心体验迁移到原创世界观。

## 证据分级

| 级别 | 类型 | 用途 |
|---|---|---|
| A | Steam 全量评测统计 | 判断大样本玩家情绪、语言分布、常见主题 |
| A | 多家媒体共同提到的同一问题 | 判断确定性设计风险 |
| B | YouTube 长视频评论区高赞反馈 | 判断具体体验点是否被玩家感知 |
| B | 评测视频字幕/口播 | 捕捉动态体验和手感描述 |
| C | 单家媒体观点 | 作为线索，不直接下结论 |
| C | 官方 trailer | 判断官方想卖什么，不等于玩家真实感受 |

## 分析口径

### 1. IP 红利拆分

必须把以下内容从可复用优势中剥离：

- Vampire Survivors 品牌信任。
- 角色、武器、怪物、音乐、梗和前作记忆。
- 低价带来的宽容。
- 粉丝愿意忍受慢热或重复的耐心。

对我们的意义：

如果没有这些红利，原型必须更早展示“为什么爽”和“为什么新”。

### 2. 核心体验拆分

把体验拆成可迁移的系统：

- 第一人称推进：玩家感觉自己在深入危险空间。
- 卡牌顺序连击：玩家不是随机出牌，而是在拼一个爆发序列。
- 战斗反馈：每张卡都有即时声音、数字、动画和资源反馈。
- 成长选择：每次升级都让玩家预期后续 build 会变强。
- 长线解锁：失败后也带着资源回村，下一局更强。

### 3. 负面评价转译

| 负面 | 设计解释 | 我们的突破方向 |
|---|---|---|
| 前期重复 | 早期选择空间太窄，玩家只会按数字顺序打牌 | 第一局就给 2-3 条有效路线 |
| 慢热/逼肝 | 真正爽点在长线解锁后才出现 | Demo 10 分钟内必须出现一次 build 爆发 |
| UI 信息不清 | 卡牌、状态、升级和连击关系没有讲透 | 连击预览、伤害预测、关键词高亮 |
| 地牢浅 | 地图只是战斗容器 | 房间事件、风险路径、可见目标 |
| 过于依赖 IP | 旧角色/武器提供情感捷径 | 新世界观必须把卡牌和职业绑定到幻想 |

### 4. 视频分析口径

每段视频不只看截图，要记录：

- 事件频率：每 10 秒发生几次反馈。
- 反馈层次：声音、数字、动画、粒子、UI、资源。
- 决策间隔：玩家多久做一次有意义选择。
- 爽点曲线：什么时候从普通战斗变成爆发。
- 可读性：爆发时玩家还能不能理解发生了什么。

## 输出格式

最终策划不直接说“像 Vampire Crawlers”，而使用以下结构：

1. 核心体验目标。
2. 原创世界观钩子。
3. 玩家身份和主要动词。
4. 地牢/空间结构。
5. 卡牌/连击系统。
6. 成长和解锁。
7. 第一局 10 分钟流程。
8. 与竞品相比保留什么、改掉什么、突破什么。
