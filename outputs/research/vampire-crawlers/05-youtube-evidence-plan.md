# YouTube 动态证据采集方案

## 为什么不能只截图

Vampire Crawlers 的核心卖点不是静态画面，而是动态爽感：

- 出牌顺序带来的连击节奏。
- 数字、音效、粒子、经验宝石的连续反馈。
- 地牢推进和遭遇战之间的节奏切换。
- 评论区对“上头、重复、慢热、UI 混乱、build 破局”的即时反馈。

所以视频材料必须包含短片段、字幕、评论和帧序列。

## 已授权能力

用户已明确允许读取 Chrome/YouTube cookies，用于本次公开视频和评论采集。范围限制：

- 只用于公开视频元数据、评论、字幕和短片段。
- 不导出 cookie 内容。
- 不读取或记录无关站点数据。
- 不下载大量完整视频，只抽取研究需要的小片段。

## 采集对象

| 类型 | 代表视频 | 目的 |
|---|---|---|
| 官方 announcement trailer | 官方频道 | 看官方如何定义卖点 |
| 官方 gameplay trailer | 官方频道 / IGN 转发 | 看核心循环被如何展示 |
| 官方 launch trailer | 官方频道 | 看上市时主推信息 |
| 媒体评测视频 | GameSpot、Nintendo Life、SwitchUp | 看评测者如何描述动态体验 |
| 长实况/挑战视频 | The Spiffing Brit 等 | 看后期 build 破局和玩家反应 |

## 输出材料

1. `.info.json`
   - 标题、频道、时长、播放量、点赞、评论数、评论样本。

2. `.vtt`
   - 字幕或自动字幕，用于提取口播观点。

3. 短片段
   - 每段 8-30 秒，覆盖移动、战斗、升级、商店、后期爆发。

4. contact sheet
   - 每 0.5 秒抽帧拼图，用于静态审阅动态密度。

5. Excel `Video Materials`
   - 视频、片段、时间码、素材路径、评论数、字幕路径、观察目的。

## 分析方式

每个片段记录：

- 前 3 秒是否能理解场景。
- 每 10 秒反馈事件数量。
- 是否出现可识别的连击峰值。
- UI 是否在爆发时仍可读。
- 评论区是否提到相同体验。

## 当前限制

`yt-dlp` 默认 Chrome profile 不能通过 YouTube 检测；`Profile 3` 和 `Profile 5` 已验证可获取公开视频元数据。当前采集使用 `chrome:Profile 3`。
