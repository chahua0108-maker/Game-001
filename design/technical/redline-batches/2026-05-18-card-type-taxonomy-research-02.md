# Redline 卡牌类型与机制 Taxonomy 调研 02

日期：2026-05-18  
范围：只做研究文档与 Redline 设计翻译，不改代码、不碰 runtime。  
目标：把常见卡牌游戏的卡牌类型与机制分类，收敛成 Redline Hyper-Turn 可用的 P0/P1/P2 框架。

## 结论摘要

Redline 当前不需要先扩一个完整 CCG / TCG 式类型系统。P0 应该只保留能证明“高速回合制卡牌压力”的类型：攻击牌、技能牌、资源牌、抽牌牌、修补牌、payoff 牌、状态牌、诅咒牌。这 8 类足够支撑 `0 -> 1 -> 2 -> payoff`、敌人意图、坏手牌修补、抽牌循环和奖励三选一。

单位牌、法术牌、能力牌、角色牌、事件牌更适合 P1/P2。它们不是不重要，而是会把当前样片从“验证卡牌链路”拉向“完整牌库生态”。P0 若过早引入单位站场、永久能力、剧情事件和角色牌，会稀释当前最需要证明的事情：玩家是否能在 3-5 回合内读懂手牌、接上链、修补缺口、用 payoff 清掉可见意图。

## Redline 的分类原则

卡牌分类不要只按幻想包装命名，而要同时回答三个问题：

| 维度 | 问题 | Redline 用途 |
| --- | --- | --- |
| 构筑位置 | 这张牌为什么会进牌组？ | 决定奖励三选一、牌库比例、路线标签 |
| 单局战斗位置 | 这张牌在本回合解决什么压力？ | 决定手牌读法、敌意图处理、payoff 武装 |
| 发牌循环位置 | 这张牌如何影响 draw pile / hand / discard / exhaust？ | 决定抽到时机、坏手牌风险、循环一致性 |

因此，Redline 的卡牌不应只有 `type = attack`。更推荐拆成四层标签：

```text
cardType: attack / skill / resource / status / curse / ...
chainRole: starter / bridge / expand / repair / payoff / deadweight
cycleRole: opener / bridge-density / draw-fixer / late-payoff / clog / exhaust
buildRole: damage / defense / consistency / economy / ceiling / risk
```

P0 可以先不实现这四层字段，但设计、HUD、QA 和奖励文案应该按这个框架统一说话。

## 常见类型翻译表

| 类型 | 行业含义 | Redline 翻译 | 构筑位置 | 单局战斗位置 | 发牌循环位置 | 优先级 |

| 攻击牌 | 直接造成伤害，常见于 Slay the Spire 等 deckbuilder | 清算、切割、印章、前排处理 | 基础密度；保证每回合能减少敌意图 | 处理本回合威胁，建立 `0/1/2` 链 | 希望高频出现，过多会缺修补 | P0 |
| 技能牌 | 防御、抽牌、标记、位移、控制、修补 | 不以伤害为主的行动牌 | 提供一致性和救场路线 | 续链、抽牌、降意图、改变目标 | 适合做循环润滑剂 | P0 |
| 能力牌 | 打出后给长期被动效果，常见为 Power / Relic-like card | 清算许可、回合规则改写、长期授权 | 定义 build 方向 | 战斗中通常不立刻解场，提供后续收益 | 一般希望打出后移出循环或持久存在 | P1 |
| 单位牌 | 召唤随从、站场、承伤、持续攻击 | 审计代理、列车乘员、自动防线等 | 构筑可围绕站场/保护/增益 | 改变敌意图承受者或提供持续输出 | 需要保留场面状态，循环复杂 | P2 |
| 法术牌 | 一次性效果，范围很宽，可攻击也可控制 | Redline 可把“法术”降级为技能/攻击的包装词 | 不建议作为 P0 顶层类型 | 作为瞬时清算动作 | 通常进弃牌或消耗 | P1 |

| 诅咒牌 | 长期负面牌，常由奖励代价或事件获得 | 合规污点、黑账、违约条款 | 高收益奖励的代价 | 本回合通常是坏抽或惩罚触发器 | 长期污染牌库，可保留跨战斗 | P1 |
| 资源牌 | 产生能量、授权、费用折扣、保留资源 | 临时授权、返费、修补储备、信用额度 | 决定能否打长回合 | 支付 payoff 或继续链 | 需要控制密度，过多会空转 | P0 |
| 抽牌牌 | 抽 1、检索、过滤、重抽 | 取证、抽样、翻案、调档 | 提升一致性，寻找缺口 | 让坏手牌继续运转 | 加速洗牌，也可能抽到状态/诅咒 | P0 |
| 修补牌 | Wild、复制、变费、补缺口、重排 | Wild Gap Key、Mana Stitch、费用替身 | 解决构筑不稳定 | 填补 `0 -> 1 -> 2` 中缺失的一环 | 可作为桥牌密度替代，但不能太多 | P0 |
| payoff 牌 | 需要前置条件才强的大结算牌 | Severance Burst、Red Ledger、终局清算 | 构筑天花板，奖励玩家准备路线 | armed 后清前排/清场/取消意图 | 不应稳定起手强打；抽早了是压力 | P0 |
| 角色牌 | 英雄、职业、队长、同伴，改变初始牌组或规则 | 主角许可证、审计员身份、部门流派 | 定义初始 build 和长期差异 | 常提供常驻技能或独特资源 | 不进入普通发牌循环，或作为战斗外配置 | P2 |
| 事件牌 | 地图/战斗中触发的选择、一次性遭遇 | 案件事件、审计岔路、债务合同 | 提供风险收益和牌库污染/净化 | 不属于常规手牌战斗，除非做战斗事件 | 多在战斗外，不参与普通洗牌 | P2 |

## P0：当前样片应该落地的类型

P0 目标不是“类型完整”，而是证明玩家能理解一套高速卡牌压力循环。

### 1. 攻击牌

Redline 的攻击牌应分成三个链路角色：

| 链路角色 | 成本 | 战斗职责 | 例子 |
| --- | ---: | --- | --- |
| starter | 0 | 开链、轻伤、标记或拉目标 | Debt Hook、Blood Reclaim |
| bridge | 1 | 主要单体伤害，承接 0 费 | Redline Cut、Heartbeat Spark |
| expand | 2 | 前排压力、范围伤害、取消多份意图 | Row Cleave、Clearance Order |

构筑位置：攻击牌是底盘密度。P0 牌组需要足够的 0/1/2 攻击或半攻击牌，让第一手能看见一条路线。

单局战斗位置：攻击牌负责把“我打了正确牌序”翻译成敌人 HP 和意图的可见下降。

发牌循环位置：攻击牌应反复进入弃牌再洗回。P0 不建议让基础攻击牌消耗，否则玩家很快失去链路密度。

### 2. 技能牌

P0 技能牌不应该泛化成所有非攻击效果，而应只保留三类：

| 技能子类 | 作用 | Redline 示例 |
| --- | --- | --- |
| 抽牌技能 | 找下一环或 payoff | Pulse Draw、Paper Route |
| 修补技能 | Wild、替代费用、接回断链 | Wild Gap Key、Wild Mana Stitch |
| 意图处理技能 | 降低、延迟或重定向敌意图 | Verdict Mark 可兼具标记与减意图 |

构筑位置：技能牌提供一致性。它们让玩家不是只靠运气抽到 `0 -> 1 -> 2`。

单局战斗位置：技能牌解决“这手牌缺一环怎么办”。

发牌循环位置：技能牌应该进入弃牌循环；个别强修补可以 exhaust，避免每轮无限稳定。

### 3. 资源牌

Redline P0 的资源牌不应照搬 MTG 土地或 Hearthstone 自然涨费。当前更适合三种可见资源：

| 资源形式 | P0 用法 | 限制 |
| --- | --- | --- |
| 临时授权 | 完成 `0 -> 1 -> 2` 后支付 3 费 payoff | 只支付 payoff 或 expected cost |
| 返费 | 击杀/取消意图后返还 1 MP | 必须有可见来源 |
| 修补储备 | 只用于填链路缺口 | 不能支付任意高费牌 |

构筑位置：资源牌是长回合路线，不是裸强度。

单局战斗位置：解释为什么 3 MP 底盘下能打出 payoff。

发牌循环位置：资源牌过多会导致手牌没有伤害，所以 P0 应限制数量，并让资源效果绑定链路或意图取消。

### 4. 抽牌牌

抽牌牌是发牌循环的核心润滑剂。

| 抽牌机制 | Redline 用法 | 风险 |
| --- | --- | --- |
| 抽 1 | 找下一张 bridge / expand | 如果无费用支持，抽到也打不出 |
| 检索某类 | 找 starter、repair 或 payoff | P0 可先用脚本，不做复杂检索 UI |
| 弃 1 抽 1 | 过滤状态/诅咒 | P1 更合适，P0 可先不做 |
| 重抽整手 | 类似 Redraw Bell | 会改变回合节奏，P1 |

构筑位置：抽牌牌提高一致性，但不能代替链路密度。

单局战斗位置：当手牌缺口出现时，给玩家一次“继续找解”的动作。

发牌循环位置：抽牌会加速洗牌，因此状态牌/诅咒牌的污染价值也会随之提高。

### 5. 修补牌

修补牌是 Redline 与普通费用曲线的关键差异。它让坏手牌不是死局。

| 修补机制 | Redline 解释 | P0 建议 |
| --- | --- | --- |
| Wild cost | 视为当前 expected cost | 必做 |
| 变费 | 把下一张牌临时降到 expected cost | P1 |
| 复制 | 复制上一张或下一张链路牌 | P1 |
| 重排 | 改变手牌顺序或抽牌顶 | P2 |
| 返费修补 | 修补成功后返还支付费用 | P0 可做简化版 |

构筑位置：修补牌是稳定性路线，和 payoff 路线、资源路线形成奖励三选一。

单局战斗位置：让玩家从“缺 1/缺 2”的坏手牌中救回来。

发牌循环位置：修补牌密度过高会让链路没有风险。P0 需要少量、明确、可解释。

### 6. payoff 牌

payoff 牌必须是“条件大结算”，不能是固定强牌。

| 状态 | 规则 | 玩家反馈 |
| --- | --- | --- |
| unarmed | 首打或断链后打出，只做低收益 | 告诉玩家“这不是免费清场” |
| armed by chain | 已完成 `0 -> 1 -> 2` 或 chain length >= 3 | 清前排、溢出、取消大量意图 |
| armed by repair | 修补牌成功补上缺口后触发 | 说明“修补救场成功” |
| armed by resource | 临时授权或返费支付 | 说明资源来源和 spend restriction |

构筑位置：payoff 是天花板。奖励里拿第二张 payoff 是贪上限，拿修补/抽牌是补一致性。

单局战斗位置：把前几张牌的准备转化为清场、回血、抽牌、取消意图等高潮。

发牌循环位置：payoff 不应每轮稳定起手。抽早了应该诱惑玩家犯错，抽晚了应该成为长回合目标。

### 7. 状态牌

状态牌适合 P0，因为它能直接强化发牌循环的压力，而不需要完整局外系统。

| 状态牌 | 作用 | Redline 示例 |
| --- | --- | --- |
| Wound-like | 占手牌，无效果 | 债务回执 |
| Burn-like | 回合末伤害或扣资源 | 过载条款 |
| Dazed-like | 抽到后无效，回合末消失 | 迟滞文件 |
| Slime-like | 可花资源移除 | 污染凭证 |

构筑位置：状态牌不是构筑奖励，而是敌人、透支、事件或强牌代价。

单局战斗位置：让玩家感到“上回合没处理好，现在污染了手牌”。

发牌循环位置：状态牌进入弃牌并洗回，是最直接的循环压力。P0 建议先做 1-2 种，不要扩太多。

### 8. 诅咒牌

诅咒牌比状态牌更长期，P0 可只定义，不急着实现完整局外保留。

| 诅咒用途 | Redline 翻译 | 优先级 |
| --- | --- | --- |
| 高收益奖励代价 | 拿强 payoff，同时塞一张黑账 | P1 |
| 事件代价 | 选择信用额度，加入违约条款 | P2 |
| 构筑约束 | 某些强角色自带诅咒 | P2 |

P0 若需要“坏牌”证明抽牌循环，优先用状态牌，不要先做跨战斗诅咒。

## P1：跑通后再加入的类型

### 能力牌

能力牌适合在 P1 做，因为它们会引入持久状态和规则改写。

Redline 可用方向：

- `清算许可`：每回合第一次完成 `0 -> 1 -> 2` 后抽 1。
- `红线档案`：每次 armed payoff 取消意图后，下一回合 starter +1 密度。
- `审计章程`：Wild 修补不再降低倍率，但 payoff 基础伤害降低。

构筑位置：能力牌定义 build 身份。

单局战斗位置：前期亏 tempo，后期改变规则。

发牌循环位置：建议打出后移出循环或变成持久状态，避免反复抽到没意义。

### 法术牌

法术牌作为顶层类型容易模糊。Redline 可以把它当作包装层，而不是规则层。

P1 用法：

- 一次性强控制：冻结一个敌人意图。
- 一次性检索：从抽牌堆找一张 repair。
- 一次性清理：移除一张状态牌。

若一张法术造成伤害，规则上仍应归入攻击牌；若抽牌/修补，规则上仍应归入技能牌。这样 QA 和 HUD 不需要解析“法术到底是什么”。

### 诅咒牌

P1 可以把诅咒接入奖励代价：

| 奖励 | 代价诅咒 | 设计意图 |
| --- | --- | --- |
| 强 payoff | 黑账 | 提高上限但降低一致性 |
| Max MP +1 | 违约条款 | 资源更高但抽牌污染 |
| 每回合抽 +1 | 审计噪声 | 抽得多也更容易抽到坏牌 |

这能让构筑选择从“拿最强牌”变成“我愿意承担什么长期污染”。

## P2：完整生态阶段再考虑的类型

### 单位牌

单位牌会带来场面、站位、AI、承伤、持续效果、死亡触发等整套系统。它不是当前 P0 的核心。

Redline 未来可以做：

- 审计代理：替玩家承受一个敌人意图。
- 列车乘员：每回合自动提供一种 starter 或 draw。
- 部门支援：站场后让某类牌 +1 效果。

但这会让“牌序压力”变成“场面经营”，建议 P2。

### 角色牌

角色牌适合定义 run 开局身份，而不是进入普通手牌。

可能方向：

- `执行审计员`：starter 密度高，payoff 普通。
- `债务调度员`：抽牌多，状态牌更多。
- `红线裁定官`：payoff 强，但诅咒起手。

角色牌会影响初始牌组、初始资源、奖励池和主题包装。P2 前不建议做。

### 事件牌

事件牌更像地图/遭遇层，不是战斗手牌层。

可能方向：

- 选择：删除一张状态牌，或拿一张强修补但加入诅咒。
- 选择：获得临时授权上限，下一战敌意图 +2。
- 选择：升级 payoff，但 starter 少一张。

事件牌需要 run 结构、地图或战斗间界面支撑。P2。

## 构筑、战斗、发牌循环的统一模型

### 构筑层

Redline P0 奖励三选一建议永远围绕三条路线：

| 路线 | 给什么 | 解决什么问题 | 风险 |
| --- | --- | --- | --- |
| 一致性 | starter / bridge / draw / repair | 抽不到链路 | 上限不高 |
| 资源 | 临时授权 / 返费 / 修补储备 | 打不出 payoff | 可能空转 |
| 天花板 | payoff / payoff 增幅 | 缺少清场高潮 | 抽早会卡手 |

不要在 P0 奖励里混入单位、角色、事件和复杂能力，否则玩家无法判断奖励到底是在修补哪条链路。

### 单局战斗层

P0 每回合应能被读成：

```text
看敌意图
  -> 看手牌是否有 starter
  -> 找 bridge / expand
  -> 缺口用 draw 或 repair
  -> 完成 0 -> 1 -> 2
  -> 获得授权或武装 payoff
  -> 用 payoff 取消最大压力
  -> 结束回合结算剩余意图
```

所有卡牌类型都要服务这个序列。不能服务这个序列的类型，先放 P1/P2。

### 发牌循环层

Redline 的发牌循环应让类型承担不同位置：

| 循环位置 | 需要的类型 | 设计目的 |
| --- | --- | --- |
| 起手 | starter、bridge、少量 draw | 让玩家 3 秒内看见路线 |
| 中段抽牌 | expand、repair、resource | 让坏手牌有救 |
| 后段抽牌 | payoff、强技能 | 形成高潮 |
| 弃牌洗回 | 攻击、技能、普通资源 | 保持循环密度 |
| 污染循环 | 状态、诅咒 | 让前一回合后果进入下一手 |
| 消耗区 | 强修补、强法术、部分状态 | 防止无限稳定 |

如果 P0 仍然每轮固定全卡组发 6 张，taxonomy 的价值会被削弱。类型分类必须和“小手牌 + 抽牌堆 + 弃牌堆 + 状态污染”一起验证。

## 推荐 P0 卡池比例

以 12-15 张小牌库为目标，而不是完整 roguelike 卡池：

| 类型 | 数量建议 | 目的 |
| --- | ---: | --- |
| 0 费 starter | 3-4 | 确保多数回合能开链 |
| 1 费 bridge | 3-4 | 形成主伤害密度 |
| 2 费 expand | 2-3 | 让 `0 -> 1 -> 2` 有结果 |
| draw / repair | 2-3 | 修补坏手牌 |
| resource / authorization | 1-2 | 支撑 payoff 合法性 |
| payoff | 1-2 | 提供清场高潮 |
| status | 1-2 种，由敌人塞入 | 验证循环污染 |
| curse | 只定义，不进 P0 基础池 | 留给 P1 奖励代价 |

## P0 / P1 / P2 裁决表

| 类型 | 裁决 | 原因 |
| --- | --- | --- |
| 攻击牌 | P0 | 直接支撑敌意图处理和费用链 |
| 技能牌 | P0 | 承担抽牌、修补、意图处理 |
| 资源牌 | P0 | 解释 3 MP 下的 payoff 合法性 |
| 抽牌牌 | P0 | 让发牌循环不是固定按钮组 |
| 修补牌 | P0 | 让坏手牌可救，形成 Redline 特色 |
| payoff 牌 | P0 | 证明爆发来自构筑和牌序 |
| 状态牌 | P0 | 让敌人和代价污染下一轮手牌 |
| 诅咒牌 | P1 | 适合奖励代价，P0 可先不做长期污染 |
| 能力牌 | P1 | 需要持久状态和规则改写 |
| 法术牌 | P1 | 可作为包装层，但不应成为 P0 顶层规则 |
| 单位牌 | P2 | 会引入站场和 AI 复杂度 |
| 角色牌 | P2 | 属于 run 开局身份和长期差异 |
| 事件牌 | P2 | 需要地图/战斗间结构 |

## 对 Redline 的直接建议

1. P0 的玩家可见标签用 `START 0`、`BRIDGE 1`、`EXPAND 2`、`REPAIR`、`DRAW`、`RESOURCE`、`PAYOFF`、`STATUS`，不要先暴露一大堆 CCG 类型名。
2. 攻击/技能是规则大类，starter/bridge/expand/repair/payoff 是 Redline 更重要的链路角色。
3. 状态牌应尽早进入 P0，因为它能证明“发牌循环有后果”；诅咒牌先作为 P1 奖励代价。
4. 单位、角色、事件先不要碰，它们会把样片拉到另一个游戏层级。
5. reward 文案不要写“获得一张强牌”，而要写“补 0 费开链 / 补 1 费桥接 / 补抽牌 / 补修补 / 补 payoff 上限”。
6. QA 不只检查能否出牌，还要检查每种 P0 类型是否在构筑、战斗、循环中各有位置。

## 最小验收问题

P0 taxonomy 成立，需要能回答这些问题：

1. 第一手牌里，玩家是否能一眼识别 starter、bridge、expand？
2. payoff 首打是否明显弱于 armed payoff？
3. 坏手牌是否至少有一次通过 draw 或 repair 被救回来？
4. 敌人或代价制造的 status 是否真的污染了下一轮抽牌？
5. 奖励三选一是否能明确对应一致性、资源、天花板三条路线？
6. 每张 P0 卡是否都能说清它在构筑、战斗、发牌循环中的位置？

如果上述问题还答不上来，就不应该扩 P1/P2 类型。
