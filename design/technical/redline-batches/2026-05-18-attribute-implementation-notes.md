# 2026-05-18 P0 属性实现口径

## P0 口径

- P0 只验证 3-5 回合样片，不扩成完整属性海。
- 当前核心是“终局授权”：玩家完成 0->1->2 后，获得本回合临时授权，用来支付 3 MP payoff。
- 3 MP payoff 是爆发/清场终结点，描述必须强调“临时授权支付”，不能暗示永久 MP 成长。
- Wild 修补牌用于补断链、补手牌或返还本回合 MP，定位是节奏修补，不是 Max MP +1。
- 抽牌/重排/返还 MP 的资源牌只服务本回合找链、找 2 MP 段或找 3 MP payoff。

## P1 不做项

- 不把 Max MP +1 做成 P0 默认奖励。
- 不新增完整属性成长、稀有度、局外养成或大规模 reward 字段。
- 不改 sim/runtime/HUD/tests 来适配额外系统。
- 不把 3 MP payoff 设计成普通可长期稳定支付的高费卡。

## Reward Pool 规则

- 前段优先让三类牌可见：Wild 修补、3 MP payoff、抽牌/重排资源。
- 允许早出 payoff，但必须让玩家理解它依赖 0->1->2 的本回合授权。
- Max MP +1 若要进入奖励，留到 P1 作为明确的成长奖励重新评估。

## 验收脚本

```bash
npm test -- --run
```

验收重点：

- 卡牌描述不出现“永久成长”“Max MP +1”之类误导。
- 3 MP burst/payoff 文案明确是消耗 0->1->2 后的本回合授权。
- Wild 和 draw/mana 文案明确是本回合节奏修补。
- reward pool 前段能尽早看到修补/payoff/resource 三类。

## 避免误区

- 不要把 `energyGain: 1` 解释成最大 MP 增长；它只是本回合资源返还。
- 不要为了展示属性系统而新增大量属性牌；P0 只需要证明授权链和 payoff 节奏。
- 不要让文案承诺运行时代码还没有稳定表达的长期成长。
