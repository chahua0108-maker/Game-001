## 2. 必须出现的 6 个玩家镜头

### 镜头 1：奖励后不是结束，而是进入路线选择

玩家画面：

- 第一战结算后，玩家先看到三选一奖励。
- 选择奖励后，界面不直接进入下一战，而是出现两个路线候选。
- 每条路线用短句说明“下一战会改变什么”，例如 `补给：奖励+1 / 清污机会`、`高压：+1 MP / 敌压更高`。

玩家应理解：

```text
我已经拿了奖励；现在是在决定下一战的路线，而不是点一个继续按钮。
```

失败信号：

- 奖励选择后直接重开下一战。
- 路线候选只是图标或内部 id，玩家不知道差异。
- 路线描述只说风味，不说下一战影响。

可验收条件：

- 奖励选择事件和路线选择事件在体验上分成两个明确步骤。
- 路线候选至少 2 个，且差异能被玩家口述为资源、奖励、压力、清污、强化中的一种。
- 玩家不用看日志也能回答：“我现在为什么要选 A 而不是 B？”

核心体验打分影响：

- 短 run / 地牢推进：`+1.5`。从“战后奖励”推进为“战后路线决策”。
- 奖励构筑回应：`+0.5`。奖励不再孤立，开始和路线目标连起来。
- 玩家反馈与操作手感：不加分，只维持门槛；如果看不懂路线，反扣 `-1` 到 `-2`。

### 镜头 2：路线选择后，下一战继承被提前预览

玩家画面：

- 玩家点路线后，进入下一战前看到一条短摘要。
- 摘要包含上一战奖励、所选路线、下一战继承项。
- 示例：`带入 Silt Purge / 路线 Repair Cache / 下一战 奖励+1 清污候选`。

玩家应理解：

```text
下一战不是重置样片；我刚才选的奖励和路线会一起生效。
```

失败信号：

- 只有 `Node 2/3`，没有继承说明。
- 玩家进入第二战后才发现变化，且不知道来源。
- 继承摘要混入“永久”“账号成长”“最大 MP 永久+”等局外暗示。

可验收条件：

- 下一战开始前或首屏中，至少显示 1 个来自奖励的继承项和 1 个来自路线的继承项。
- 继承项必须用玩家语言表达，不使用 raw `nextBattleContext`、`rewardPickBonus`、`modifierId`。
- 玩家能回答：“第二战哪里是我刚才造成的？”

核心体验打分影响：

- 短 run / 地牢推进：`+1`。节点推进开始有连续性。
- 局内牌成长：`+0.5`。如果继承项包含强化或临时资源，玩家能把它理解为本 run 内成长。
- 战斗决策与升序爆点：`+0.5`。继承后的牌组变化能让第二战继续围绕链路做选择。

### 镜头 3：路线真实影响奖励结构

玩家画面：

- 第二战后再次出现奖励时，奖励池明显带有所选路线的偏向。
- 走 `repair-cache` 时，玩家更容易看到清污、补桥、补给或额外 pick。
- 走 `elite-pressure` 时，玩家看到更强 payoff、资源授权或高风险高收益奖励。

玩家应理解：

```text
我刚才选的路线改变了奖励，不是战斗后随机又给三张。
```

失败信号：

- 路线说明写“奖励+1”，但第二战奖励没有任何可见变化。
- 奖励候选仍只回应上一战问题，不回应路线选择。
- 玩家无法区分不同路线后的奖励风格。

可验收条件：

- 至少一条路线能改变奖励数量、奖励标签、奖励排序或奖励解释。
- 奖励卡上有短原因，例如 `路线补给`、`高压回报`、`清污回应`、`资源补偿`。
- 同一上一战问题下，不同路线至少产生一个不同的奖励倾向。

核心体验打分影响：

- 奖励构筑回应：`+1.5`。奖励从“回应上一战”升级为“回应上一战 + 路线计划”。
- 短 run / 地牢推进：`+0.5`。路线有后续回声，玩家才会相信地图选择。
- 卡牌池机制覆盖与内容密度：`+0.5`。如果路线偏向能带出不同卡牌类型，可小幅加分。

### 镜头 4：路线真实影响资源或压力

玩家画面：

- 第二战开局，玩家能看到路线带来的资源或压力变化。
- 例子：
  - `repair-cache`：多一次 reroll、奖励多选一、清污资源、开局补给。
  - `elite-pressure`：本战 +1 临时 MP、敌意图更高、击败后奖励更强。
- 玩家能在第一回合根据这个变化调整出牌。

玩家应理解：

```text
这条路线不是只改奖励文案，它改了我这一战怎么打。
```

失败信号：

- 路线资源只在 run history 里，战斗首屏看不到。
- `+1 MP` 或补给没有改变实际可出牌选择。
- 高压路线只给收益，没有相应风险；或风险只扣血但不可预判。

可验收条件：

- 路线至少影响下一战一项可见战斗参数：临时 MP、起手资源、敌压、污染、奖励 pick、清污机会。
- 该参数必须在第二战首回合或奖励界面产生可观察差异。
- 如果路线带风险，End Turn 预览或敌意图必须提前告诉玩家风险。

核心体验打分影响：

- 短 run / 地牢推进：`+1`。路线成为真正节点，不是菜单。
- 敌意图、血量与失败压力：`+0.5`。高压路线如果可预判并有后果，压力维度可加。
- 局内牌成长：`+0.5`。临时资源或强化路线能让本 run 成长更可感。

### 镜头 5：第三节点前形成路线计划，而不是继续随机点

玩家画面：

- 到第三节点前，玩家能看到当前 run 的路线历史：奖励、路线、继承、风险或资源结果。
- 第三次选择不是重复流程，而是基于前两战形成计划：继续清污、追 payoff、保命、强化核心牌、打高压结算。

玩家应理解：

```text
我这局已经有方向了，接下来不是随便选最亮的按钮。
```

失败信号：

- 每次节点选择都像互不相关的三选一。
- 玩家不知道上一条路线是否赚到了。
- UI 只显示当前节点，不显示路线历史或本 run 计划。

可验收条件：

- 至少 3 个节点内，玩家能看到一条短历史：`拿了什么 -> 走了哪里 -> 下一战变了什么`。
- 第三节点候选至少一个回应当前 run 状态，而不是固定模板。
- 玩家能回答：“我现在这局是什么构筑方向？”

核心体验打分影响：

- 短 run / 地牢推进：`+1`。3 节点开始像短地牢，而不是两场 demo。
- 奖励构筑回应：`+0.5`。奖励和路线共同形成 build plan。
- 玩家反馈与操作手感：维持门槛；若历史难读，不能给短 run 加分。

### 镜头 6：最终节点结算

玩家画面：

- 到达最终节点后，不再继续生成路线候选。
- 系统给出明确结算：胜利、失败、撤离、或本次短 run 结束。
- 结算摘要说明本局关键选择：拿过的奖励、走过的路线、最终资源/污染/HP、核心强化或 payoff。

玩家应理解：

```text
这次 run 结束了；我能看出自己为什么赢、为什么亏，下一局想调整什么。
```

失败信号：

- 最终节点后仍生成下一路线，run 没有边界。
- 结算只是 `victory` 或 `complete` 字段，没有玩家语言。
- 结算不提路线贡献，玩家感觉前面的路线选择没有价值。

可验收条件：

- 最终节点完成后，路线候选清空，状态进入明确结算。
- 结算摘要至少包含 3 项：核心奖励、关键路线、最终后果。
- 玩家能回答：“这局最关键的选择是什么？下局我想换什么？”

核心体验打分影响：

- 短 run / 地牢推进：`+1.5`。有终点，才算完整短 run。
- 奖励构筑回应：`+0.5`。结算能回看奖励/路线是否解决问题。
- 体验边界与版权安全：不加分但必须保持；结算不能引入局外永久成长误导。

## 3. 一条合格第16轮玩家流程

最小合格流程必须覆盖：

```text
第一战
  -> 奖励选择：拿一个回应问题的奖励
  -> 路线选择：在补给 / 高压之间做选择
  -> 下一战预览：显示奖励 + 路线继承
第二战
  -> 路线 modifier 改变资源、压力或奖励倾向
  -> 第二次奖励体现路线影响
  -> 第三节点前形成构筑计划
最终节点
  -> 不再生成路线
  -> 显示本次 run 结算
```

如果只完成前半段，即“奖励后选路线、下一战继承”，但没有路线影响奖励/资源，也没有最终节点结算，第16轮最多只能记 `72 -> 76`。  
如果四个核心环节全部玩家可见，才允许记录 `72 -> 80`。

## 5. 一票否决项

- 奖励后没有路线选择，仍然直接进入下一战。
- 路线选择不影响下一战任何可见资源、奖励、压力或牌组状态。
- 路线只影响日志，不影响玩家首屏、奖励面板或出牌决策。
- 走不同路线后奖励结构完全一样，且没有短原因解释。
- 最终节点没有结算边界，仍像无限 demo。
- 结算把 run 内奖励描述成永久账号成长。
- QA 分数、截图数量、脚本通过被写成核心体验加分理由。

## 6. 交付给后续实现 agent 的优先级

P0：

1. 奖励选择完成后进入 route pending，而不是直接开始下一战。
2. 路线选择后写入下一战可见继承摘要。
3. 至少一条路线影响奖励候选，至少一条路线影响资源或敌压。
4. 最终节点完成后进入结算，不再生成路线候选。

P1：

1. 路线历史压成三段短 token：`奖励 / 路线 / 下一战结果`。
2. 第二战奖励原因同时显示上一战问题和路线来源。
3. 高压路线必须同时有收益和可预判风险。

P2：

1. 结算页增加“下局计划”短句，但不要做长教程。
2. 多做 1-2 条路线模板，先保证差异可读，再扩内容量。
