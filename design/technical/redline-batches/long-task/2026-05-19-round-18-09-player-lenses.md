## 0. 复测裁决

第17轮已经把 build plan 从“奖励/路线/升级存在”推进到“玩家能看到五类计划语言”：补桥、补终结、清污染、堆关键牌、补资源。第18轮不能再只证明这些标签可见，而要证明玩家在连续 3-5 个节点里会因为真实压力改变计划。

本轮 10 个镜头都按玩家当下体验判定：

```text
看到什么 -> 做什么选择 -> 压力来自哪里 -> build plan 怎么变
-> 失败时怎么理解 -> 是否更像竞品的机制结构
```

合格目标不是“每个节点都有奖励理由”，而是：

```text
玩家跑完 3-5 节点后，能复述自己为什么从 A 计划转到 B 计划，以及哪次失败是路线风险、污染、缺桥、缺终结或资源短缺造成的。
```

## 1. 10 个独立玩家视角镜头

### 镜头 1：节点 1 第一眼，玩家识别这局不是单场战斗

- 玩家当下看到什么：首战开局除了敌意图、手牌费用和链路提示外，还看到一个很短的 run 计划状态，例如 `当前计划：待识别`、`本节点目标：压低结束伤害并找出牌组短板`。界面不应只像一场独立遭遇。
- 做什么选择：玩家先按 0-1-2 或可用桥牌尝试压低意图，而不是马上追求最大伤害。若有目标选择，优先打会造成最大结束伤害的前排敌人。
- 压力来自哪里：压力来自本回合可见的结束伤害和有限手牌，而不是实时倒计时或系统自动打怪。
- build plan 怎么变：从 `待识别` 变成第一类短板，例如 `补桥候选`、`补终结候选`、`清污染候选` 或 `补资源候选`。
- 如果失败会怎么理解：如果第一战掉血，玩家应理解为“我没有把结束伤害压下来”或“我没接上链路”，而不是“系统突然扣血”。
- 是否更像竞品的机制结构：更像卡牌 roguelite 的首战诊断结构，也接近 Vampire Crawlers 的 hyper turn-based 压力；不像 Vampire Survivors 的自动输出生存结构。

### 镜头 2：节点 1 奖励，玩家在两个合理计划之间做取舍

- 玩家当下看到什么：战后奖励不是泛用强度列表，而是至少两个不同 build plan 回应，例如 `补桥：下战更容易接 0-1-2`、`补终结：提高终结候选`、`补资源：下战 Max MP +1，本次冒险`。
- 做什么选择：玩家根据刚才暴露的问题选一个方向。如果首战链路断在中段，就选补桥；如果链路已成但怪没死，就选补终结；如果漏伤害严重，则考虑补资源或清污染。
- 压力来自哪里：压力来自“下一节点会验证我选错没有”，而不是奖励稀有度本身。
- build plan 怎么变：从第一战诊断短板变成明确计划，例如 `主计划：补桥后找终结` 或 `主计划：先补资源承压，再堆关键牌`。
- 如果失败会怎么理解：如果下一战仍断链，玩家应理解为“这次奖励没有解决真正短板”或“我为了终结忽略了桥”。
- 是否更像竞品的机制结构：更像 Slay the Spire / Monster Train 式战后牌组修正结构；只要奖励能回应上一战问题，就不会像单纯开宝箱。

### 镜头 3：路线 1，玩家明确选择稳修还是高压回报

- 玩家当下看到什么：路线候选把当前计划改写成两种走法，例如 `维修补给：保留补桥计划，降低污染` 对比 `精英压力：+1 MP，但敌意图更高`。
- 做什么选择：玩家决定是否承受高压。如果当前牌组刚补到桥但缺终结，可能走高压拿资源赌终结；如果污染已经进牌库，则走维修补给稳定抽牌质量。
- 压力来自哪里：压力来自路线预告的下节点敌压、污染、奖励宽度或资源变化，而不是地图装饰。
- build plan 怎么变：选择稳修时，计划从 `补桥后找终结` 变成 `先稳定桥和清污染，再找终结`；选择高压时，变成 `用 +1 MP 抢节奏，失败会漏更多血`。
- 如果失败会怎么理解：如果高压路线失败，玩家应能把失败归因到“我接受了更高敌意图/污染风险”，而不是随机难度飙升。
- 是否更像竞品的机制结构：更像 roguelite 路线风险收益选择；比 Vampire Survivors 的连续波次更有节点策略，也比纯线性关卡更接近竞品卡牌结构。

### 镜头 4：节点 2 开局，上一节点选择必须被看见

- 玩家当下看到什么：下一战首屏能看到上一奖励和路线的实际带入，例如 `已拿 Pulse Draw`、`路线：高压 +1 MP`、`当前计划：用资源补桥，优先找终结`。这些信息要短，不应占据主操作区。
- 做什么选择：玩家先验证上一节点选择是否真的能改变出牌路线，比如用新增桥牌接上 0-1-2，或用 +1 MP 打出原本差 1 费的组合。
- 压力来自哪里：压力来自“验证窗口很短”：手牌有限、敌意图在结束回合结算、污染或缺终结会马上暴露。
- build plan 怎么变：如果上一选择有效，计划从 `补桥` 推进到 `找终结 / 堆关键牌`；如果无效，计划退回 `继续补桥 / 清污染`。
- 如果失败会怎么理解：失败应被解释为“上一节点选择没有解决当前战斗的短板”，而不是“奖励没有生效”。例如桥牌没抽到是抽牌质量问题，桥牌抽到但仍杀不掉是缺终结问题。
- 是否更像竞品的机制结构：更像卡牌 roguelite 的跨战验证；如果只显示路线记录但实战无变化，就只像 UI 日志，不像竞品机制。

### 镜头 5：节点 2 中段，污染或坏手迫使玩家偏离原计划

- 玩家当下看到什么：手牌中出现污染、无效牌或费用断点，HUD 用短句说明机会成本，例如 `污染占手：本回合少一张桥牌`、`缺 1 费桥，终结无法放大`。
- 做什么选择：玩家选择临时修补：抽牌、清污染、保留桥牌、放弃终结转为减伤，或提前结束回合承受较小损失。
- 压力来自哪里：压力来自抽牌质量下降和结束伤害预览，而不是单纯敌人数变多。
- build plan 怎么变：从原来的 `补桥后找终结` 改成 `先清污染 / 先补资源 / 下一奖励继续找桥`。如果玩家原本走高压，这里应出现“高压路线的代价兑现”。
- 如果失败会怎么理解：玩家应理解为“污染或坏手破坏了计划，我需要修正牌组”，不是“这手牌随机烂所以输”。
- 是否更像竞品的机制结构：更像 Slay the Spire 的状态牌/坏手压力，也保留 Vampire Crawlers 的短回合连续修链；如果污染不进手牌，只是日志数字，就不像竞品核心。

### 镜头 6：节点 2 结算，失败或半失败要生成下一步修正

- 玩家当下看到什么：战后不只显示胜负，而是显示本节点 plan 结果，例如 `计划半成：桥接上了，缺终结漏 9` 或 `计划失败：污染占手导致断链`。
- 做什么选择：玩家在奖励或路线里选择修正入口：补终结、清污染、补桥、补资源，或继续堆关键牌但承认风险。
- 压力来自哪里：压力来自 run 内代价开始积累：血量下降、污染增加、路线风险、下一节点敌压变高。
- build plan 怎么变：计划必须出现一次明确转向，例如 `补桥` 成功后转 `补终结`，或 `高压抢资源` 失败后转 `清污染/降压`。
- 如果失败会怎么理解：如果玩家仍然看不懂失败原因，只看到扣血，本镜头不合格。失败要能被玩家说成一句话：`我为了 MP 走高压，但污染让我没抽到桥`。
- 是否更像竞品的机制结构：更像竞品的 run 复盘和下一战修正；如果每战都独立开奖，就不像卡牌 roguelite 的连续构筑。

### 镜头 7：节点 3，build plan 进入“坚持”还是“转向”的选择

- 玩家当下看到什么：第 3 节点前，界面能展示当前 plan 已经累积的证据，例如 `桥已稳定 2 次`、`终结仍缺`、`污染 2 张未清`、`关键牌已升级 1 次`。
- 做什么选择：玩家决定坚持原计划堆关键牌，还是转向解决最痛短板。如果终结仍缺，选补终结；如果污染已经影响两次抽牌，优先清污染。
- 压力来自哪里：压力来自机会成本：继续堆关键可能爆发更爽，但短板不修会在本节点失败。
- build plan 怎么变：从单一标签变成优先级队列，例如 `主：补终结；次：清污染；放弃：继续堆桥`。
- 如果失败会怎么理解：玩家应理解为“我坚持原计划但没有处理短板”，或者“我转向太晚”，而不是“敌人血量太高”。
- 是否更像竞品的机制结构：更像竞品中期 deck shaping 的结构；玩家开始在 archetype 和生存修补之间权衡，而不是只按最高数值。

### 镜头 8：节点 3 战斗爆点，payoff 必须像计划兑现而不是随机清场

- 玩家当下看到什么：玩家通过前两节点的选择终于接出一条链：启动、桥、资源/抽牌修补、终结或关键牌 payoff。画面和文案要让玩家知道这次爆发来自自己的 build plan。
- 做什么选择：玩家选择是否立即打终结，还是先再打一张桥/抽牌追求更大 payoff。这个选择应有风险：贪一张可能断链或漏伤害。
- 压力来自哪里：压力来自“现在收掉还是继续贪”的短期判断，以及结束回合预览仍在逼近。
- build plan 怎么变：成功时从 `补终结` 推进到 `堆关键牌 / 强化 payoff`；失败时从 `贪爆点` 退回 `补资源 / 稳定终结`。
- 如果失败会怎么理解：如果玩家贪连锁导致没清掉，应理解为“我选择了更高收益但错过收束窗口”。如果终结牌未出现，应归因到缺终结或抽牌质量，而不是系统不给爽点。
- 是否更像竞品的机制结构：更像 Vampire Crawlers 的 hyper turn payoff 和卡牌 roguelite 的构筑兑现；只要 payoff 依赖前置链路，就不像 Vampire Survivors 的自动大招。

### 镜头 9：节点 4，可选高压节点验证 build plan 的承压能力

- 玩家当下看到什么：第 4 节点前出现一个明显的高压选择：更高敌意图、更强前排、更多污染或更窄奖励，但有更高收益。界面应说明“当前 plan 能否承压”的判断依据。
- 做什么选择：如果玩家已经补资源并有终结，可以挑战高压；如果污染未清或桥不稳，应该避开或选择维修路线。
- 压力来自哪里：压力来自已知风险，不是隐藏惩罚。玩家要看到当前血量、污染、桥稳定度和终结概率会如何影响高压选择。
- build plan 怎么变：选择高压时，计划变成 `用当前成型链路兑现收益`；选择避开时，计划变成 `先修短板，再找爆点`。
- 如果失败会怎么理解：失败应被解释为“plan 承压不足”，并指出具体薄弱点：缺资源、缺桥、污染多、终结不稳定或目标优先级错。
- 是否更像竞品的机制结构：更像精英房/高压节点的风险收益结构；如果高压只是怪更多、奖励更多，而不考验 build plan，就仍然不像竞品深层结构。

### 镜头 10：节点 5 或阶段结算，玩家能复述整条计划线

- 玩家当下看到什么：阶段结算用 4-5 条短句回看连续节点：`首战暴露缺桥`、`奖励补桥`、`高压路线带来污染`、`节点 3 转补终结`、`最终 payoff 成功/失败原因`。
- 做什么选择：玩家选择下一阶段优先方向：继续堆关键牌、清污染、补资源、补桥、找终结，或者在失败后降低风险。
- 压力来自哪里：压力来自下一阶段会继承本 run 的部分后果：牌组质量、血量、污染、关键牌升级和路线记录。
- build plan 怎么变：结算不是终点，而是把当前 plan 压缩成下一阶段入口。例如 `已成型：桥稳定，下一步补终结密度` 或 `未成型：污染过高，下一步先清牌`。
- 如果失败会怎么理解：玩家要能说清楚“我输在什么计划层面”，而不是只知道某场战斗没打过。失败原因应落在五类 build plan 或路线风险上。
- 是否更像竞品的机制结构：更像竞品的 run 记忆和自我复盘；如果结算只列分数、击杀和奖励，就不像卡牌 roguelite 的学习闭环。

## 2. 必须落地的体验改动建议

### P0-1：每个节点都要记录并显示一句 build plan 状态变化

从第 1 节点开始，玩家每经过战斗、奖励、路线、升级或结算，都必须看到一句短状态变化，例如 `缺桥 -> 已补桥，下一战找终结`。没有这条连续状态，3-5 节点会退化成独立战斗和独立奖励。

### P0-2：失败必须归因到五类 build plan 或路线风险

掉血、断链、污染卡手、终结没来、目标杀错，都要被归因到 `补桥 / 补终结 / 清污染 / 堆关键牌 / 补资源 / 路线风险` 中的一个或多个。失败如果只表现为扣血，本轮不能算玩家体验成立。

### P0-3：路线选择必须实质改写下一战 plan

路线不能只给奖励宽度、MP 或敌压数值。每条路线都要改写玩家当前 plan：加强、延后、冒险、修复或转向。下一战首屏必须能看到这次改写被带入。

### P1-1：污染和坏手要在 3-5 节点内至少一次迫使玩家转向

污染不能只做计数或日志。它至少要在一个节点中造成抽牌质量下降、桥牌缺失、终结延迟或资源成本，并让玩家在后续奖励/路线中转向清污染或补资源。

### P1-2：阶段结算要做 4 条以内的计划复盘

结算页应回看 `原计划 / 路线变化 / 验证结果 / 下一步补点`，控制在 4 条以内。它的目的不是教学，而是让玩家确认这 3-5 个节点是一条连续构筑线。

## 3. 验收口径

第18轮玩家体验镜头通过的最低标准：

- 10 个镜头中至少 8 个能被玩家用一句话口述原因。
- 连续 3 个节点内至少出现一次 plan 转向。
- 连续 5 个节点内至少出现一次可解释失败或半失败。
- payoff 爆点必须能被追溯到前序奖励、路线或升级选择。
- QA 只能证明门禁通过，不能替代玩家对 build plan 演变的理解。
