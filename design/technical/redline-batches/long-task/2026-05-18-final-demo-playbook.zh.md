# 2026-05-18 最终玩法说明：Redline 核心循环 Demo

## 当前 Demo 是什么

这是一个回合制卡牌战斗切片。玩家每回合先读敌人意图和 `结束-N`，再用有限 MP 把手牌按 `0 -> 1 -> 2` 接起来。接链成功后会获得本回合临时授权，授权可以支付 3 费终结牌，把高压回合一次性压下去。战斗中的奖励会进入下一手或后续抽牌，用来修正刚才暴露的牌序问题。

这不是完整 roguelike 卡牌游戏，也不是完整 1:1 机制底座。当前交付重点是证明核心循环、压力读数、奖励反馈、受限整备和移动端 UI 可验收。

## 玩家每回合看什么

1. 看敌人意图和 `结束-N`：如果现在结束回合，会扣多少血。
2. 看 MP：当前能不能负担这手牌的关键费用。
3. 看链路：当前是否能按 `0 -> 1 -> 2` 接上，还是会断链。
4. 看目标：单体牌默认打前排最高压力目标，群体牌看总意图下降。
5. 看授权：完成链路后出现 `授权+3`，它只服务终结牌，不是永久 MP。
6. 看奖励：选牌后要进入下一手或后续抽牌，解决刚才的断链、缺 MP、缺终结问题。

## 当前可玩的机制

- 默认目标：单体攻击优先瞄准一线高敌意图目标。
- 费用链：0 费开链、1 费承接、2 费展开，断链仍可出牌但收益回落。
- 临时授权：完成链路后获得 `授权+3`，可支付 3 费终结牌。
- 终结牌：授权支付后形成清场或大幅减压窗口。
- Wild 修补：只有真实修补当前链路缺口时，才触发当前 MP 回补。
- 抽牌修补：`抽N仍-X` 表示抽牌是找解，不是立刻减压。
- 奖励三选一：奖励牌进入牌组并影响下一手。
- 受限整备：`paper_shatter` 只从抽牌堆找第一张 payoff 置顶，命中时 HUD 显示 `整备：顶终结`。

## 当前不承诺的机制

- 不承诺完整牌库搜索或手动重排。
- 不搜索弃牌堆。
- `lantern_captain` 当前不触发真实整备。
- 不承诺消耗、保留、状态牌、升级、删牌、遗物、商店、路线节点和局外成长。
- 不把授权或奖励写成永久成长。

## 推荐演示路径

```text
发牌
-> 读敌意图和结束伤害
-> 打 0 费开链
-> 打 1 费承接
-> 打 2 费展开拿授权
-> 用授权支付 3 费终结
-> 进入奖励三选一
-> 选择能修正下一手缺口的奖励
-> 展示 paper_shatter 的“整备：顶终结”样片
```

## 当前最需要复测的问题

- 玩家是否能在 3 秒内读出 `结束-N` 和下一张费用。
- 玩家是否真的在有限手牌中做选择，而不是按提示看机制演示。
- 断链或抽牌未找解时，失败后果是否足够清楚。
- 授权终结是否来自前面接链铺垫，而不是像系统赠送按钮。
- 奖励是否能回应上一手的真实缺口。
