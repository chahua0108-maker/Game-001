| 缺口 | 预计仍缺分 | 原因 |

| 玩家复测与内容密度 | 3-6 | 3-5 回合样片还不是完整可反复游玩的 card roguelike。 |

硬停止条件不变：

```text
score >= 95
AND npm run check pass

AND qa:ui pass
AND lifecycle evidence pass
AND no P0 blocker
AND no mobile overflow / raw token leak

```

## 5. 第14轮验收裁决规则

1. 没有自动化证据时，只能记录“预期分”，不能记录“最终验收分”。
2. 生命周期 runtime 存在但玩家看不到，生命周期相关加分最多拿 60%。
3. HUD token 可见但 runtime 事件不真实，生命周期相关加分最多拿 40%。
4. 状态/污染牌如果能推进 `0 -> 1 -> 2` 或授权，视为 P0，状态牌相关加分归零。
5. 保留如果突破手牌上限、跨 reward 或跨 node，视为 P0，保留相关加分归零。
6. 消耗如果进入 discard 或参与洗回，视为 P0，消耗相关加分归零。
7. 任何第 12 轮核心回归退化，新机制加分不能抵消旧分扣分。
8. 移动端出现横向溢出、关键文本超框或 raw enum 泄漏，本轮总分封顶 `76`。
9. 浏览器 cleanup 失败，本轮总分封顶 `79`。
10. 出现竞品受保护表达复制，本轮总分封顶 `70`，严重时直接 fail。
