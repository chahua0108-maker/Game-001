## 3. 不可加分项

以下项目可以作为门禁、证据或工程卫生，但不得直接增加核心体验分：

| 项目 | 处理方式 |
| --- | --- |
| `npm run check` 通过 | 门禁；失败则本轮不能合格，通过不加分。 |
| `qa:similarity`、`qa:ui`、`qa:lifecycle` 通过 | 门禁；证明没有退化，不等于核心体验更像竞品。 |
| 浏览器 page/context/browser/server cleanup 正常 | 门禁；避免残留端口和假红，不加分。 |
| 新增 scorecard、report、trace、JSON 证据 | 证据可追溯，不加分。 |
| 只新增 run/node 字段但玩家看不到路线推进 | 不加分。 |
| 只新增奖励池数据但不进入下一战 | 不加分。 |
| 只新增卡牌、关键词、稀有度、数值标签 | 不加分；除非它们直接改变短 run 中的构筑回应。 |
| 只把 cardUpgrades 继续停在 sim 层 | 不加分；玩家不能选择、预览、继承时仍算未进入体验。 |
| UI 防溢出、移动端不超框 | 门禁；除非新增反馈显著解释“为什么进入下一节点/下一战”。 |
| 版权安全声明 | 必须保持，但满分项不能重复加分。 |

## 4. 第15轮验收门槛

第15轮如果要记录 `72 / 100`，至少满足以下体验门槛：

1. 玩家完成一场战斗后能进入奖励选择，而不是只看一段结算文案。
2. 奖励选择会改变当前 run 状态，并在下一场战斗中产生可见影响。
3. 至少有一个路线、节点、下一战或胜败边界让玩家知道自己处在同一次短 run 中。
4. 下一战不是完全重置样片；至少继承一项构筑变化、污染压力、牌成长或牌组变化。
5. 玩家能读懂“为什么进入下一节点、为什么奖励有用、为什么下一战变了”。
6. 不复制竞品卡名、原文、美术、素材或 UI 构图，只对齐机制结构和体验节奏。

QA 门禁仍需通过，但 QA 结果只能决定“本轮可否验收”，不能把 `72` 推到更高。
