## 0. 本轮裁决

第 15 轮短 run 的玩家验收不应再只看“战斗样片是否成立”，而要看玩家是否自然经历这条闭环：

```text
第一战承压
  -> 用现有牌组解决或失败
  -> 奖励回应上一战问题
  -> 走节点并承担节点后果
  -> 污染 / 强化 / 新牌进入本次 run
  -> 下一战开局牌组真的变化
  -> 强化牌或回应牌被打出并改变局势
```

玩家能说出“我为什么拿这张奖励、为什么走这个节点、下一战我的牌组哪里变了”，才算核心体验加分。脚本能跑通、截图不超框、浏览器能关闭，只是门禁。

## 1. 10 个必须出现的短 run 镜头

| # | 镜头 | 玩家画面 | 玩家应理解 | 失败信号 | 可验收条件 |

| 1 | 第一战首屏承压 | 第一战开局，HP、MP、4 手牌、敌意图、End Turn 预计扣血同屏可见。至少有一名敌人正在施压。 | “我现在如果不处理，会掉多少血；这一手不是随便出牌，是先解压力。” | 玩家只看亮牌，不知道结束回合会发生什么；敌意图像背景数字。 | 开局 5 秒内可见 `enemyIntentSummary.totalDamage > 0` 或等价意图；End Turn 前后伤害预览一致；玩家口述能指出最大威胁和预计扣血。 |
| 2 | 第一战成功链启动 | 玩家用 0 MP 起手，再接 1 MP / 2 MP，看到倍率或链路反馈逐步上升。 | “我是在铺 `0 -> 1 -> 2`，不是只按费用从低到高乱点。” | 0 费只被理解成免费按钮；1/2 费承接没有手感差异。 | 事件顺序出现 0、1、2 三段；`effectMultiplier` 至少呈 `1 -> 2 -> 3`；HUD 或日志有短反馈如 `开链`、`接x2`、`授权+3`。 |

## 2. 最小验收脚本

一条合格短 run 至少需要 2 场战斗和 1 次节点选择：

```text
第一战
  镜头 1：承压首屏
  镜头 2：成功链启动
  镜头 3：承压失败或半失败
奖励
  镜头 4：奖励回应上一战问题
节点
  镜头 5：走节点
  镜头 6：污染被回应，或镜头 7：选择局内强化
第二战
  镜头 8：开局牌组变化
  镜头 9：强化牌 / 回应牌打出
结算
  镜头 10：形成下一步构筑欲望
```

如果本次短 run 没有自然产生污染，也不能跳过“污染问题被回应”的验收口径；应安排一个固定承压种子或半失败分支，让污染至少出现一次并被处理一次。否则第 15 轮只能证明奖励和强化，不能证明压力进入牌库后的 run 闭环。

## 3. 玩家口述验收

每个镜头的人工验收只问一句话，不做教程引导：

| 镜头 | 观察者提问 | 合格回答 |
| ---: | --- | --- |
| 1 | “你现在最怕什么？” | 指向敌意图和预计扣血。 |
| 4 | “为什么拿这张？” | 能关联上一战问题。 |
| 5 | “为什么走这个节点？” | 能说明下一战收益或风险。 |
| 6 | “污染怎么被处理了？” | 能指出清理、稀释或规避方式。 |
| 7 | “强化了谁？” | 能指出具体牌和本 run 内效果。 |
| 8 | “下一战哪里变了？” | 能指出新牌、强化、污染变化或节点结果。 |
| 9 | “这张牌为什么现在更强？” | 能关联前面的强化 / 奖励。 |
| 10 | “下一步你想补什么？” | 能说出构筑方向，而不是“随便选”。 |

玩家说不出原因时，即使自动化全绿，也只算门禁通过，不算核心体验通过。

## 4. 一票否决项

- 短 run 只有一场战斗，没有奖励到下一战的继承。
- 奖励不回应上一战问题，只是固定三张。
- 节点选择没有改变第二战的可见状态。
- 污染出现后没有任何处理路径，或污染能推进 `0 -> 1 -> 2` 链路。
- 强化被表现成永久成长，或重开 run 后仍保留。
- 第二战开局无法看出牌组、污染、强化、节点结果中至少一项变化。
- QA 报告被写进分数理由。QA 只能证明没有阻塞，不能替代玩家体验判断。
