# Round 15-05 短 run UI/HUD 体验设计与实现记录

负责范围：

- `prototype-web/src/ui/hud.ts`
- `prototype-web/src/style.css`
- `prototype-web/src/tests/ui/hud-target-selection.test.ts`

## 目标

让玩家在短 run 中能直接读到三件事：

1. 当前节点位置。
2. 战斗后奖励候选与路线候选。
3. 下一战会继承什么。

QA 不计入分数，但 HUD 读不懂会扣核心体验分，所以本轮优先压短 token，并避免文字撑框。

## 实现

### Run Layer

`hudRunLayerState` 从旧的“局外档案占位”改成当前 run 信息：

- `节点 1/3`
- `奖励候选 修补/终结/路线`
- `路线候选 Spark Tap` 或 `路线候选 0`
- `下一战 / 选1入组`
- `牌组N · 仅本run`

未进入奖励态时显示：

- `奖励 0`
- `路线候选 待奖励`
- `继承当前牌组`

已选过奖励时显示：

- `已拿 <card name>`
- `带入 <card name>`

### Reward Panel

奖励面板 header 增加 run 状态短句：

- 当前节点。
- 奖励候选分支。
- 路线候选。
- 下一战带入动作。

奖励卡继续使用已有结构，但样式改成稳定 grid，并对 header small、reward-card span/small/strong/em 做省略或两行限制，避免长卡名、长机制说明把卡片撑开。

### 测试

新增/更新 `hudRunLayerState` 单测：

- 当前 run 节点与已拿奖励显示。
- reward pending 时显示 `奖励候选 修补/终结/路线`。
- route 候选能识别 `route-bridge` 奖励牌。
- 下一战带入信息显示 `选1入组` 与 `牌组N · 仅本run`。
- 不出现 `永久`、`永久升级`、`最大 MP +3` 等误导局外成长词。
