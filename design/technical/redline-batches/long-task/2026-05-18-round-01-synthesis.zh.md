# 2026-05-18 第 1 轮综合结论

## 轮次结果

第 1 轮已新增 10 个独立专家视角，全部落到 `design/technical/redline-batches/long-task/`：

1. 核心循环导演：当前最缺牌面级意图变化预览，玩家不知道一张牌能把结束回合伤害从多少降到多少。
2. 卡牌机制复刻负责人：P0 已有费用链、临时授权、payoff、奖励三选一；后续完整复刻要补消耗、保留、状态、实例、触发器和效果解释器。
3. 卡组内容架构师：当前 16 张卡能验证链路，但不像完整牌池；P0 不应先大扩卡，而应先让过牌、修补、payoff 和奖励选择可读。
4. 效果引擎工程师：现有 `Intent -> Event -> Rule -> Command` 可保留；后续应逐步把卡牌效果迁到数据驱动解释器。
5. UI 溢出 QA：最高风险是移动端固定定位层互相抢高度，尤其手牌多行、run-layer、enemy-peek、reward 面板。
6. 移动端交互设计师：移动端主路径被顶部控制台、远距离结束按钮和小目标按钮打断；P0 先修主操作线和触控尺寸。
7. 数值与压力曲线分析师：压力位置不在实时倒计时，而在敌人意图；当前奖励太晚、未武装 payoff 偏强，但本轮可先用意图变化验收。
8. 敌人意图设计师：当前敌人只有 attack；本轮先统一意图真相源、前排显示本回合/下轮、默认目标可预测、卡牌显示 intent delta。
9. 自动化验收工程师：机制 Vitest 覆盖较厚，最大缺口是真实 DOM/CSS 布局验收和替代旧 90 秒验收的固定短切片。
10. 制作人/PM：第 1 轮要收敛成 P0 Hyper-Turn 样片，不扩局外成长、不扩大量新系统。

## 本轮裁决

本轮不做完整效果解释器、不新增大量卡牌、不做永久 Max MP、不恢复实时心跳压力。

本轮只落地三个窄改动：

- 单体牌未手动选目标时，不再随机前排，改为默认锁定本回合最高敌意图前排。
- 卡牌按钮显示“打出后本回合意图变化”，让玩家看到 `意图 17 -> 12` 这样的卡牌后果。
- 移动端修复容易超框/遮挡的 HUD：横向手牌、44px 结束/目标按钮、奖励面板可滚动、局外占位移动端降噪。

## 验收口径

- 正确链路下，玩家不用打开 debug 就能看到敌意图变化。
- 未选择目标时，UI 预览和 runtime 默认目标一致，不再出现随机目标导致预览失真。
- 移动端不依赖 hover/title 读取核心后果。
- 桌面与 390x844/360x640 验收时，卡牌、奖励、敌人前排和 run-layer 不出现关键文字超框或按钮不可点。
- `npm test -- --run` 与 `npm run build` 必须通过。
