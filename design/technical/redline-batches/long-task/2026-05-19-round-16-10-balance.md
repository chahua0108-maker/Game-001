## 0. 结论

第15轮新增内容整体方向可保留，但有两个需要收口的数值风险：

1. `crimson_receipt` 过强：1 MP、基础 8、原升级曲线 `maxLevel: 3` + `damagePerLevel: 3`，满级仅靠等级就到 17 单体伤害，已经接近或越过 3 MP 终结牌的定位。
2. `elite-pressure` 路线过强：如果每次选择都把 `maxEnergy` 直接 `+1`，短 run 连选两次会从 3 MP 堆到 5 MP，削弱 0->1->2 授权链的核心压力。

本轮已做小改：

- `crimson_receipt.runUpgrade` 从 `3 x +3` 下调为 `2 x +2`。
- `maxEnergyThisRunPlusOne` runtime 应用从可重复 `+=1` 收口为当前 run 最多推到 `4`。
- 新增 focused test 锁住第15轮新增卡数值带宽和路线经济 modifier 不叠加失控。

| `crimson_receipt` | 1 MP 基础 8 已经高于多数桥牌，原升级曲线会把它推成低费核心输出。 | 下调升级曲线 |

`crimson_receipt` 调整后仍是“强化受益牌”，但满等级基础伤害为 `8 + 2*2 = 12`，低于 `severance_burst` 的 3 MP 全场 16，避免低费桥牌挤占终结牌价值。

## 2. 路线 Modifier 审查

`repair-cache` 的 `rewardPickBonus: 1` 是小型奖励控制增益，当前可接受。  
`elite-pressure` 的 `maxEnergyThisRunPlusOne` 更危险，因为 MP 上限会直接改变出牌压力；本轮将 runtime 应用改成“最多到 4”，避免重复选择堆到 5。

保留的平衡边界：

- 每个路线候选最多只带一个经济杠杆。
- `rewardPickBonus <= 1`。
- `maxEnergyThisRunPlusOne` 不能多次叠加。
- 3 MP payoff 仍应是授权链的主要爆点，不被 1 MP 强化牌替代。

下一轮平衡重点不应继续加卡，而应看真实路线闭环中的选择率和失败压力：
