## 0. 总裁决

第 13 轮生命周期 v1 的内容目标不是扩卡池，而是用 3 张牌证明三个牌区事实：

```text
消耗：牌可以离开抽弃循环。
保留：牌可以跨到下一手，但占手牌位。
状态 / 污染：负面牌是物理卡，会占抽牌和手牌节奏。
```

三张牌必须全部是 Redline 自有表达，不复制竞品卡名、规则文案、图标、美术构图、职业包装或固定数值模板。允许使用卡牌游戏通用概念，例如消耗、保留、状态、抽牌、弃牌、费用链；但玩家读到的主题、文本和镜头必须围绕 Redline 的词：

```text
链路、意图、修补、整备、授权、终结、当前 MP、污染、清算
```

本轮不新增大卡包，不做升级、删牌、商店、遗物、局外成长、永久诅咒、完整净化生态、CardInstanceId 迁移或新 UI 面板。生命周期 v1 的内容只服务连续 3-5 回合样片：玩家能看到坏手、修补、授权、payoff、高压意图和污染风险。

## 1. 三张原创牌

### 1.1 消耗牌：`short_fuse_permit` / Short Fuse Permit / 短熔许可

定位：一次性 0 MP 开链补丁。  
核心用途：当玩家手里有 1/2 段但缺 0 段时，提供一次 `0 > 1 > 2` 的入口；打出后消耗，避免它把整局稳定性永久抬高。

| 字段 | 建议值 |
| --- | --- |
| 内部 id | `short_fuse_permit` |
| 展示名 | Short Fuse Permit / 短熔许可 |
| 类型 | `attack` 或 `skill`，推荐先用 `attack` 贴近当前运行时 |
| 费用 | 0 MP |
| 目标 | 前排单体 |
| 基础数值 | 造成 2 点伤害 |
| 链路角色 | `starter` / 0 段入口 |
| 生命周期 | `onPlay: exhaust` |
| 关键词 | `开链`、`消耗` |
| 卡面短文案 | `造成2。开链。消耗。` |
| 移动端短效 | `单体2 消耗` |
| 奖励位置 | P1 reward 后备，不进起始牌组 |

设计边界：

- 它不是强攻击牌，伤害必须低于 `debt_hook`。
- 它不是永久入口密度提升；消耗后不参与 discard 回填。
- 它不提供当前 MP、不抽牌、不顶 payoff。
- 它可以帮助 `0 > 1 > 2` 成立，但不能直接给 payoff 额外授权。

### 1.2 保留牌：`held_lane` / Held Lane / 留档车道

定位：可跨回合保留的 2 MP 展开段。  
核心用途：让玩家在本回合无法完成链路时，把关键 2 段留到下一手；它提高计划感，但通过占手牌上限付出代价。

| 字段 | 建议值 |
| --- | --- |
| 内部 id | `held_lane` |
| 展示名 | Held Lane / 留档车道 |
| 类型 | `attack` |
| 费用 | 2 MP |
| 目标 | 前排 |
| 基础数值 | 对前排造成 4 点伤害 |
| 链路角色 | `expand` / 2 段授权前置 |
| 生命周期 | `onTurnEnd: retain`，`onPlay: discard` |
| 关键词 | `授权`、`保留` |
| 卡面短文案 | `前排4。保留。完成链给授权。` |
| 移动端短效 | `前排4 保留` |
| 奖励位置 | route-bridge 后备，不早于首奖 |

设计边界：

- 它必须弱于 `row_cleave` 的前排伤害，因为保留本身就是强稳定性。
- 保留只跨到下一次 `DealHand`，不跨奖励选择，不跨节点，不跨 run。
- 保留牌占下一手手牌位，不提高发牌数，不绕过手牌上限。
- v1 不做“保留后降费”“保留后升级”“保留触发抽牌”。

### 1.3 状态 / 污染牌：`signal_smear` / Signal Smear / 噪点传票

定位：物理污染牌。  
核心用途：让敌人、风险牌或未来节点把一张无收益卡塞进玩家循环，制造抽牌质量下降；玩家可以打出它来清掉，但它不参与费用链。

| 字段 | 建议值 |
| --- | --- |
| 内部 id | `signal_smear` |
| 展示名 | Signal Smear / 噪点传票 |
| 类型 | `status` |
| 费用 | 0 MP |
| 目标 | 自身 |
| 基础数值 | 无伤害、无抽牌、无 MP |
| 链路角色 | 不参与链路，`countsForChain: false` |
| 生命周期 | `onPlay: exhaust`，`onTurnEnd: discard`，`purgeAt: node-end` |
| 关键词 | `状态`、`污染`、`消耗` |
| 卡面短文案 | `无效果。不接链。打出后消耗。` |
| 移动端短效 | `污染 消耗` |
| 进入方式 | v1 只允许测试、敌人意图或风险牌创建；不进 reward pool |

设计边界：

- 它不能作为 0 MP 开链牌，否则状态牌会反向增强 `0 > 1 > 2`。
- 打出它不推进 chain、不触发倍率、不触发 payoff 授权。
- v1 建议打出后不打断当前 chain，避免污染清理变成过重惩罚；真正惩罚是占据抽牌 / 手牌位置。
- 回合末未清理则进 discard，未来会洗回；节点结束时统一 purge，避免 v1 变成永久诅咒系统。

## 2. 如何服务 `0 > 1 > 2 / Wild / payoff`

| 机制位置 | `short_fuse_permit` | `held_lane` | `signal_smear` |
| --- | --- | --- | --- |
| 0 段入口 | 提供一次低伤害开链。 | 不参与。 | 明确不能当 0 段。 |
| 1 段桥接 | 给现有 1 MP 牌创造接上的机会。 | 依赖现有 0/1 段铺垫。 | 通过占手牌让 1 段更难自然出现。 |
| 2 段展开 | 消耗牌帮玩家到达 2 段。 | 自身就是可保留的 2 段。 | 可能挤掉 2 段，制造坏手。 |
| Wild 修补 | 降低缺 0 的事故，不替代 Wild。 | 让 Wild 有机会在下一手补 1 或补缺口。 | 让 Wild / draw 的修补价值更可见。 |
| payoff 授权 | 只帮助完成授权前置，不直接增强 payoff。 | 作为弱伤害 2 段，帮助下回合拿授权。 | 不给授权；污染会推迟 payoff 爆发。 |
| 牌区生命周期 | 打出后进 `exhaustPile`。 | 回合末进 `retainedCards`，下次发牌回 hand。 | 在 draw/hand/discard 中流动，打出后消耗，节点结束清理。 |

一句话：消耗牌给一次入口、保留牌给一次计划、污染牌制造一次未来手牌压力。三者都必须让玩家更关心 `0 > 1 > 2` 和 payoff，而不是绕开核心循环。

## 3. 卡牌镜头与数值风险

| # | 卡牌镜头 | 目标体验 | 数值风险 | 推荐护栏 |
| ---: | --- | --- | --- | --- |
| 1 | 玩家手里有 `redline_cut / row_cleave`，但缺 0 段，抽到 `short_fuse_permit`。 | 一次性补入口，顺利打出 `0 > 1 > 2`。 | 0 段过密会让链路太稳定。 | 首轮最多 1 张；不进起始牌组。 |
| 2 | `short_fuse_permit` 打出后进入消耗区，本节点后续不再洗回。 | 玩家理解“这张是急救许可，不是永久起手”。 | 如果消耗区不可见，玩家会以为牌丢了。 | HUD 只给短 token：`消耗` / `移出循环`。 |
| 3 | 玩家用 `short_fuse_permit -> heartbeat_spark -> row_cleave` 完成授权，但没有 payoff。 | 证明它只铺垫，不自动爆。 | 如果同时给 topdeck 或抽牌，会变成单卡答案。 | 不附带抽牌、不顶牌、不返 MP。 |
| 4 | 玩家本回合抽到 `held_lane` 但已经花掉 MP，回合末它被保留。 | 保留让玩家计划下一手，不是当回合硬解。 | 保留过多会减少随机性，变成背板。 | v1 保留牌数量建议最多 1 张；保留占手牌位。 |
| 5 | 下一手 `held_lane` 先进入手牌，再补抽到 4 张。 | 玩家看见“我带着 2 段准备开局”。 | 如果保留不占手牌位，会变成免费额外抽牌。 | 手上 1 张保留时只补抽 3 张。 |
| 6 | 玩家用 `0 -> 1 -> held_lane` 完成 2 段授权。 | 保留牌服务核心链路。 | 4 前排伤害乘倍率后仍可能接近 `row_cleave`。 | 基础伤害 4，不高于 5；不附带额外收益。 |
| 7 | 奖励选择后手牌被清理，`held_lane` 不跨奖励界面保留。 | 保留是回合内计划，不是跨节点存牌。 | 如果跨奖励，会让玩家囤关键 2 段。 | `reward-selected-discard` 强制清理 retained。 |
| 8 | 敌人或测试把 `signal_smear` 加入 discard，之后洗回并被抽到。 | 污染是未来手牌压力。 | 注入太早会让新手首回合无解。 | 3-5 回合样片中最多 1 张污染；首回合不注入。 |
| 9 | `signal_smear` 占了一个手牌位，玩家少了一张桥接牌。 | draw / Wild 修补的价值上升。 | 污染过多会让玩家觉得系统不给牌。 | 单节点污染密度不超过当前 deck 的 12%-15%。 |
| 10 | 玩家打出 `signal_smear`，它消耗，但不推进 chain。 | 清污染是节奏修复，不是免费开链。 | 若误计为 0 段，会强化 payoff 速度。 | `countsForChain=false`，并有测试锁住。 |
| 11 | 玩家不打 `signal_smear`，回合末它进 discard，之后可洗回。 | 状态牌有物理存在，不是一次 UI debuff。 | 长期循环会拖慢样片节奏。 | 节点结束 purge；v1 不做永久诅咒。 |
| 12 | 玩家用 Wild 修补坏手，但污染牌仍占手牌，payoff 没能当回合出现。 | 修补不是万能救场，失败也成立。 | 如果所有污染都免费清且不影响链路，状态牌太弱。 | 污染主要惩罚抽牌质量；不额外加 HP 惩罚。 |
| 13 | `short_fuse_permit` 和 `held_lane` 同时出现。 | 玩家能选“现在开链”或“留 2 段到下手”。 | 两张都过强会让 0/2 段事故消失。 | 消耗牌低伤、保留牌低伤；两者不在同一首奖层密集出现。 |
| 14 | `held_lane` 被污染挤压后仍保留，下一手少补一张。 | 牌区生命周期产生真实取舍。 | 手上状态 + 保留可能让手牌过窄。 | v1 不同时堆多张保留和多张状态；测试覆盖 1+1 极限。 |
| 15 | payoff 回合前抽到 `signal_smear`，导致缺 1 段或缺终结。 | 状态牌让 payoff 爆发不再每次稳定。 | 失败率过高会破坏第 12 轮相似度样片。 | 污染注入只作为反例镜头，不作为默认每回合压力。 |

## 4. 数值边界

### 4.1 消耗牌边界

| 项 | 建议 |
| --- | --- |
| 费用 | 0 MP |
| 伤害 | 2，最高不超过 3 |
| 副效果 | 无抽牌、无返 MP、无 topdeck |
| 出现密度 | 前 2 次奖励最多出现 1 张 |
| 核心风险 | 让 0 段过密、让 `0 > 1 > 2` 没有坏手 |

### 4.2 保留牌边界

| 项 | 建议 |
| --- | --- |
| 费用 | 2 MP |
| 伤害 | 前排 4，低于 `row_cleave` |
| 保留数量 | v1 建议同一时间最多 1 张自然保留牌 |
| 发牌规则 | 保留牌占手牌位，手牌上限仍是 4 |
| 核心风险 | 免费提高下一手质量、让玩家背固定链路 |

### 4.3 状态 / 污染牌边界

| 项 | 建议 |
| --- | --- |
| 费用 | 0 MP |
| 效果 | 无收益；打出后消耗 |
| 链路 | 不参与、不推进、不授权 |
| 注入密度 | 3-5 回合样片中最多 1 张；单节点污染密度不超过 deck 的 12%-15% |
| 清理 | 节点结束 purge；v1 不跨 run |
| 核心风险 | 过强会让手牌崩坏，过弱会像无意义点击 |

## 5. 文案与美术边界

三张牌的文案必须短，不能把设计解释塞进卡面。

| 牌 | 卡面文案 | 不写 |
| --- | --- | --- |
| Short Fuse Permit | `造成2。开链。消耗。` | “这是一次性补 0 段，帮助完成 0>1>2” |
| Held Lane | `前排4。保留。完成链给授权。` | “下回合先进入手牌并减少补抽数量” |
| Signal Smear | `无效果。不接链。打出后消耗。` | “污染会降低未来抽牌质量并鼓励净化系统” |

美术边界：

- `short_fuse_permit`：Redline 自有的临时通行证 / 热熔纸片 / 即将烧穿的许可，不使用竞品卡牌边框、职业符号或熟悉图标。
- `held_lane`：被夹在红线档案板里的车道批文，表达“留档”，不要画成竞品式手牌保留标记。
- `signal_smear`：屏幕噪点、错误传票、污损账单，不使用竞品的伤口、诅咒、黏液、眩晕等经典视觉包装。

命名边界：

- 不使用竞品卡名、怪物名、职业名、专属关键词。
- 不使用 `Burn`、`Wound`、`Dazed`、`Slimed`、`Void` 这类强竞品联想词作为最终名。
- 当前英文名只是内部占位，落 UI 前可优先使用中文短名：`短熔许可`、`留档车道`、`噪点传票`。

## 6. 进入实现前的验收口径

实现者不需要把三张牌一次性做成完整卡包。通过口径是：

1. `short_fuse_permit` 打出后进入消耗区，不进弃牌，不洗回。
2. `held_lane` 回合末进入保留区，下次发牌先进手牌，并占手牌位。
3. `signal_smear` 能进入 draw / hand / discard，抽到占手牌；打出后消耗；不参与 chain；节点结束 purge。
4. 三张牌都不直接增强 `payoff` 数值，不新增 Max MP，不绕过 `payoff-only` 授权。
5. 3-5 回合样片仍能覆盖成功链、坏手修补、奖励响应、爆发清场和失败反例。
6. 移动端只显示短 token，不新增长规则面板。

最终内容裁决：

```text
生命周期 v1 的第一批内容只要三张牌：
一次性开链、可保留 2 段、物理污染。

它们的价值不是让玩家更容易赢，
而是让抽、弃、洗、消耗、保留、污染这些牌区事实
第一次变成玩家在 0>1>2 / Wild / payoff 中能感到的取舍。
```
