# 2026-05-18 Round 07-07：保压型 reorder 体验裁决

角色：第 7 轮专家 07，玩家体验与压迫节奏设计师  
工作目录：`/Users/roc/Game-001`  
任务边界：只新增本文档；不改源码、不提交 git。  
问题：真实 `reorder` 应该让玩家感觉“计划下一抽”，还是“直接找答案”。  

## 1. 体验目标

本轮体验裁决：**真实 `reorder` 首先应该让玩家感觉“计划下一抽”，不是“直接找答案”。**

原因很直接：第 6 轮已经把 self draw 的核心反馈压成 `抽N仍-X`，这说明当前战斗压力的中心不是“我有没有多抽牌”，而是“敌人意图还在不在”。`reorder` 如果一实现就变成“从牌堆里把 payoff / 2 费段找出来”，玩家会把它读成答案按钮，`抽N仍-X` 建立起来的承压语义会被冲掉。

正确的心理顺序应该是：

```text
敌人意图压着我
-> 我用抽牌 / 修补争取信息和路线
-> reorder 让我安排下一抽
-> 真正降意图仍然要靠后续攻击、打断、授权或 payoff
```

不应该是：

```text
敌人意图压着我
-> 我点 reorder
-> 系统把答案牌给我
-> 压力变成等我点击清场
```

`reorder` 的体验目标不是“救场成功”，而是“我在压力下多掌握了一步”。它可以减少盲抽焦虑，但不能替代出牌兑现。

因此 P0/P1 的保压合同是：

- `reorder` 不直接降低敌人意图。
- `reorder` 不给当前 MP、最大 MP、临时授权或免费支付。
- `reorder` 不从全牌库 / 弃牌堆无条件找关键牌。
- `reorder` 的确认反馈必须继续带 `仍-X` 或 `未解意图`。
- 如果要做“直接找答案”，应另命名为 `检索 / tutor / 找终结`，并按更高强度机制收费、限频、后置。

## 2. reorder 强弱分层

### 2.1 Tier 0：文案整备

当前状态属于 Tier 0：

```text
utilities: ['reorder']
关键词 / HUD 写“整备 / 找牌”
runtime 没有真实重排、预视或检索事件
```

体验作用：只能作为角色定位标签，不能承诺“重排牌库”。  
压力风险：低。  
文案要求：仍用 `抽N仍-X`，不要写 `已重排`、`已找到`、`安全`。

### 2.2 Tier 1：看顶 N，安排下一抽

推荐作为真实 `reorder` 的第一层。

规则感：

```text
查看抽牌堆顶部 2-3 张
选择 1 张放到牌顶，或调整这 2-3 张顺序
不额外抽牌
不搜索弃牌堆
不搜索全牌库
不把牌直接加入手牌
```

体验作用：玩家感觉“我知道下一抽是什么，并且我为下一步做了计划”。  
压力风险：低到中。因为当前敌人意图没有被解决，玩家还需要后续抽到、支付、打出。  
适合短语：`看顶3`、`定下抽`、`下抽：红线切割`、`仍-17`。

这是最符合 `抽N仍-X` 口径的一层：玩家获得信息和顺序控制，但当前回合压力仍原样挂在场上。

### 2.3 Tier 2：看顶 N，优先置顶路线牌

这是“计划下一抽”和“找答案”之间的中间层。

规则感：

```text
只看抽牌堆顶部 3-5 张
如果其中有 route / repair / payoff，允许置顶其中 1 张
如果没有，就明确显示未找到
不看弃牌堆
不全牌库搜索
不直接入手
```

体验作用：玩家觉得自己在筛选路线，而不是系统凭空发答案。  
压力风险：中。因为如果 top N 经常命中 payoff，会接近“弱检索”。  
使用条件：只能放在 2 MP 支援牌或中后期奖励上，不进起手，不和高密度 Wild / 返 MP 同批增强。

### 2.4 Tier 3：SearchAndTopdeck 指定类型

第 6 轮工程视角提出的 `SearchAndTopdeck(payoff / route)` 属于这一层。

规则感：

```text
按偏好搜索抽牌堆，找到 payoff / route 后置顶
随后复用现有 DrawCards
```

体验作用：更像“直接找答案”，尤其当 `paper_shatter` 在接链后真实抽 3 时，置顶 payoff 基本等于把 payoff 拉进手牌。  
压力风险：高。它会把 `paper_shatter / lantern_captain` 从支援牌推向“找终结按钮”。  

这层不是不能做，但不能再叫普通 `reorder`。如果为了 demo 稳定必须做，应把它明确包装为强机制：

```text
检索终结
整备：置顶终结
找路线
```

并保留压力后缀：

```text
置顶终结 · 仍-17
已找终结，未解意图
```

### 2.5 Tier 4：全牌库 / 弃牌堆 tutor 或直接入手

这一层 P0/P1 不建议开放。

危险形态：

```text
从全牌库选 1 张加入手牌
从弃牌堆找 payoff 加入手牌
抽 3 后任意重排剩余牌堆
搜索 route + 返 MP + Wild 修补同卡叠加
```

体验作用：玩家会读成“直接找答案”。  
压力风险：极高。它会让敌人意图从“本回合后果”退化成背景数字。  

如果未来要做这层，必须有清晰代价：耗血、弃牌、消耗本牌、每回合一次、不能找 payoff、或只能在低 HP 高风险状态下触发。否则不要放进早期奖励池。

## 3. 反馈文案

### 3.1 核心文案规则

所有 `reorder` 反馈都必须同时表达两件事：

```text
你改变了什么：看顶 / 置顶 / 下抽
压力还在：仍-X / 未解意图 / 重击仍在
```

推荐主格式：

```text
看顶3 · 定下抽 · 仍-17
下抽：Row Cleave · 仍-17
整备未见终结 · 仍-17
置顶终结 · 仍-17
已规划下抽，未解意图
```

移动端短格式：

```text
看3仍-17
下抽仍-17
置顶仍-17
未见仍-17
```

### 3.2 卡牌按钮

Tier 0 当前无 runtime 时：

| 场景 | 桌面按钮 | 移动按钮 |
| --- | --- | --- |
| `paper_shatter` 接链 | `抽3找终结 · 仍-17` | `抽3仍-17` |
| `lantern_captain` 接链 | `抽3找路线 · 仍-17` | `抽3仍-17` |
| 未接链 | `抽1整备 · 仍-17` | `抽1仍-17` |

Tier 1 真实计划下一抽时：

| 场景 | 桌面按钮 | 移动按钮 |
| --- | --- | --- |
| 可看顶 | `看顶3 · 定下抽 · 仍-17` | `看3仍-17` |
| 已确认 | `下抽：Row Cleave · 仍-17` | `下抽仍-17` |
| 跳过 | `跳过整备 · 仍-17` | `跳过仍-17` |

Tier 2 / Tier 3 带偏好找牌时：

| 场景 | 桌面按钮 | 移动按钮 |
| --- | --- | --- |
| 找到路线 | `置顶路线 · 仍-17` | `置顶仍-17` |
| 找到终结 | `置顶终结 · 仍-17` | `终结仍-17` |
| 未找到 | `未见终结 · 仍-17` | `未见仍-17` |

不建议写：

```text
找到答案
救场成功
压力解除
安全
已解决
终结到手
```

这些词都会把“计划下一抽”读成“已经赢了这一回合”。

### 3.3 Combat Director

`reorder` 发生后，Director 不需要讲长规则，只需要维持压力锚点。

推荐：

```text
敌人意图：结束回合 -17 HP
行动提示：已规划下抽，未解意图
```

如果存在 `红线重击`：

```text
重击仍在：结束回合 -22 HP
下抽已定，仍需打断 / 击杀
```

这能把 reorder 的价值放在“下一步计划”，同时把真正要处理的敌人意图留在玩家眼前。

### 3.4 战斗日志

日志要把抽牌收益、重排收益和压力保留写在同一条因果里。

推荐：

```text
Paper Route x3 · 抽3 · 意图仍-17
Paper Route · 看顶3 · 下抽 Row Cleave · 意图仍-17
Lantern Captain · 置顶路线 · 意图仍-17
整备未见终结 · 意图仍-17
Severance Burst 置顶 · 仍需打出
```

如果后续攻击真的降意图，再另写：

```text
Row Cleave 命中 · 意图17->9
Severance Burst 兑现 · 意图17->0
```

这样玩家能读出边界：`reorder` 负责准备，攻击 / payoff 负责解压。

### 3.5 详情 / tooltip

推荐解释：

```text
整备：查看并安排抽牌堆顶部。它不造成伤害，不降低敌意图。
```

如果是 Tier 3 强检索：

```text
检索：按条件置顶一张牌。置顶后仍需抽到并打出，敌意图不会因此下降。
```

不要在 tooltip 里把 `reorder` 解释成“找答案”。应把“答案”留给攻击、打断、授权和 payoff。

## 4. 本轮建议

### 4.1 体验裁决

第 7 轮从玩家压力体验角度，不建议把真实 `reorder` 的第一版做成“直接找答案”。第一版应做成 **Tier 1：看顶 N，安排下一抽**。

如果后续实现时考虑 `paper_shatter / lantern_captain`，推荐体验合同是：

```text
先保持 self draw 的 `抽N仍-X`
再提供一个明确的下抽规划反馈
玩家必须继续打出后续牌，敌意图才会下降
```

也就是说，`paper_shatter` 可以让玩家感觉“我把下一步安排好了”，但不应该让玩家感觉“我已经把终结牌拿到手，压力结束了”。

### 4.2 与第 6 轮的衔接

第 6 轮已经形成三条关键约束：

1. `抽N仍-X` 是 self draw / 修补牌的压力口径。
2. 强 Wild / 强 reorder 会把硬坏手率压得过低，削弱敌人意图。
3. 第 6 轮综合裁决没有实现 reorder runtime，而是把它留到专门轮次。

因此第 7 轮建议不要直接接 `SearchAndTopdeck(payoff)` 作为默认真实 reorder。工程上它最省 UI，但体验上它更像 Tier 3 检索。若一定要保留这个方向，应满足这些限制：

- 只搜索 `drawPile`，不搜索 `discardPile`。
- 只看前 3-5 张，不全牌库搜索。
- 找到后只置顶，不直接入手。
- 不给 MP，不补 Wild，不绕过 `payoff-only` 授权。
- 同一早期奖励池不要同时出现强 Wild、强 self draw、强检索。
- 所有反馈都必须写 `仍-X` 或 `未解意图`。

### 4.3 推荐落点

推荐下一个可实现切片不是“找 payoff”，而是：

```text
整备：看顶3，选1张作为下一抽。
```

最小体验验收：

1. 玩家打出 `paper_shatter / lantern_captain` 后，能看到 `抽N仍-X`。
2. 触发整备时，能看到顶部 2-3 张候选，而不是全牌库。
3. 确认后显示 `下抽：某牌 · 仍-X`。
4. End Turn 按钮和敌人意图总伤害不因整备下降。
5. 只有玩家随后抽到并打出攻击 / payoff，意图才下降。
6. 未找到或跳过整备时，日志明确写 `未见 / 跳过 · 仍-X`，不静默失败。

### 4.4 暂缓项

本轮不建议推进：

- 全牌库 tutor。
- 弃牌堆检索。
- 直接加入手牌。
- 自动找 payoff 并立即抽入手的默认早期奖励。
- `reorder + Wild + 返 MP + 抽牌` 同卡叠加。
- 把 `重排` 写成 `找到答案`。

这些都属于“直接找答案”强度，会削弱第 6 轮刚建立的敌人意图压迫。

## 5. 最终裁决

真实 `reorder` 的首个可玩版本应让玩家感觉：

```text
我在 -17 / -22 的压力下，把下一抽计划好了。
```

而不是：

```text
我点了一张牌，系统直接把答案给我。
```

`抽N仍-X` 是这一点的核心护栏。只要 `reorder` 的每个可见反馈都继续承认“意图仍在”，它就能增强卡牌游戏的计划感，而不是削弱战斗压迫。强检索可以作为后续机制存在，但必须从 `reorder` 中分层出来，按更高强度、更高代价、更晚出现的位置处理。
