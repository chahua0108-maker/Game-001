## 2. 最终玩法说明

这版 Demo 的可演示核心不是“无限堆系统”，而是一个短时卡牌战斗循环：

1. 开局进入 Web 原型，玩家看到 HP、MP、费用链、敌意图、手牌和战斗日志。
2. 点击或键盘出牌。单体攻击默认瞄准当前第一排最高敌意图目标，玩家也可以选择第一排目标。
3. 通过 0 费开链、1 费承接、2 费展开完成 `0 -> 1 -> 2` 链。
4. 完成链后获得本回合临时 `授权+3`，只能支付 3 费终结牌。
5. 3 费终结牌如 `Severance Burst` 可用授权支付，清理高压敌意图。
6. 若手牌卡住，可用抽牌/修补牌找路线；`wild_mana_stitch` 只有真实修补成功才返当前 MP。
7. 奖励层在升级后提供新牌选择；奖励牌进入抽牌堆顶，下一手能直接进入循环验证。
8. `paper_shatter / Paper Route` 是当前最明确的第 8 轮样片：抽牌前从抽牌堆找第一张终结牌置顶，然后立刻抽到，HUD 日志显示 `整备：顶终结`。
9. 局外层当前只显示档案占位，不影响当前战斗，不承诺永久成长。
10. 移动端以短 token 读数为主：`授权+3`、`缺MP1`、`授权付`、`整备`、`回合损N`，避免长规则文本挤爆操作区。

推荐演示路径：

```text
Deal / 开局发牌
-> 打 0 费开链
-> 打 1 费承接
-> 打 2 费展开拿授权
-> 用授权打 3 费终结
-> 进入奖励选择
-> 选择能改善下一轮的奖励牌
-> 展示 Paper Route 顶终结样片或读取第 9 轮 QA 证据
```
