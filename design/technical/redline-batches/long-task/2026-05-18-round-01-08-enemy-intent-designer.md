## 1. 当前敌人意图表现问题

### 1.1 敌人差异仍主要是 HP / damage

当前敌人定义只有三类：

| 敌人 | HP | damage | XP | 当前实际差异 |
| --- | ---: | ---: | ---: | --- |
| `debt_wisp` | 10 | 2 | 1 | 低血低伤。 |
| `redline_brute` | 22 | 5 | 2 | 高血高伤。 |
| `pulse_collector` | 16 | 3 | 2 | 中等血量和伤害。 |

`speed`、初始 `lane`、初始 `z` 在当前 hyper-turn 主体验里不构成敌人意图；槽位会重新派生 lane/z。玩家真正需要读的是“谁在本回合结束时打我多少”，而不是敌人有什么不同策略。

问题：敌人只用 HP 和伤害区分时，卡牌选择会退化成：

- 先打血少的，拿 XP。
- 先打伤害高的，少扣 HP。
- 有群攻就打群攻。

这还不是“意图反制”，只是算伤害账。

### 1.2 `EnemyIntent` 只有 attack，缺少反制窗口

当前 `EnemyIntent` 结构是：

```text
enemyId
kind: 'attack'
amount
slot
description
willRefill
```

它能支撑 P0 的“结束回合会掉多少血”，但不能表达：

- 充能中，下一回合爆发。
- 防御中，需要破盾或绕过。
- 诅咒中，不处理会污染牌堆。
- 召唤中，不打断会补出新敌人。
- 反击中，用普通攻击会付代价。
- 可打断窗口，只在本回合 / 下一张牌 / 指定费用段有效。

结果是所有敌人意图都被同一种反制方式处理：把它杀掉。这样会把卡牌系统挤压成伤害牌优先，防御、修补、抽牌、清除、打断卡都没有舞台。

### 1.3 当前 target selection 不足以支撑“我反制了谁”

HUD 已有前排显影和目标选择，但默认状态仍是：

- `front-enemy` 牌未选择目标时，runtime 会随机选择一个前排敌人。
- UI 明示“未选择，单体牌随机前排”。
- 目标按钮只显示类型、名字、HP，不显示本回合意图、伤害、打断条件或反制收益。

如果敌人意图要成为决策压力，随机默认目标会直接破坏核心体验。玩家必须能读到：

- 我点这只敌人，是因为它本回合要打 5。
- 我这张牌是否能杀它。
- 如果杀不了，能不能打断它。
- 出牌后本回合总意图会从多少变到多少。

### 1.4 UI peek 仍像目标面板，不像威胁读盘器

当前 HUD 已经有 `意图` chip、combat director、结束回合按钮伤害提示、前排 peek。方向正确，但前排 peek 本身还缺三类信息：

| 缺口 | 为什么重要 |
| --- | --- |
| 每个敌人的 active intent | 玩家要知道谁本回合会结算。 |
| 补位 / 下轮标记 | 击杀后补上来的前排不应被误读成本回合攻击者。 |
| 反制标签 | 玩家要知道该用打断、破盾、清除、群攻还是防御。 |

现在玩家看到的是“这个槽位有一个 HP 多少的敌人”，不是“这个敌人正在做什么，我能怎样阻止它”。

### 1.5 战斗结算能记录避免伤害，但还没有成为卡牌反馈

当前结算有一个好基础：

- 发牌时快照本回合攻击者。
- 死亡敌人会刷新 intent。
- 结束回合只结算快照中仍然存活且仍在前排的攻击者。
- `PayoffResolved` 能记录 `preventedIntentDamage`。

问题是这些信息主要停留在事件和调试层。卡牌按钮没有显示“打出后避免多少伤害”，敌人槽位也没有显示“这只已经不属于本回合压力”。于是玩家很难把“我打了这张牌”和“我反制了敌人意图”连起来。

### 1.6 现有规则只有玩家牌规则，没有敌人行为规则

`redlineRules.ts` 当前覆盖：

- 单体前排伤害。
- 前排群体伤害。
- 全场 payoff。
- self 抽牌 / 返还当前 MP。
- 敌人死亡给 XP、压缩、补位。

缺少敌人行为层规则：

- 敌人如何选择下一意图。
- 敌人防御 / 充能 / 诅咒 / 召唤如何结算。
- 打断成功后意图如何降级或取消。
- 反制卡如何与敌人意图建立明确契约。

所以当前系统有“敌人要攻击”的事实，但还没有“敌人正在计划一个可反制动作”的游戏性。

## 2. 10 种原创敌人意图模板

下面 10 个模板不是建议全部进 P0。它们用于建立 Redline 后续敌人意图语言：每个意图都要有预告、结算、打断窗口和至少一种卡牌反制方式。

| # | 意图模板 | Telegraph 文案 | 结算 | 打断窗口 | 反制卡关系 |
| ---: | --- | --- | --- | --- | --- |
| 1 | 红线重击 | `充能重击 -10，本回合受 12+ 伤害可打断` | 若未打断，结束回合造成高额单体伤害。 | 当前玩家回合内；受到阈值伤害或死亡。 | 单体高伤、前排群攻、armed payoff；让 `redline_cut` 和 `row_cleave` 有明确救场目标。 |
| 2 | 护债屏障 | `护卫：给相邻敌人 +8 护盾` | 结束回合前为同排或同列敌人加护盾，保护高威胁目标。 | 护盾释放前击杀或破盾。 | 破盾、穿透、标记、先清护卫；让玩家不再只按伤害最高敌人打。 |
| 3 | 脉冲蓄息 | `蓄息：下轮攻击 +4；可用打断牌取消` | 本回合不伤害，下轮变成更高伤害或多段攻击。 | 当前回合用 `interrupt`、足量伤害或特定费用段牌。 | 打断牌、延迟收益牌、抽牌找解；制造“现在处理还是赌下轮”的压力。 |
| 4 | 逾期诅咒 | `逾期：结束回合塞 1 张罚单` | 向弃牌堆或抽牌堆加入状态 / 诅咒牌，降低后续手牌质量。 | 当前回合击杀、清除、沉默或支付代价。 | 清除、弃牌、净化、抽牌过滤；让非伤害卡有价值。 |
| 5 | 血印锁定 | `锁定：下次攻击翻倍；防御可消除` | 给玩家挂一层标记，下一次 enemy attack 增伤。 | 标记生效前使用防御 / cleanse / 打断。 | 防御牌、净化牌、临时护盾；推动“本回合不只清怪，也要解除未来风险”。 |
| 6 | 债务召唤 | `召唤：下轮同列补 2 个 Wisp` | 若未阻止，在后排或空槽生成低血敌人，增加后续前排密度。 | 当前回合击杀召唤者，或用反召唤牌封锁槽位。 | 前排群攻、全场 payoff、封槽、驱散召唤；让群攻不只是伤害，还控制队列。 |
| 7 | 反击姿态 | `反击：受到攻击时回击 2；技能可绕过` | 玩家用 attack 命中它时触发小额反伤或给玩家塞状态。 | 反击姿态持续本回合，可被技能 / interrupt 移除。 | 技能牌、沉默、非攻击伤害、先打别的敌人；让 cardType 产生决策差异。 |
| 8 | 收割倒计时 | `倒计时 2：每出 2 张牌后结算` | 不等到 end turn，而是在玩家出到指定张数后触发轻伤害或诅咒。 | 倒计时归零前击杀 / 打断。 | 高效率单卡、打断、保留关键牌；制造“能不能先处理它再继续连锁”的压力。 |
| 9 | 后排仪式 | `仪式：后排通道，前排未清则无法打断` | 后排敌人持续充能，若前排护卫存在则不能被单体命中。 | 清前排后才开放打断；或用全场 / 穿透牌。 | `front-row`、`all-enemies`、穿透、拉拽；让队列结构本身进入决策。 |
| 10 | 清算转嫁 | `转嫁：本回合首次伤害转给最低血友军` | 把第一段伤害转移给另一个敌人，保护关键目标。 | 用低伤害探掉转嫁，或用群攻绕开。 | 低费试探、群攻、标记、破防；让 0 费牌不是纯开链，也能拆机制。 |

## 3. 与卡牌机制的反制关系

### 3.1 当前已有卡牌能承担的反制角色

| 当前机制 | 已有卡牌例子 | 能反制什么 | 现在缺的前台反馈 |
| --- | --- | --- | --- |
| 单体前排伤害 | `debt_hook`、`redline_cut`、`heartbeat_spark` | 杀死或压低单个 active intent 敌人。 | 需要显示 `打出后意图 17 -> 12`、`可打断重击`。 |
| 前排群体伤害 | `row_cleave`、`clearance_order` | 同时降低多个前排 attack intent；适合处理召唤后的密集前排。 | 需要显示每个 active intent 是否会被清掉。 |
| 全场 payoff | `severance_burst`、`red_ledger_burst` | armed 后处理后排仪式、召唤堆积、全场高峰。 | 需要区分 armed / unarmed 反制收益。 |
| Wild 修补 | `wild_mana_stitch`、`wild_gap_key` | 在缺费用段时继续走到打断 / payoff。 | 需要显示“修补后能接上哪张反制牌”。 |
| 抽牌 / 整备 | `blood_tithe`、`pulse_draw`、`paper_shatter`、`lantern_captain` | 找到打断、清除、payoff 或关键费用段。 | `reorder` 还没有 runtime，文案要避免承诺过度。 |
| 当前 MP 返还 | `wild_mana_stitch` | 延长回合，争取多处理一个 intent。 | 需要显示它是当前回合资源，不是永久成长。 |

### 3.2 需要补的反制卡关键词

P0 不需要一次补完整卡池，但敌人意图语言要预留这些关键词：

| 关键词 | 作用 | 对应意图 |
| --- | --- | --- |
| `interrupt` / 打断 | 取消或降级充能、召唤、仪式。 | 红线重击、脉冲蓄息、债务召唤、后排仪式。 |
| `block` / 防御 | 抵消玩家将承受的 end-turn 伤害，不要求杀敌。 | attack、锁定、倒计时。 |
| `cleanse` / 清除 | 移除玩家身上的血印、诅咒、锁定。 | 逾期诅咒、血印锁定。 |
| `pierce` / 穿透 | 绕过护盾或前排保护，命中后排仪式目标。 | 护债屏障、后排仪式。 |
| `shatter` / 破盾 | 对护盾、护卫、防御姿态有额外收益。 | 护债屏障、反击姿态。 |
| `silence` / 沉默 | 暂停敌人非攻击意图，但不一定造成伤害。 | 诅咒、召唤、充能。 |
| `seal-slot` / 封槽 | 阻止补位或召唤填入指定列。 | 债务召唤、后排仪式。 |

这些关键词的价值不在于复杂，而在于让玩家看到“这张牌为什么是这一回合的答案”。

### 3.3 反制不应只等于击杀

建议把反制拆成四档结果：

| 结果档 | 玩家感受 | 示例 |
| --- | --- | --- |
| 取消 | 完全阻止意图结算。 | 杀死 active intent 敌人；打断召唤。 |
| 降级 | 高威胁变成普通威胁。 | 充能重击 -10 被打断后降为普通 -5。 |
| 延迟 | 本回合不解决，但推迟到下轮或转为较弱状态。 | 后排仪式被沉默 1 回合。 |
| 承受但缓解 | 没杀掉敌人，但通过防御/清除降低后果。 | 结束回合 -17 变成 -7。 |

当前系统主要支持“取消”。P0 下一步应先补“降级”，因为它比完整防御、诅咒、召唤更小，也最容易让现有攻击牌产生新意义。

### 3.4 UI 必须把“反制关系”前置到卡牌按钮

每张可打牌至少应能回答一个问题：

```text
打这张后，本回合敌人意图会怎样变化？
```

建议的最小文案：

- `杀 BRU：意图 17 -> 12`
- `打断 BRU：重击 -10 -> 普攻 -5`
- `横扫：意图 17 -> 5`
- `未击杀：COL 仍会 -3`
- `抽牌：找 3费终结 / 找打断`

这会把卡牌选择从“打多少伤害”转成“处理哪种压力”。

## 4. P0 / P1 实现顺序

### P0：只做一个可验收的意图压力闭环

P0 不建议扩 10 个意图模板。先把“一个非普通 attack 意图”跑通，证明敌人意图真的能改变卡牌决策。

| 顺序 | 任务 | 目的 | 验收口径 |
| ---: | --- | --- | --- |
| P0-1 | 统一 intent 真相源 | HUD、卡牌预览、结束回合都读 `enemyIntentSummary` 和 per-enemy `enemyIntents`。 | 打出牌后，UI 的剩余意图和 runtime 结算一致。 |
| P0-2 | 前排 peek 显示意图和下轮状态 | 槽位上显示 `本回合 -5`、`充能 -10`、`下轮`。 | 击杀补位后，新前排不会被误读成本回合攻击者。 |
| P0-3 | 默认目标从随机改成可预测规则 | 未手动选目标时，默认锁定“当前最高 active intent 前排敌人”，并在 UI 显示。 | 单体牌不再随机破坏反制判断。 |
| P0-4 | 卡牌按钮显示 intent delta | 出牌前显示击杀 / 打断 / 降低后的意图数。 | 玩家不用看 debug，也能预测结束回合扣血。 |
| P0-5 | 落地一个 `充能重击` 意图 | 让高威胁敌人可被打断或降级，不再只是 HP 多。 | 正确打击路线能把 `-10` 降成 `-5` 或取消；忽略则吃高伤。 |
| P0-6 | 加固定验收脚本 | 三条路径：忽略、错误目标、正确反制。 | 三条路径的 HP 变化、intent delta、事件记录都不同。 |

### P1：扩成敌人意图生态

P1 再把敌人意图从“攻击 + 一个充能”扩成轻量生态：

| 顺序 | 任务 | 目的 |
| ---: | --- | --- |
| P1-1 | 护卫 / 防御意图 | 引入破盾、穿透、先后手目标选择。 |
| P1-2 | 诅咒 / 状态污染 | 让不处理敌人不一定只掉血，也可能污染后续手牌。 |
| P1-3 | 召唤 / 封槽 | 让队列和补位成为压力，而不是纯视觉。 |
| P1-4 | 玩家防御牌 | 允许玩家不杀敌也能反制伤害峰值。 |
| P1-5 | 打断牌与沉默牌 | 让技能牌有明确反制身份。 |
| P1-6 | 奖励池按反制类型出牌 | 三选一不只是伤害 / 数值，而是“打断、清除、payoff、稳定”。 |
| P1-7 | 敌人意图调度器 | 3-5 回合内安排普通攻击、充能峰值、诅咒、召唤，形成压力曲线。 |

### 暂缓项

- 不先做大量新敌人。
- 不先做完整防御系统。
- 不先做复杂多回合 AI。
- 不把诅咒、召唤、反击全塞进 P0。
- 不恢复 realtime 自动扣血或固定 60 秒 burst。

## 5. 本轮建议先落地的一个敌人压力点

### 建议：`Redline Brute` 获得“红线重击 / 可打断充能”

这是本轮最适合先落地的压力点，因为它能最大化复用现有卡牌和规则：

- `Redline Brute` 已经是最高伤害和最高血量敌人，天然适合作为威胁教学。
- 起手牌里已有 `debt_hook`、`redline_cut`、`row_cleave`，能形成不同反制路线。
- 不需要先做诅咒、召唤、防御、清除卡。
- 可以直接验证“正确选目标”和“正确费用链”是否降低敌人意图。

### 设计规格

`Redline Brute` 在某些回合不再只是普通 `attack -5`，而是显示：

```text
红线重击：结束回合 -10
打断条件：本回合受到 12+ 伤害，或死亡
打断结果：重击降级为普通攻击 -5；死亡则取消
```

### 玩家决策

同一手牌里玩家会看到三个真实选择：

1. `Debt Hook -> Redline Cut` 集火 Brute：用 0->1 链直接打断或击杀高威胁。
2. `Debt Hook -> Heartbeat Spark -> Row Cleave` 走群体路线：处理多只前排，但可能对 Brute 差一点。
3. 忽略 Brute 去清低血 Wisp：拿击杀 / XP，但结束回合吃重击。

这三个选择的差异不是“谁血多”，而是“我用这一手牌处理哪种意图后果”。

### 最小 UI 文案

前排槽位：

```text
BRU HP 22/22
充能 -10
打断 12
```

卡牌按钮：

```text
Redline Cut
命中 BRU：打断重击，意图 17 -> 12
```

结束回合按钮：

```text
结束回合 -17
```

打断后：

```text
结束回合 -12
BRU 普攻 -5
```

击杀后：

```text
结束回合 -7
BRU 已取消
```

### 验收路径

| 路径 | 操作 | 预期 |
| --- | --- | --- |
| 忽略 | 不打 Brute，直接结束或打低威胁目标 | Brute 结算 `-10`，玩家明确感到忽略意图的代价。 |
| 打断 | 对 Brute 造成 12+ 但未击杀 | Brute 从 `重击 -10` 降级为 `普攻 -5`，总意图下降 5。 |
| 击杀 | 用正确链路杀死 Brute | Brute intent 完全消失，总意图下降 10。 |
| 错误目标 | 同样伤害打到非 Brute | 伤害有收益，但 Brute 重击仍在，证明 target selection 重要。 |
| 群攻 | 使用 `row_cleave` 或 `clearance_order` | 多个意图一起下降；若 Brute 达到阈值则重击降级。 |

### 为什么不是先做诅咒或召唤

诅咒和召唤很重要，但它们会马上要求新增状态牌、牌堆污染规则、召唤槽位规则和清除卡。当前 P0 最大风险不是系统不够多，而是玩家还没有把“卡牌选择”和“敌人意图变化”连起来。

`红线重击` 是最小切入点：只增加一个可打断的攻击变体，就能验证 target selection、UI peek、卡牌预览、结算事件和 payoff 反馈是否真的形成闭环。

## 结论

当前 Redline 已经有“敌人意图伤害”的骨架，但还没有“敌人正在做可反制动作”的设计语言。下一步不要先扩大量敌人或卡牌，应先让一个 Brute 充能重击成为可读、可打断、可结算、可验收的压力点。

如果这个点成立，后续再扩防御、诅咒、召唤、反击和后排仪式，都会有统一的接口：telegraph、counter window、counter cards、intent delta、settlement proof。
