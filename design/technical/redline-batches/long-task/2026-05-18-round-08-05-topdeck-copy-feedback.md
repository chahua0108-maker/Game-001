## 0. 结论

`paper_shatter` 如果从“抽牌支援”升级为真实 topdeck payoff，玩家可见层不要写“重排牌库”或“检索牌库”。P0 文案应把它读成一个非常短的战斗动作：

```text
整备：顶终结 -> 抽N -> 下抽/入手终结
```

卡面和 HUD 只给决策词：`整备`、`抽3`、`顶终结`、`未命中`。完整规则放 `title/detail`：说明它会优先把 payoff 放到牌堆顶，再让当前抽牌结算抽入手牌；若没有可用 payoff，则照常抽牌。

## 1. 词表原则

| 原则 | 执行口径 |
| --- | --- |
| 玩家可见层短词 | 2-4 字机制词优先：`整备`、`终结`、`顶终结`、`抽3`、`未命中`。 |
| 不让文字超框 | 卡面主文案不超过 8 个中文字符；日志单条不超过 18-24 字；奖励卡首行不超过 12-16 字。 |
| 完整规则后置 | `title`、详情面板、debug trace 才解释“先置顶，再抽牌”。 |
| payoff 语义明确 | `paper_shatter` 只写找 `终结`，不要写泛化的 `找牌` 作为核心收益。 |
| 不承诺手动牌库 UI | 如果是自动 topdeck，不写 `选择`、`重排`、`看牌库`、`控顶`。 |
| 失败也有反馈 | 未命中必须有短日志和 detail，避免玩家以为牌没生效。 |

## 2. `paper_shatter` 卡面文案表

假设真实运行时是 `SearchAndTopdeck(preference = payoff)`，命中 payoff 后再复用现有 `DrawCards` 抽牌。

| UI 字段 | 推荐文案 | 长度预算 | 说明 |
| --- | --- | ---: | --- |
| 正式名 | `Paper Route` | 英文 11 | 保留当前名；短 UI 可用 `Paper`。 |
| 中文短名 | `纸路` | 2 字 | 只在空间极窄或本地化 UI 使用。 |
| 角色 token | `整备` | 2 字 | 替代 `整备/找牌`。 |
| 卡面主规则，基础 | `抽1顶终结` | 5 字 | 非接链状态也清楚：抽 1，并尝试顶终结。 |
| 卡面主规则，接链预览 | `抽3顶终结` | 5 字 | 正确 `0 -> 1 -> 2` 时的最重要状态。 |
| mobileEffect | `抽1/3` | 4 字 | 移动端只保数值，收益由 role/preview 补。 |
| 桌面 effect | `抽3顶终结` | 5 字 | 不写“重排”。 |
| intent preview | `抽3仍-12` | 6 字 | 压力反馈仍优先；不要被 topdeck 文案挤掉。 |
| 不可用/缺费 | `缺MP2` | 4 字 | 不追加整备说明。 |
| title 短句 | `Paper Route：先顶终结，再抽牌。` | 不进卡面 | 桌面 hover 可读。 |
| detail 完整规则 | `优先从抽牌堆/可回填弃牌中找一张终结牌置顶，然后结算本次抽牌。没有终结牌时照常抽牌。` | 详情层 | 允许长；不放手牌按钮。 |

### 禁用文案

| 禁用词 | 原因 |
| --- | --- |
| `重排牌库` | 误导为手动 reorder 或完整牌库管理。 |
| `检索终结牌` | 读起来像从全牌库直接拿牌，强度误解。 |
| `选择一张终结牌` | 如果 P0 是自动 topdeck，就没有选择。 |
| `把 Severance Burst 放到牌顶` | 过长，并泄露具体牌名；结果日志用分类更稳。 |
| `找 payoff` | 玩家可见层不用英文机制词。 |

## 3. 运行结果短 token

| 状态 | HUD / Director token | 战斗日志 token | title/detail 解释 |
| --- | --- | --- | --- |
| 命中 payoff 并置顶 | `顶终结` | `整备：顶终结` | `找到一张终结牌并放到下一抽。` |
| 置顶后被本次抽入 | `入手终结` | `抽到终结` | `置顶牌被本次抽牌抽入手牌。` |
| 命中但抽牌数不足或被边界跳过 | `下抽终结` | `下抽：终结` | `终结牌留在牌堆顶，下一次抽牌会先抽它。` |
| 无 payoff 可找 | `整备无牌` | `整备无牌` | `没有可置顶的终结牌，本次照常抽牌。` |
| payoff 已在手牌 | `终结在手` | `整备：终结在手` | `终结牌已经在手牌中，不重复置顶。` |
| 牌堆/弃牌都空 | `无牌可整` | `整备无牌` | `没有可被置顶的牌。` |
| 被规则排除自身 | 不显示 | debug only | `不会把刚打出的 Paper Route 立刻置顶抽回。` |

## 4. 战斗日志文案

战斗日志是结果反馈，不是规则说明。每条都应先给动作，再给结果。

| 事件 | 推荐日志 | 备用短日志 | 避免 |
| --- | --- | --- | --- |
| 打出卡牌 | `出牌 Paper · x3 · 抽3` | `Paper · 抽3` | `Paper Route 触发抽牌和整备效果` |
| topdeck 命中 | `整备：顶终结` | `顶终结` | `已将终结牌放到抽牌堆顶部` |
| 抽到 payoff | `抽到终结` | `入手终结` | `抽到了被置顶的 Severance Burst` |
| topdeck miss | `整备无牌` | `无终结` | `没有找到可用的 payoff 目标` |
| payoff already in hand | `终结在手` | `已持终结` | `目标终结牌已经在玩家手牌中` |
| 正常抽牌后仍受压 | `抽3仍-12` | `仍-12` | `虽然抽了三张牌但敌人意图没有降低` |
| 奖励选中 Paper | `奖励 Paper` | `拿 Paper` | `选择 Paper Route 作为路线桥接奖励` |

推荐日志顺序：

```text
出牌 Paper · x3 · 抽3
整备：顶终结
抽到终结
```

如果未命中：

```text
出牌 Paper · x3 · 抽3
整备无牌
照常抽3
```

## 5. Tooltip / Detail 文案

### 5.1 卡牌 title

| 状态 | title 文案 |
| --- | --- |
| 静态卡牌 | `Paper Route：抽1。优先顶终结，再抽牌。接链可抽3。` |
| 正确接链预览 | `Paper Route：本次抽3；先找终结置顶，再抽入手牌。` |
| 非接链预览 | `Paper Route：本次抽1；先找终结置顶，再抽入手牌。` |
| 无 payoff 候选 | `Paper Route：没有可置顶的终结牌时照常抽牌。` |

### 5.2 详情面板

推荐完整规则：

```text
抽1。若按费用链打出，本次抽牌可被倍率放大。抽牌前，优先找一张终结牌放到牌堆顶；随后结算本次抽牌。没有可用终结牌时照常抽牌。不会把刚打出的这张牌立刻抽回。
```

更短的移动详情：

```text
先顶终结，再抽牌。接链时抽3。无终结则照常抽。
```

### 5.3 关键词详情

| 关键词 | detail |
| --- | --- |
| `整备` | `抽牌前调整下一抽。P0 为自动置顶，不打开手动牌库。` |
| `顶终结` | `把一张终结牌放到牌堆顶，让本次或下次抽牌更容易拿到终结。` |
| `终结` | `高费用 payoff 牌，用来在完整费用链后收束战斗。` |
| `未命中` | `没有找到符合条件的牌；原抽牌流程不变。` |

## 6. 奖励卡文案

奖励卡的目标不是解释完整 topdeck 算法，而是让玩家在三选一里明白它是“找终结的路线桥接”。

| 奖励字段 | 推荐文案 | 说明 |
| --- | --- | --- |
| reward title | `Paper Route` | 保留正式名。 |
| reward role | `整备 · 自身` | 不写 `整备/找牌`。 |
| reward summary | `MP2 · 抽1/3` | 数值优先，避免长句。 |
| reward rule short | `抽牌前顶终结。` | 第一眼看到核心收益。 |
| reward rule with chain | `接链抽3，先顶终结。` | 如果奖励卡能显示第二行，用这句。 |
| reward detail | `加入一张 2 MP 整备牌。它会在抽牌前优先把终结牌置顶；正确接链时通常能抽3张。` | 详情层允许长。 |
| reward tooltip | `适合已经有终结牌、但经常抽不到收束牌的构筑。` | 帮助奖励选择。 |

奖励三选一中的推荐显示：

```text
Paper Route
整备 · 自身
MP2 · 抽1/3
接链抽3，先顶终结。
```

移动端压缩版：

```text
Paper
整备
抽1/3 顶终结
```

## 7. Copy Feedback 场景表

| 玩家处境 | 可见短反馈 | 详情层解释 | 设计目的 |
| --- | --- | --- | --- |
| 玩家已有 `Severance Burst` 在牌堆中 | `顶终结` | `终结牌已放到下一抽。` | 立即确认 Paper 的价值。 |
| 玩家接链后打出 Paper | `抽3顶终结` | `费用链放大本次抽牌。` | 让玩家知道正确链路提升收益。 |
| 玩家没接链就打出 Paper | `抽1顶终结` | `未接链时只抽1。` | 不隐藏惩罚。 |
| 当前敌人仍有高意图 | `抽3仍-17` | `这张牌不直接降意图。` | 防止玩家误以为找牌等于防御。 |
| 没有 payoff 候选 | `整备无牌` | `没有可置顶终结，照常抽牌。` | 失败可解释。 |
| payoff 已在手牌 | `终结在手` | `不会重复搜索已经在手的终结牌。` | 避免“为什么没顶牌”的困惑。 |
| 置顶后抽入 | `入手终结` | `置顶牌被本次抽牌抽入。` | 连接 cause/effect。 |
| 奖励选择时 | `找终结` | `适合补足终结牌到手的稳定性。` | 三选一决策明确。 |
