## 2. 本轮能加到 80 的原因

第16轮的关键不是多一个 route 数据结构，而是路线推进已经进入玩家主流程的关键断点：

```text
Reward pending
-> select reward
-> record run reward
-> advance currentNode
-> discard old hand
-> refill enemies
-> deal next player turn
```

这让第15轮的“短 run 看起来存在”变成第16轮的“玩家真的从奖励进入下一战”。玩家能读到当前节点、奖励候选、路线候选、下一战带入，也能看到奖励牌或升级选择进入后续战斗。  

因此短 run / 地牢推进维度从 `7 / 14` 提到 `12 / 14`，奖励构筑回应、局内牌成长、玩家反馈各小幅加 1 分，合计达到 `80 / 100`。

## 3. 不能停止原因

不能停止。`80 / 100` 距离硬停止线 `95 / 100` 仍差 `15` 分。

当前接入解决的是“主流程不断线”，还不是“可反复玩的高相似度短地牢”。具体缺口如下：

| 缺口 | 影响分 | 为什么还扣分 |
| --- | ---: | --- |
| 路线分叉仍偏弱 | 3-4 | 玩家还没有稳定的“选左/选右路线并承担差异后果”的路线决策；route 候选更多是奖励/标签层提示。 |
| route modifier 未形成 build plan | 3-4 | 下一战变化已存在，但路线 modifier、奖励回应、牌升级还没有合成清晰的构筑计划。 |
| 失败/胜利边界浅 | 2-3 | 有 run 状态和节点上限，但失败压力、精英节点代价、路线风险回报仍不足。 |
| 内容复玩密度不足 | 3-4 | 3-5 节点 run 还缺更多可重复组合、不同奖励走向和不同压力曲线。 |
| 玩家语言还不够彻底 | 1-2 | HUD 已压短 token，但仍有部分信息更像系统状态，不像玩家复盘语言。 |

只要这些缺口存在，即使 QA 全绿，也不能把核心体验分推到 95。

## 4. 下一轮缺口

第17轮不应继续堆纯 sim 或单纯加牌。下一轮应把第16轮的路线主流程变成真正的玩家路线决策：

1. 路线选择显性化：奖励后出现 2 个可点击路线节点，例如维修补给、精英压力、污染清理、强化工坊。
2. route modifier 实装到下一战：不同路线必须改变下一战的能量、奖励数、敌人压力、污染概率或升级机会。
3. 奖励、路线、升级合成 build plan：玩家能说出“我拿这张奖励，是为了走这条路线，下一战用它解决什么问题”。
4. 失败/胜利边界更硬：至少一个路线选择带来高压收益，一个路线选择带来稳妥修复，让短 run 有风险管理。
5. 复测 3-5 节点连续镜头：验收重点是玩家是否愿意继续下一节点，不是 QA 数字。

第17轮目标建议：`80 -> 86`。  
仍然只有核心体验 `>= 95 / 100` 且 QA 门禁全部通过，才允许停止。
