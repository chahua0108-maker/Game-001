第17轮唯一主目标是：

```text
玩家在 3-5 节点 run 中，能看懂自己每次选牌/选路线是在完成哪一种 build plan：
补桥、补终结、清污染、堆关键牌、补资源。
```

本轮不是 QA 工程轮。`npm run check` 和浏览器 QA 仍然必须通过，但它们只证明可以验收，不给核心体验加分。

## 1. 第17轮体验问题

当前 `80 / 100` 的核心体验已经具备这些基础：

- 升序费用链、Wild MP3 延链和 payoff 终结没有退化。
- 状态污染、消耗、保留、洗回等生命周期已经有玩家反馈。
- 奖励能回应缺桥、缺终结、缺资源、污染等问题。
- 奖励后能进入路线选择，路线选择后进入下一战。
- 路线 context 能影响下一战的奖励倾向、资源或压力。

但玩家仍可能只是在“看系统给了什么”，而不是主动制定构筑计划。第17轮要补的是这层可理解性：

```text
上一战暴露的问题 -> 奖励/路线候选解释 -> 玩家选择 -> 下一战验证该选择是否真的补上
```

这个闭环必须跨节点成立，而不是只在一场战斗的 HUD 里出现一个短 token。

## 3. Build Plan 五类语言

本轮所有玩家可见解释都要收敛到五类，不允许把工程枚举直接显示给玩家。

| Build Plan | 玩家要懂的话 | 候选来源 | 下一节点验证方式 | 不合格表现 |
| --- | --- | --- | --- | --- |
| 补桥 | 这张牌能把费用链接起来，让 `0 -> 1 -> 2` 更稳定。 | 低费桥牌、Wild 延链支持、抽牌/保留桥。 | 下一战至少一次断链风险被桥牌修上，或缺桥提示下降。 | 只写“combo +1”；玩家不知道它补的是费用缺口。 |
| 补终结 | 这张牌/路线提高终结牌出现、授权或爆点兑现概率。 | payoff、顶终结、授权、终结奖励倾向。 | 下一战能看到终结候选、终结授权或 payoff 伤害变得更可达。 | 终结牌只是随机出现，没有解释和上一战缺口的关系。 |
| 清污染 | 这张牌/路线减少状态污染对抽牌、手牌或节奏的伤害。 | 清状态、消耗污染、过滤/弃置、稳定路线。 | 下一战污染数量、污染抽到后的处理或污染提示有改善。 | 只显示“清理”，但污染仍像 debug 事件。 |
| 堆关键牌 | 这次选择在围绕某张关键牌/关键路线堆叠收益。 | 升级、复制、保留、定向奖励、路线加成。 | 后续节点能看到该关键牌更强、更常出现或更容易触发。 | 玩家看不到“我在围绕哪张牌构筑”。 |
| 补资源 | 这次选择补 MP、抽牌、手牌容量、保命或奖励复核。 | MP 上限、抽牌、护盾/回血、奖励数量/重选。 | 下一战资源短缺提示下降，或高压路线下能解释为什么能承压。 | 资源变化静默生效，或误写成永久成长。 |

### 3.1 文案规则

文案必须短，优先使用玩家语言：

- `补桥：下一战更容易接 0-1-2`
- `补终结：提高终结牌候选`
- `清污染：减少状态牌卡手`
- `堆关键：强化纸刃终结`
- `补资源：下一战 Max MP +1，本次冒险`

禁止玩家可见：

- `rewardBranchHint`
- `repair-resource`
- `maxEnergyThisRunPlusOne`
- `payoffOnly`
- `routeChoiceId`
- `gateScore`
- 任何暗示永久成长的“账号、档案、永久、局外已解锁”

## 4. 3-5 节点 run 验收脚本

第17轮验收不看单个按钮是否存在，而看一个 3-5 节点 run 是否能读出构筑计划。

最低合格 run：

```text
节点1：暴露一个问题，例如缺桥、污染卡手、缺终结或资源不足
节点1奖励：候选至少两类 build plan，玩家能看懂差异
路线1：路线解释强化其中一个 plan，进入节点2
节点2：上一选择产生可见验证，且暴露新问题或加强原计划
节点2奖励：继续围绕同一 plan 或转向另一个 plan
节点3：能看出这次 run 的构筑方向，而不是随机奖励堆叠
```

优秀 run：

```text
3-5 节点内同时出现：
1. 一次补桥或补资源，解决坏手/断链压力。
2. 一次补终结或堆关键牌，形成 payoff 爆点。
3. 一次清污染或失败反例，证明污染/坏手不是装饰。
4. 路线选择改变下一节点奖励或压力，并能被 HUD 解释。
5. 玩家可以复述：我先补了 X，然后为了 Y 走 Z 路线，最后靠 W 打出爆点。
```

第17轮的核心验收句是：

```text
玩家不是在选随机奖励；玩家在修自己的牌组计划。
```
