## 0. 评分纪律

第18轮评分只看玩家实际玩的核心体验是否变强：

```text
连续 3-5 节点 run
-> 污染、掉血、路线风险持续升级
-> build plan 随战斗结果演变
-> 玩家被迫调整奖励、路线、升级和下一战打法
```

QA 仍只作门禁，不计入核心体验分。`npm run check`、`qa:lifecycle`、`qa:similarity`、`qa:ui`、截图、报告、cleanup、端口关闭都只能证明本轮可以验收；它们不能把 `86` 提到 `90`、`92` 或 `95`。

如果只是脚本更稳、文档更完整、报告字段更清楚、测试覆盖更多，但玩家在 3-5 节点 run 中的决策压力、构筑回应、失败风险和 build plan 演变没有变化，本轮核心体验分保持 `86 / 100`。

## 3. 不加分项

以下成果可以作为门禁、维护或证据，但不能提高核心体验分：

| 成果 | 评分处理 |
| --- | --- |
| 新增 QA 脚本、修复 cleanup、端口无残留、浏览器关闭更稳 | 门禁通过条件，不加分。 |
| `gateScore` 提高、测试数量增加、截图数量增加 | 证据更完整，不加分。 |
| 文档更清楚、评分表更完整、汇总更漂亮 | 项目管理改善，不加分。 |
| HUD 多显示一段 build plan 文案，但奖励、路线、升级和下一战没有变化 | 最多算表达改善；不能突破 `88-89`。 |
| 新增孤立牌或孤立事件，但不会改变连续 run 决策 | 不算内容密度加分。 |
| 只把 sim 能跑通，没有玩家可点击、可读、可复盘的体验 | 不能计入 90+。 |
