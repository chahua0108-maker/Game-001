# 2026-05-20 多局难度阶梯第4轮汇总：路线风险可读性

## 1. 本轮目标

本轮继续 `air-loop` 长循环，处理 D1-D3 已能自然通关后暴露出的路线误读问题。

本轮假设：

```text
如果玩家能一眼区分安全推荐路和高风险贪心路，D1-D3 的通关曲线就不再因为路线按钮误读而掉线。
```

## 2. 审核边界

新增 spec：

- `design/framework/2026-05-20-redline-route-risk-readability-spec.zh.md`

框架守门结论：`approve`，轻审核即可。

允许范围：

- 只改 HUD 展示层。
- 允许给 route choice 派生展示字段。
- 允许改顶部 route summary，让它比较候选集合。
- 允许改 UI test。

明确未做：

- 不新增 route kind。
- 不改路线生成器。
- 不改 `select-route` runtime 结算。
- 不做低血禁用、自动改选或二次确认。
- 不扩 D4-D10。

## 3. 已落地

HUD route choice 新增展示语义：

- `repair-cache` 显示 `推荐/安全 · 无HP代价 · 无污染`。
- `elite-pressure` 显示 `高风险/贪心 · -N HP · 无污染/+污染`。
- 路线按钮仍保留原本收益 token，例如 `复核+1 · 偏修补`、`MP+1 · 偏终结`。
- 顶部“下一战后果”不再只读第一条候选，改为同时比较安全路和高风险路，例如：

```text
安全复核+1/偏修补 | 高风险MP+1/偏终结
```
