## 1. QA 口径

QA 只作为门禁：

- `npm run check` 必须通过。
- `qa:lifecycle` 必须通过。
- `qa:similarity` 必须通过。
- `qa:ui` 必须通过。
- 浏览器验收后必须关闭 page/context/browser/server，端口无残留。

QA 通过不加分；QA 失败则本轮不能验收。

第 14 轮门禁结果：

- `npm run check`：`142 passed / 2 skipped`，build 通过。
- `QA_ROUND=round-14-main npm run qa:lifecycle`：通过，cleanup 通过。
- `QA_ROUND=round-14-main npm run qa:similarity`：通过，cleanup 通过。
- `QA_ROUND=round-14-main npm run qa:ui`：通过，cleanup 通过。
- 三个 browser QA 均记录 `pidAlive=false`、`portListening=false`。

## 2. 本轮核心体验增量

第 14 轮真正增加的是：

- 局内牌强化雏形：`cardUpgrades` 已进入 sim，能影响后续伤害结算。
- 状态/污染压力：污染牌不推进链路，并能挤压抽牌质量。
- 牌生命周期读法：消耗、保留、抽牌、洗回有短事件反馈。
- 卡牌机制密度：补了机制标签、稀有度、护盾、费用变化、抽牌+消耗样本。
- UI 手感底线：手牌和 HUD 在桌面/移动端不会因为这些新增信息直接崩掉。

这些让核心体验从第 13 轮的 `57` 推到 `64`，但仍没有形成完整的 card roguelike run。

## 3. 为什么不是 95

当前最致命缺口不是测试，而是玩家体验仍停在“战斗样片 + sim 能力”：

| 缺口 | 当前问题 |
| --- | --- |
| 短 run 推进 | 玩家还没有从战斗进入路线节点、奖励节点、下一场战斗的连续体验。 |
| 奖励构筑回应 | 奖励还不能稳定回应上一战的缺桥、缺终结、被污染、缺资源。 |
| 局内强化可见选择 | 强化在 sim 层成立，但玩家还不能自然选择、预览、带入下一战。 |
| 内容密度 | 机制样本已经增加，但还不像可反复玩的牌池。 |
| 信息层 | HUD 可读性改善，但仍有 debug 味，缺更自然的玩家语言。 |

## 4. 第15轮方向

第 15 轮必须聚焦短 run 核心闭环：

```text
战斗结束 -> 奖励选择 -> 路线/节点推进 -> 下一场战斗继承构筑变化
```

第 15 轮不要把目标写成“继续加 QA 分”。验收仍要跑 QA，但分数只看玩家是否真的进入了更像竞品的 run 体验。

建议第 15 轮目标分：`64 -> 72`，保守提升 8 分。即使完成，也仍未到 95，需要继续第 16 轮。
