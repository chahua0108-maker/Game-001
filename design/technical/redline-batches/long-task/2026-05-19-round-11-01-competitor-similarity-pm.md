## 0. 一句话裁决

第 11 轮应该继续写代码。

理由不是第 10 轮做得不够，而是成功标准已经改变。第 10 轮裁决基于“工程内 P0 样片可交付”，所以应该停手；第 11 轮基于“越像 Vampire Crawlers 越成功”，所以当前样片只能算相似度中低的核心战斗切片，必须继续迭代。

当前已像的部分：

- 升序 mana / MP 连锁已经存在。
- Wild 修补断链已经存在。
- 快速出牌的短 token 和移动端防溢出已经存在。
- 击杀 XP、升级奖励、run 内加卡已经存在。
- 有一张极窄整备置顶牌，能制造“找终结”的小爽点。

当前不像或明显缺的部分：

- 没有足够稳定的长 stack 延长镜头。
- Wild 还不够像“延长本轮 stack 的引擎”，更像一次修补工具。
- 局内卡牌增强、宝石、升级牌实例还没有真实落地。
- 地牢推进还只是短 run 节点，不像可选择路线。
- 局外升级没有真实玩家目标，只是 preview 边界。
- 奖励、卡牌生命周期、状态/保留/消耗还不足以支撑竞品式构筑变化。

## 1. 相似度 PM 口径

本轮不追求“原创差异化”，也不追求完整系统一次做完。验收方式改为镜头相似度：

```text
玩家连续 3-5 分钟内是否反复看到：
升序连锁 -> Wild 延长 -> 快速多次出牌 -> XP 升级拿卡 -> 局内强化/宝石 -> 地牢推进 -> 局外成长目标。
```

只要这个镜头链不成立，即使测试全绿，也不能按新成功标准判定成功。

## 3. P0-P2 实施分层

### P0：第 11 轮必须继续写的相似度代码

P0 目标是让玩家在 3-5 分钟内更像在玩主标杆的核心循环，而不是在看工程 demo。

1. 升序连锁重复出现：至少两次不同手牌的 `0 -> 1 -> 2` 或 `0 -> Wild -> 2`。
2. Wild 成功修补必须显示“延长 stack”的玩家反馈。
3. 快速出牌验收：5 秒内完成三张合法牌，不牺牲移动端防溢出。
4. 奖励路线标签：延长 / 终结 / 整备 / 保命。
5. 局内卡牌增强最小切片：至少一种 run 内升级牌。
6. 成功链和失败链都可复现，避免脚本化必胜。
7. 版权安全检查进入新增文案验收。
8. 相似度 scorecard 文档化。

### P1：第 11 或第 12 轮可继续写的相似度代码

P1 目标是把核心相似度从“战斗一手牌”扩展到“短 run”。

1. 线性地牢推进 HUD。
2. 宝石/附魔式卡牌增强。
3. 5 节点短 run 奖励节奏。
4. 最小消耗、保留、状态/污染牌生命周期。
5. 局外临时 profile 和下局起始选项。

### P2：等 P0/P1 稳定后再做

P2 目标是完整 roguelite/card framework，不应该抢第 11 轮。

1. 通用地图路线、商店、营地、精英、Boss。
2. 完整卡实例、同名不同升级、临时修改、复制牌。
3. 通用宝石 socket、稀有度、权重、reroll、删牌。
4. 遗物、通用触发器、复杂状态、敌人多意图。
5. 账号存档、永久科技树、云同步。

## 5. 最终 PM 判断

当前 Redline 原型按工程样片可给 `7/10`，按 Vampire Crawlers 竞品相似度只能给 `4/10`。

相似度缺口主要不在“有没有 0->1->2”，而在以下四件事：

1. stack 延长不够频繁、不够爽。
2. 局内强化/宝石还没有玩家可见闭环。
3. 地牢推进和局外目标还没有形成 roguelite 期待。
4. 奖励分叉还没有让玩家觉得自己在选一条构筑路线。

所以第 11 轮的正确方向是：继续写代码，但只写能提高竞品相似度评分的 P0/P1 小切片。任何不能让玩家更明显感到“升序连锁、Wild 延长、快速出牌、升级强化、地牢推进”的改动，都应推迟。
