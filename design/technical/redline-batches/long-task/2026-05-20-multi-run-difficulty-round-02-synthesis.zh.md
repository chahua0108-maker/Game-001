# 2026-05-20 多局难度阶梯第2轮汇总：D1-D2 自然通关验收

## 1. 本轮目标

本轮按 `air-loop` 长循环执行，主线程只做制作人 / 集成监督，不直接承担重体力验证。

本轮假设：

```text
如果 D1 和 D2 能被保守玩家自然打通，第一套闯关活动的初级入口才算站住；如果 D3 仍然断崖，就不能继续提高多局核心体验分。
```

本轮不改基础框架、不扩 D4-D10、不做路线批量实现。

## 2. 并行专家批处理

本轮派出 5 个独立专家：

- 玩家体验审查：判断 D1-D3 是否符合新手可通 -> 轻压力 -> 中级入口。
- 10 档难度 / 数值曲线：提出 D1-D10 曲线和当前 D1-D3 风险。
- 路线 / 关卡节奏：审查 repair-cache / elite-pressure 是否帮助玩家学习。
- QA 自动化工程师：接管自然通关回归测试，只允许修改 sim 测试文件。
- 框架 / spec 守门人：判断本轮是否需要新的框架 spec，划定下一步越界边界。

## 3. 已落地

只改测试，不改生产代码：

- 在 `prototype-web/src/tests/sim/redline-activity-difficulty.test.ts` 增加保守玩家自动出牌 helper。
- 新增 D1 自然通关回归：不强制奖励、不强制结算，完整跑到 D1 胜利。
- 新增 D1 -> D2 自然通关回归：先自然通 D1，再 `continue-activity` 到 D2，再自然通 D2。
- D2 的 HP 余量断言收紧到 `> 24`，避免只证明濒死通关。

## 4. 专家结论

### 4.1 D1-D2

D1 / D2 当前成立：

- D1 是 3 节点、72 HP、敌伤 x0.45、4 奖励候选、高压路线无污染。
- D2 仍是 3 节点、66 HP、4 奖励候选，高压路线只扣 3 HP 且无污染。
- 自动保守策略可以自然通关 D1 与 D2，说明初级门槛不是旧单局那种硬压迫。

### 4.2 D3

D3 当前有断崖风险：

- 节点从 3 直接到 6。
- 敌人 HP / 伤害同时回到 x1.0。
- 奖励候选从 4 降到 3。
- 高压路线同时打开 `-6 HP` 和污染。
- D1 / D2 奖励不会永久带到 D3，D3 实际仍是 starting deck 开局。

结论：D3 现在更像旧高压单局入口，不足以证明“中级入口”。

### 4.3 路线风险

路线专家判断：D1/D2 的路线伤害已经温和，但学习语义不足。

- `repair-cache` 不是即时回血，只是偏修补 / 资源奖励。
- `elite-pressure` 在低血时仍可能误选致死。
- HUD 会显示路线 preview，但缺少安全 / 危险 / 推荐 / 致命风险的强提示。

## 7. 下一轮建议

下一轮不应直接铺 D4-D10。建议先开一个小 spec delta，并让框架程序专家审核：

1. D3 first-clear 调整：把 D3 从断崖改成真正中级入口，优先只动一个主杠杆，例如敌伤倍率、奖励候选或高压污染。
2. 路线误选保护：让 D1/D2 的安全路和贪心路在 HUD 上更明显，低血致死路线要保护或二次确认。
3. D4 只作为污染首秀候选，不急着进入完整 D10 可玩链路。

必须先写 spec 并审核的边界：

- D4-D10 进入 `ActivityLevelId` / `playableLevelIds` / 结算推进合同。
- 批量路线节点、路线图结构、跨节点压力模型。
- 永久活动进度、localStorage、账号成长或跨 run 保留奖励。
- 修改活动 / run / 战斗 / 发牌四层边界。
