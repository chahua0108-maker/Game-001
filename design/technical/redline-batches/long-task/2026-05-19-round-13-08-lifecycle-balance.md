## 0. 总裁决

第 13 轮可以引入生命周期 v1，但不能把它当作“给玩家更多强牌”的入口。消耗、保留、状态牌的正确作用分别是：

```text
消耗：给一次性爆点或一次性修补，但避免永久压缩牌库。
保留：让玩家提前承诺下一回合计划，但必须占 4 手牌位。
状态牌：制造可读坏手和牌库污染，让 reward/净化/抽牌有真实回应对象。
```

对 3-5 回合节奏的核心影响：

- 消耗会让当回合更强、后续循环更薄；如果放在 0 费抽牌或 payoff 上，容易把节奏变成脚本爆发。
- 保留会降低随机性，让 payoff、Wild、2 MP 段更容易被安排到同一回合；如果不占手牌位，会直接破坏 4 手牌压力。
- 状态牌会抬高坏手率和 End Turn 压力；如果状态牌能参与 0->1->2 升序链，就会从惩罚变成免费链段，必须禁止。

一句话裁决：**生命周期 v1 的目标不是让 3-5 回合更稳，而是让“好手、坏手、补救、奖励回应”更像卡牌解局。95 分路径需要至少一次生命周期造成的问题，以及一次玩家主动用奖励或计划回应这个问题。**

## 1. 当前基线

当前可确认的节奏基线：

| 项 | 当前值 | 平衡含义 |
| --- | ---: | --- |
| 玩家 HP | 60 | 可承受约 3 个无操作 End Turn，但第 4 个会接近失败。 |
| 当前 MP / Max MP | 3 / 3 | 0->1->2 正好花完，防止 Wild MP3 默认成立。 |
| 发牌数 | 4 | 每回合只有 4 个选择，坏手与取舍才成立。 |
| 起始牌 | `debt_hook / heartbeat_spark / redline_cut / row_cleave` | 0/1/1/2 教学骨架。 |
| 首奖 XP 阈值 | 12 | 通常应在 3-4 回合触发。 |
| 首排意图 | 约 17 | 不行动约三回合后 HP 到 9。 |
| 奖励选择数 | 3 | 修补、终结、路线三类回应。 |

生命周期 v1 应建立在这组值上，不应在同一刀里顺手调整基础经济。

## 2. 三类生命周期的节奏影响

### 2.1 消耗

消耗的正面价值：

- 允许奖励给玩家一个“现在救场”的高影响工具，但不会每轮都稳定复现。
- 可以承载高效率抽牌、净化、临时防压或小爆发，避免这些效果永久进入循环。
- 能制造竞品感里的“这一手赌下去”的短期决策。

主要风险：

- 消耗牌如果从牌库循环中永久拿掉，会让 deck 变薄，后续更稳定抽到 0/1/2/payoff。
- 0 费消耗抽牌如果也推进链路，会同时给开链、抽牌和压缩牌库，过强。
- payoff 消耗如果数值过高，会变成“拿到就赢一次节点”，奖励选择失去比较。

平衡口径：

- P0 消耗最好放在 reward/reserve-test，不要改起始牌。
- 消耗只表示本节点/本段循环离开抽弃，不应改变 run deck 所有权。
- 消耗牌的强度可以高于同费普通牌，但必须用“一次性”和“不可反复洗回”付费。

### 2.2 保留

保留的正面价值：

- 让玩家把 payoff、Wild 或 2 MP 段留到下一手，形成“我现在不打，是为了下一回合”的计划感。
- 帮助坏手玩家减少纯随机挫败，提高 3-5 回合可读性。
- 可以让奖励回应上一手缺口：缺终结就拿保留终结，缺路线就保留 2 MP 段。

主要风险：

- 如果保留不占手牌位，实际等于提高发牌数，4 手牌压力消失。
- 如果保留能跨 reward / node，玩家会把所有爆点带到下一节点开局，破坏节点节奏。
- 如果多张保留同时存在，3-5 回合会从“解局”变成“预排脚本”。

平衡口径：

- 保留牌必须占下一手 `HAND_SIZE=4` 的位置。
- `reward selected` / 节点切换不应触发保留，手牌必须强制丢弃或清理。
- P0 早期最多允许 1 张保留牌进入自然牌池；多保留先留给测试场景。

### 2.3 状态牌

状态牌的正面价值：

- 它是最直接的坏手制造器：占抽牌、占手牌、降低 action density。
- 它让 `blood_tithe / pulse_draw / paper_shatter / 净化` 有明确问题可回应，而不是纯粹变强。
- 它能让 3-5 回合出现一次公平失败：玩家知道自己被污染拖慢，而不是 UI 或随机坏。

主要风险：

- 当前升序链按费用推进；如果状态牌可打且能参与费用链，0/1/2 状态牌会变成免费链材料。
- 不可打状态牌会新增灰按钮/死牌 UI 负担，移动端读错概率高。
- 过早、过多状态牌会把首奖前节奏压垮，玩家还没学会链路就被污染。

平衡口径：

- 状态牌必须是“链路惰性”：可打也不能推进 `0->1->2`、不能给倍率、不能给授权。
- P0 自然流程中，首奖前最多插入 1 张状态牌；2 张以上只用于压力测试。
- 普通状态牌应在节点结束清理，不进入 reward pool，不变成 run/meta 诅咒。

## 3. 95 分路径定义

这里的 95 分路径不是最高胜率脚本，而是一次高质量 3-5 回合体验：

| 维度 | 分值 | 达标口径 |
| --- | ---: | --- |
| 压力可读 | 20 | 玩家能在出牌前说出本回合会掉多少血。 |
| 升序链高光 | 20 | 至少一次 0->1->2 授权或 Wild MP3 延链成立。 |
| 生命周期取舍 | 20 | 消耗、保留、状态牌至少一类制造真实取舍，而不是纯收益。 |
| 奖励回应 | 20 | 奖励选择能解释为回应上一手缺口：缺牌、缺段、缺终结、被污染。 |
| 失败公平 | 10 | 至少一次断链、抽牌未命中、状态污染或缺 MP 的后果与预览一致。 |
| UI 短读 | 10 | 移动端只靠短 token 读懂：`消耗`、`保留`、`状态`、`仍-X`、`授权付`。 |

95 分路线应长这样：

```text
回合 1：17 意图可读，玩家打 0->1->2，看到授权但仍有压力。
回合 2：抽到状态牌或缺关键段，坏手被看见；玩家用抽牌/修补尝试找解，但压力不消失。
回合 3：首奖出现，三选一回应真实问题：拿消耗救场、拿保留规划、拿净化/抽牌处理污染。
回合 4：玩家用保留或消耗兑现一次 x4 payoff；如果想 x5，仍需要额外当前 MP，不是默认脚本。
回合 5：消耗牌离开循环、状态牌被净化或继续污染，下一轮仍有压力和选择。
```

如果 3-5 回合每次都稳定走到 `0->1->2->Wild->payoff`，即使爽，也只能算 80 分左右，因为它缺少坏手、取舍和奖励回应。

## 5. 不要动的数值与规则

这些值在生命周期 v1 中不要动；如果后续要调，应作为单独平衡轮处理：

| 项 | 当前值/规则 | 不动理由 |
| --- | --- | --- |
| 基础 `maxEnergy` | 3 | 0+1+2 正好花完，是 Wild MP3 不能默认成立的核心护栏。 |
| 基础发牌数 | 4 | 保留必须通过占手牌位付费，不能顺手升到 5。 |
| 起始牌费用 | 0/1/1/2 | 生命周期不应破坏最初教学链。 |
| 起始牌生命周期 | 默认打出进 discard，回合末进 discard | 不要让新机制抢占首屏学习成本。 |
| 玩家 HP | 60 | 当前 17 意图压力已可读；生命周期先不靠改血量解决问题。 |
| 首排总意图 | 约 17 | 状态牌引入后先观察压力，不要同时降敌意图。 |
| `tempAuthorizationMP` | +3，payoff-only | 保证授权只服务 3 MP 终结。 |
| `wild_gap_key` printed cost | 1 | 防止 `0->1->2->Wild` 在默认 3 MP 下无成本成立。 |
| `wild_gap_key` MP3 延链 | 只延链，不返 MP，不追加授权 | 防止 x5 变常规路线。 |
| `wild_mana_stitch` | 真实修补才当前 MP +1，不延 MP3 | 防止 0 费抽牌变成长链引擎。 |
| `paper_shatter` 搜索范围 | 只搜 `drawPile` | 防止 2 MP 支援变全牌库 tutor。 |
| 首奖选择数 | 3 | 奖励回应必须维持修补/终结/路线的比较。 |
| reward 切换 | 不允许保留跨 reward / node | 防止存牌开局和节点压力失真。 |
| 状态牌奖励池 | 不进正式 reward pool | 污染是问题，不是玩家主动构筑奖励。 |

## 6. 可调项

生命周期 v1 的调参应集中在这些项上：

| 项 | 建议初始值 | 可调方向 | 触发条件 |
| --- | ---: | --- | --- |
| 首奖前状态牌数量 | 0-1 | 压力不足时增至 1；严禁自然 2+。 | 玩家 3 回合内总能稳定 x4，无坏手。 |
| 状态牌进入时机 | 第 2-3 回合 | 可提前/延后 1 回合。 | 玩家还没学会链路就被污染，需延后。 |
| 状态牌链路行为 | 链路惰性 | 可打但不推进链；或暂时不可自然出现。 | 当前实现无法禁止状态推进链时。 |
| 状态牌清理时间 | node-end purge | 可增加净化牌提前清理。 | 状态在节点间残留导致 run 变诅咒。 |
| 消耗牌数量 | 每节点 1-2 张可见 | 可按 reward 稀有度调整。 | deck 过薄或救场不足。 |
| 消耗牌强度 | 同费普通牌 +20%-40% 效率 | 降低伤害/抽牌，或提高费用。 | 拿到消耗后奖励固定最优。 |
| 保留牌自然数量 | 0-1 张 | 后续可增至 2，但先不进 P0。 | 坏手太多且玩家无法计划。 |
| 保留上限 | 占 4 手牌位，不额外扩手 | 可加“每回合最多保留 1”。 | 多保留导致固定 combo。 |
| 首奖 XP 阈值 | 12 | 可试 14/15。 | 生命周期让奖励稳定早于第 3 回合。 |
| payoff 伤害 | 16 / 12 | 可降 `severance_burst` 到 14-15，或限制 x5。 | 保留让 x4/x5 过稳。 |
| 抽牌倍率上限 | 线性倍率 | 可对 self draw cap 到 3。 | 保留 + 抽牌让找牌过稳。 |
| 净化奖励权重 | 低频回应 | 可在状态压力高时提升。 | 玩家被污染但 reward 没有回应。 |

## 7. 奖励响应建议

生命周期 v1 后，奖励不应只问“哪个伤害高”，而应问“上一手暴露了什么问题”：

| 上一手问题 | 推荐奖励回应 | 不推荐回应 |
| --- | --- | --- |
| 缺 0/1/2 路线段 | route-bridge 或保留路线段 | 直接给最高 payoff。 |
| 缺终结 | 温和 payoff 或可保留 payoff | 直接给 x5 所需全套资源。 |
| 被状态污染 | 净化、抽牌、链路惰性清理工具 | 纯伤害补偿，掩盖污染问题。 |
| 当前压力过高 | 一次性消耗救场 | 永久 Max MP 或永久清场。 |
| 抽牌未命中 | `paper_shatter` 类窄整备或保留计划 | 全牌库 tutor。 |
| 缺当前 MP | 受控修补，不提高 Max MP | 默认 Max MP +1。 |

首奖可以出现一张生命周期牌，但不应三张都在讲新规则。推荐结构：

```text
修补/净化回应一张
+ 温和 payoff 一张
+ 路线/保留计划一张
```

这能让奖励既回应问题，又保留比较，而不是强迫玩家学习三种新机制。

## 8. P0 平衡验收

生命周期 v1 合格，不看“是否更强”，看这些结果：

| 指标 | 合格线 |
| --- | --- |
| 起手教学 | 第 1 回合仍能自然读出 0->1->2。 |
| 状态污染 | 首奖前最多自然出现 1 张状态牌。 |
| 状态链路 | 状态牌不会触发 `ChainAdvanced`、`AuthorizationGranted` 或 payoff 倍率。 |
| 保留 | 保留牌占下一手 4 手牌位，不额外补满到 5。 |
| reward 边界 | reward selected 后没有手牌跨节点保留。 |
| 消耗 | 消耗牌离开抽弃循环，但不从 run deck 所有权中永久删除。 |
| 压力 | End Turn 预览和实际扣血一致，生命周期不隐藏 `仍-X`。 |
| 奖励响应 | 至少一个 reward 选择能解释为回应状态/坏手/缺终结。 |
| 脚本率 | 3-5 回合不应稳定每局 x5。 |
| 移动端 | `消耗`、`保留`、`状态` token 不超框，不挤掉压力读数。 |

## 9. 结论

消耗、保留、状态牌可以把 Redline 的 3-5 回合从“成功链样片”推进到“卡牌解局样片”，但前提是它们都要带代价：

- 消耗的代价是离开循环，不是永久变强。
- 保留的代价是占下一手手牌位，不是免费多抽。
- 状态牌的代价是占抽牌和选择，不是免费链段。

第 13 轮最稳的平衡路径是：先放一张受控消耗牌、一张占位保留牌、一张链路惰性状态牌；不改 3 MP、4 手牌、17 意图、0/1/2 授权和 Wild MP3 护栏。这样 95 分路径会来自“玩家看见问题并回应问题”，而不是来自系统把完整爆发脚本交到手里。
