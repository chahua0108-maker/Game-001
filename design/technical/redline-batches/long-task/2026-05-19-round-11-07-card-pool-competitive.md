## 0. 一句话裁决

第 11 轮不要堆新卡。当前最缺的不是数量，而是让现有短切片在 3-5 回合内更像竞品的“构筑爆发”：

```text
0 费起手建立路线
-> 1/2 费桥接给出可读链路
-> Wild / draw / refund / cost-down 修补坏手
-> 3 费 payoff 在授权窗口清场
-> 奖励回应这条路线，并改善下一手
```

P0 最小建议：只允许新增或开放 2 张以内内容牌，优先复用已有 `blood_tithe` / `pulse_draw` 作为奖励后备；如果必须新增，只新增 1 张“成本降低”支援牌。不要同时新增宝石、遗物、升级、完整 tutor、弃牌搜索或新 UI 面板。

## 1. 设计边界

### 1.1 本轮目标

让卡池读法从“我有几张伤害牌”变成：

```text
我在找 0 -> 1 -> 2。
我有办法修一次坏手。
我能把修补转成 3 费终结。
奖励会让我下一手更稳定或更爆。
```

### 1.2 本轮不做

- 不做完整新卡包。
- 不做通用宝石系统。
- 不做遗物、局外成长、删牌、升级商店。
- 不做完整 scry / tutor / discover UI。
- 不做 discardPile 搜索。
- 不把奖励池改成复杂权重系统。
- 不复制竞品的卡名、关键词、文本、数值模板、美术、UI 构图或音效。

## 2. 当前卡池的核心问题

当前样片已经有 0/1/2 链、Wild 修补、抽牌、topdeck payoff、奖励进入下一手和 3 费终结。问题是玩家内容镜头还不够连续：

| 问题 | 玩家感受 | 第 11 轮修法 |
| --- | --- | --- |
| 0 费入口存在，但奖励不总是强化入口密度 | “我只是抽到能打的牌” | 首奖 route 固定给清楚 0 费入口 |
| 1/2 桥接像费用顺序，不像 build 选择 | “我按规则出牌” | 桥接牌要分成伤害桥、抽牌桥、授权桥 |
| 3 费 payoff 强，但像系统奖励 | “游戏让我放大招” | payoff 必须被 0/1/2 或整备亲手打开 |
| Wild / draw / refund 都有，但分工重叠 | “修补牌都是万能按钮” | 每类修补只解决一个缺口 |
| 奖励三选一可用，但爆发预告弱 | “拿了牌，不知道下一手会怎样” | 奖励文档口径必须标注补 0、补桥、补终结 |

## 3. 至少 10 个内容镜头

这些是第 11 轮应该服务的玩家瞬间。镜头比“新增多少张牌”更重要。

### 镜头 01：0 费起手不是低级伤害，而是路线开关

玩家第一眼看到 0 费牌时，应读成“这张牌让我开始构筑”，不是“免费刮痧”。

P0 内容要求：

```text
0 费起手牌至少承担一个路线角色：
开链 / 标记 / 低伤害补刀 / 为 1 费桥接提供条件
```

建议保留 `spark_tap` / `blood_reclaim` 作为 route 首层，不新增更多 0 费攻击。奖励池首个 route 槽优先给 `spark_tap`，因为它比 2 费 self draw 更像“我补了入口密度”。

### 镜头 02：1 费桥接要让玩家感觉“接上了”

1 费牌不是中间数值，它要证明玩家没有乱打。

P0 内容要求：

```text
0 -> 1 成功后，反馈应让玩家知道：
我保持了链路
我获得了更高倍率或下一段期待
我正在靠近 2 费授权
```

卡池上不要新增第三张相似 1 费攻击。优先让 `verdict_mark` 或 `heartbeat_spark` 在奖励后备位承担 1 费承接。

### 镜头 03：2 费桥要区分“展开”和“整备”

2 费应该有两种读法：

| 类型 | 玩家读法 | P0 卡池位置 |
| --- | --- | --- |
| 展开桥 | 我现在解决前排压力 | `clearance_order` / 类似 2 费范围牌 |
| 整备桥 | 我牺牲当前输出，准备抽到 payoff | `paper_shatter` 后置 |

P0 不建议再加新的 2 费整备牌。`paper_shatter` 已经足够承担 topdeck 样片，首奖 route 不应先给它。

### 镜头 04：3 费 payoff 必须像“我构筑出来的爆发”

3 费牌不能像独立大招。它的高光必须来自玩家刚完成的 0/1/2 链路或刚用整备置顶。

P0 内容要求：

```text
未授权：3 费 payoff 只是可用但不爆。
已授权：3 费 payoff 变成前排清算 / 全场清算 / 溢出终结。
```

奖励池首个 payoff 建议仍用较温和的 `red_ledger_burst`，高上限的 `severance_burst` 放第二层。这样玩家第一次拿 payoff 是学会方向，不是立刻被数值答案盖住。

### 镜头 05：Wild 修补要像“补缺口”，不能像万能外挂

Wild 的竞品感不是“任何时候都强”，而是“我差一段时，它救了链”。

P0 内容要求：

```text
Wild 只在真实修补缺口时返当前 MP 或给主要收益。
没有修补缺口时，它仍可打，但收益降档。
```

`wild_gap_key` 更适合作为首个 repair 奖励，因为它教学清楚。`wild_mana_stitch` 过于万能，建议继续后置，除非同步降档。

### 镜头 06：抽牌修补要制造“坏手被我救回来”

抽牌牌的价值不是多抽，而是在坏手里找到下一段。

P0 内容要求：

```text
0 费 draw：坏手先开链，找 1/2。
1 费 draw：接在 0 后牺牲伤害，找 2/payoff。
2 费 prep draw：接在路线后，把 payoff 置顶再抽。
```

建议开放 `blood_tithe` 和 `pulse_draw` 作为奖励池后备，而不是起始牌组。它们能让第二、第三次奖励开始出现“我拿了修补包”的构筑感。

### 镜头 07：mana refund 要变成“奖励我打对顺序”

返费不能泛化成资源膨胀。它应该是玩家按正确路线出牌后的局部奖励。

P0 内容要求：

```text
返当前 MP 的触发条件必须绑定：
真实修补成功 / 正确接链 / 支援牌完成它承诺的动作
```

不要新增新的纯返费牌。现有 Wild 修补已能承担 refund 样片；新内容只需要在说明和排序上把它放到正确镜头里。

### 镜头 08：成本降低是比“宝石系统”更小的 P0 替代品

玩家想要竞品式构筑爆发，常见路径是升级、宝石、附魔、减费、复制。第 11 轮不能开大系统，因此 P0 只建议一个最窄替代：

```text
新增 1 张奖励支援牌：
本回合下一张 1/2 费桥接牌 -1 当前 MP，若因此接链成功，再抽 1 或返 1。
```

暂定内容名只作内部占位，不作为最终卡名：

| 占位 | 费用 | 分支 | 作用 |
| --- | --- | --- | --- |
| `redline_discount_patch` | 1 | `repair-resource` | 降低下一张桥接牌成本，只服务接链，不服务 payoff 逃费 |

约束：

- 不降低 3 费 payoff。
- 不跨回合保留。
- 不叠加。
- 不需要新 UI，只用短反馈：`降费：下一桥 -1`。

如果实现风险高，本轮可以只写入 backlog，不进 P0 代码切片。

### 镜头 09：卡牌增强 / 宝石的替代方案要先用“奖励变体”

不要在第 11 轮做宝石孔、镶嵌、升级树。最小替代是奖励池出现“同一角色的变体”，让玩家感觉自己在调构筑。

P0 替代结构：

```text
不是给卡装宝石。
而是在奖励池里给三类变体：
补入口 / 补桥 / 补修补。
```

示例读法：

| 玩家想要的宝石感 | P0 替代 |
| --- | --- |
| 让牌更稳定 | 给 0 费 draw 或 Wild 后备 |
| 让链更长 | 给 1 费 draw 或 2 费整备 |
| 让爆发更大 | 给第二张 payoff，但放在奖励第二层 |
| 让费用更顺 | 只给一次性成本降低支援，不改卡本体 |

这个替代方案更安全，因为它不需要卡实例、升级状态、存档和 UI 新面板。

### 镜头 10：奖励节奏要像“下一手答案”，不是随机礼物

奖励三选一应按玩家刚经历的问题回应：

```text
缺入口 -> 给 0 费 route
缺 1/2 -> 给 bridge
缺牌 -> 给 draw
缺费 -> 给 Wild / refund
能接链但缺爆发 -> 给 payoff
```

P0 不要求动态推荐算法，但奖励池排序必须让首奖读成：

```text
wild_gap_key / red_ledger_burst / spark_tap
```

开放抽牌修补后，建议第二层 repair 后备是：

```text
blood_tithe -> pulse_draw -> wild_mana_stitch
```

这样玩家连续拿修补时，会从 Wild 缺口修补转到抽牌续航，再到强力万能修补，形成成长感。

### 镜头 11：版权边界必须锁死在“抽象结构”

竞品可以启发的是机制结构，不是可复制资产。

允许抽象：

- 低费启动。
- 中费桥接。
- 高费终结。
- 抽牌修补。
- 返费奖励。
- 成本降低。
- 顶牌 / 整备。
- 奖励池按构筑角色分支。

禁止复制：

- 竞品卡名、关键词、规则文本。
- 竞品 UI 排版、图标、音效、动画节奏。
- 竞品特定数值模板和卡组套路。
- 竞品角色、怪物、主题包装和叙事语气。

Redline 的表达必须围绕自己的词：

```text
链路、意图、修补、整备、授权、终结、当前 MP
```

### 镜头 12：失败也要服务爆发

竞品的构筑爆发通常需要一次“差点断”的前戏。Redline 应保留失败镜头：

```text
手牌缺 2
-> 用 0 费 draw 找不到
-> 用 Wild 补缺口但伤害不足
-> 结束回合吃一点意图
-> 奖励拿到 bridge / draw
-> 下一手完成 0 -> 1 -> 2 -> payoff
```

P0 不要把修补牌做成永远救场成功。坏手被部分修好，比每回合都爆更像构筑游戏。

## 4. P0 最小内容建议

### 4.1 最小切片 A：零新增卡，仅改奖励意图

如果第 11 轮完全不能加卡，P0 只建议锁奖励顺序和镜头口径：

```text
repair 首奖：wild_gap_key
payoff 首奖：red_ledger_burst
route 首奖：spark_tap
repair 后备：wild_mana_stitch
payoff 后备：severance_burst
route 后备：verdict_mark -> clearance_order -> paper_shatter
```

效果：玩家第一轮奖励会清楚看到“修补 / 终结 / 入口”，而不是过早看到 2 费整备支援。

### 4.2 最小切片 B：开放 2 张已有 reserve 修补牌

如果允许开放极少内容，优先开放：

| 卡 | 角色 | 放入位置 | 理由 |
| --- | --- | --- | --- |
| `blood_tithe` | 0 费 draw 修补 | 第二层 repair | 坏手先开链，找 1/2，不抢首奖 |
| `pulse_draw` | 1 费 draw 修补 | 第三层 repair / route 后 | 接 0 后找 2/payoff，有代价 |

推荐奖励读法：

```text
wild_gap_key
red_ledger_burst
spark_tap
blood_tithe
severance_burst
verdict_mark
pulse_draw
wild_mana_stitch
clearance_order
paper_shatter
```

这仍然不是“堆很多新卡”，只是把已有修补牌放到正确的奖励节奏。

### 4.3 最小切片 C：仅 backlog 1 张成本降低牌

如果制作人认为抽牌修补还不够像构筑爆发，可以把 `redline_discount_patch` 放入 backlog，而不是本轮实现：

```text
1 MP 支援。
下一张 1/2 MP route-bridge 当前 MP -1。
若因此接链成功，抽 1 或返 1。
不影响 3 MP payoff。
```

它的价值是替代宝石 / 升级 / 附魔系统，让玩家感到“我正在改造费用曲线”，但技术面仍是单张卡效果。

## 5. 奖励节奏建议

### 5.1 前三次奖励应该讲一个故事

```text
第 1 次：修补 / 温和 payoff / 0 费入口
第 2 次：抽牌修补 / 高上限 payoff / 1 费桥
第 3 次：强力 Wild / 2 费展开 / 整备 topdeck
```

这样 3-5 回合内会出现：

```text
第一次知道路线
第二次补稳定性
第三次把稳定性转成爆发
```

### 5.2 奖励文案标签

即使不改 UI，本轮文档口径也应要求后续卡面 / HUD 用短标签描述奖励用途：

| 标签 | 含义 |
| --- | --- |
| 补 0 | 提高起手开链概率 |
| 补 1 | 接 0 后稳定进入第二段 |
| 补 2 | 进入授权前的展开 |
| 修缺口 | Wild 或降费，用于断链 |
| 补手牌 | draw，用于找下一段 |
| 备终结 | topdeck / prep payoff |
| 终结 | 3 费 payoff |

标签目标是让奖励选择变成“我修哪条构筑线”，不是“哪张数字高”。

## 6. 验收口径

第 11 轮内容切片不以卡牌数量验收，而以镜头验收：

| 验收项 | 通过标准 |
| --- | --- |
| 0 费起手 | 第一手或首奖能读到开链入口 |
| 1/2 桥 | 至少一条路线能稳定 0 -> 1 -> 2 |
| 3 费 payoff | payoff 爆发来自授权或整备，不是裸打 |
| Wild | 至少一次真实补缺口，而不是无条件返费 |
| 抽牌 | 至少一次 draw 让坏手继续找下一段 |
| mana refund | 返费绑定正确顺序或真实修补 |
| 成本降低 | 若进入 backlog，明确不影响 3 费 payoff |
| 宝石替代 | 不开宝石系统，用奖励变体替代 |
| 奖励节奏 | 首奖三分支清楚，二奖不断供 |
| 版权边界 | 只抽象结构，不复制竞品表达 |

## 7. 最终 P0 建议

优先级从高到低：

1. 锁定奖励首奖读法：`wild_gap_key / red_ledger_burst / spark_tap`。
2. 开放已有 `blood_tithe`、`pulse_draw` 作为奖励后备，不进起始牌组。
3. 继续后置 `paper_shatter`，让它作为第三层“整备终结”高光，而不是首奖 route。
4. 暂不做宝石 / 升级系统；用奖励变体承担增强感。
5. 成本降低只写入 backlog，除非本轮需要一个明确的新内容点；若做，只做 1 张且不影响 3 费 payoff。

一句话：第 11 轮最小内容切片不是“多加卡”，而是把现有牌池排成一条可被玩家复述的爆发路径：起手、桥接、修补、授权、终结、奖励再强化下一手。
