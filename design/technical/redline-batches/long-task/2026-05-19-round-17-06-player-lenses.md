## 0. 本轮裁决

第17轮的玩家体验重点不是再证明“奖励、路线、升级这些系统存在”，而是证明玩家能在一次短 run 中形成并修正计划：

```text
上一战暴露问题
  -> 奖励给出回应
  -> 玩家拿奖励后形成下一战 plan
  -> 路线选择改变 plan 的优先级或风险
  -> 升级选择进入下一战验证
  -> 污染 / 缺桥 / 缺终结让玩家感到具体压力
  -> 下一次奖励或路线继续回应这些问题
```

如果玩家只能说“这张牌强”“这条路线奖励多”，但说不出“我现在这局的计划是什么、路线改变了什么、下一战验证了什么”，第17轮不应给核心体验大幅加分。  
如果玩家能在 3-5 个节点内自然说出上述因果链，第17轮才可以从第16轮的 `80 / 100` 向 `86 / 100` 推进。

## 2. 10 个玩家视角镜头

### 镜头 1：奖励后立刻形成下一战 plan

玩家画面：

- 第一战结算展示上一战问题标签，例如 `缺桥`、`缺终结`、`污染进牌库`、`承压漏血`。
- 奖励选项不只显示收益，还显示它解决哪个问题。
- 玩家选择奖励后，界面给出一句 plan 摘要：`下一战计划：用 Bridge Kit 接上 0->1->2，再找终结牌收束`。

玩家应理解：

```text
我不是随便拿一张强牌；我拿它是为了下一战解决刚刚暴露的问题。
```

失败信号：

- 奖励原因只写 `rare`、`damage+`、`value`，玩家无法关联上一战。
- 选择奖励后没有 plan 摘要，玩家只是在继续点按钮。
- plan 摘要过长或像 debug 字段，玩家读完仍不知道下一战要做什么。

可验收条件：

- 奖励选择前后都能看到至少一个上一战问题与一个回应理由。
- 选择后生成 1 句短 plan，必须包含 `要补什么` 和 `下一战怎么验证`。
- 玩家能回答：“你刚拿的奖励是为了下一战解决什么？”

核心体验打分影响：

- 奖励构筑回应：`+0.8`。奖励从战后抽奖变成下一战计划入口。
- 短 run / 地牢推进：`+0.4`。战斗之间开始有明确连续目标。
- QA 不计分；自动化只能证明这个镜头没有断线，不能替代玩家口述。

### 镜头 2：路线选择前，玩家能比较两个 plan

玩家画面：

- 奖励后出现两个路线候选，每条路线都用玩家语言改写当前 plan。
- 示例：
  - `维修补给：保留补桥计划，下一战多一次清污机会`
  - `精英压力：放弃稳清污，换 +1 临时 MP 和更高奖励`
- 两个候选必须让玩家看到“同一个奖励 plan 会被路线改成两种方向”。

玩家应理解：

```text
我现在不是选图标，而是在决定刚才的奖励计划走稳妥修复还是高压回报。
```

失败信号：

- 路线只显示名字，没有说明对当前 plan 的影响。
- 两条路线都写“奖励更好”，没有风险/节奏差异。
- 路线候选不引用上一战问题或刚拿的奖励。

可验收条件：

- 每条路线都必须写出对当前 plan 的影响：保留、加强、改变、延后或冒险。
- 至少一条路线偏修复，至少一条路线偏风险/收益。
- 玩家能在选择前说出：“选 A 我的 plan 是什么，选 B 我的 plan 会怎么变。”

核心体验打分影响：

- 短 run / 地牢推进：`+0.7`。路线节点从流程按钮变成计划分叉。
- 奖励构筑回应：`+0.4`。奖励和路线开始共同塑造 build plan。
- 敌意图、血量与失败压力：`+0.2`。如果高压路线风险可预判，可小幅加分。

### 镜头 3：路线选择后，plan 摘要发生可见变化

玩家画面：

- 玩家点路线后，旧 plan 被改写为新 plan，而不是只记录 `routeId`。
- 示例：选择 `精英压力` 后，摘要从 `先补桥再终结` 变成 `用 +1 MP 强行接桥，优先找终结，否则会漏 8 点伤害`。
- 下一战开局前能看到“奖励计划 + 路线修正 + 风险后果”三段短 token。

玩家应理解：

```text
路线已经改变了我的下一战计划；我知道自己获得了什么，也知道代价是什么。
```

失败信号：

- 路线选择后只出现 `selected` 或 `node advanced`。
- 计划摘要没有变化，玩家感觉路线没有影响。
- 只强调收益，不说明风险或改动后的打法。

可验收条件：

- 路线选择前后的 plan 摘要至少有一处实质变化：资源、敌压、奖励、清污、升级机会或目标优先级。
- 下一战首屏能复述这条变化。
- 玩家能回答：“刚才路线把你的 plan 改成什么了？”

核心体验打分影响：

- 短 run / 地牢推进：`+0.8`。路线选择有即时反馈，节点连续性更强。
- 玩家反馈与操作手感：不建议加分；若摘要不清楚，应反扣对应镜头。
- 敌意图、血量与失败压力：`+0.3`。风险被写进计划后，压力更可读。

### 镜头 4：升级后下一战首回合验证

玩家画面：

- 玩家在路线或奖励中升级一张本 run 内用过的牌。
- 下一战首回合，升级牌带有明确标记和前后差异，例如 `2伤害 -> 4伤害`、`授权+1 -> +3`、`抽1 -> 抽2`。
- 打出升级牌时，局势发生可见变化：少漏血、补上桥、成功触发终结或清掉关键敌意图。

玩家应理解：

```text
我上一节点升级的不是永久账号数值，而是这次 run 里下一战马上能验证的战术改变。
```

失败信号：

- 升级只在卡详情里，实战中没有可见差异。
- 升级牌下一战没出现，玩家无法验证选择。
- 升级被描述成永久成长，破坏 run 内边界。

可验收条件：

- 升级对象、升级前后差异、下一战验证结果三者必须连起来。
- 下一战至少一次出现升级牌被打出或被抽到的可见证据。
- 玩家能回答：“这张升级牌刚刚帮你改变了什么？”

核心体验打分影响：

- 局内牌成长 / gem-like 成长：`+1.0`。升级从数值选择变成可验证成长。
- 战斗决策与升序爆点：`+0.4`。如果升级帮助完成 0->1->2 或 payoff，可加战斗分。
- QA 不计分；截图中能看到升级标记只是门禁，不是加分理由。

### 镜头 5：污染进入手牌，玩家感到抽牌质量下降

玩家画面：

- 污染因上一战漏压、路线风险或事件进入牌库。
- 下一战抽到污染时，玩家看到一个有效行动位被占用，或者必须花资源清理/绕过。
- HUD 用短句说明：`污染占手：本回合少一张可用桥牌`。

玩家应理解：

```text
污染不是背景惩罚；它真的让我这一手更难接链、更难处理敌意图。
```

失败信号：

- 污染只存在于 deck count 或日志，手牌里没有体感。
- 污染不影响出牌计划，甚至能参与有利链路。
- 玩家不知道污染来源，以为是随机塞牌。

可验收条件：

- 污染至少一次进入抽牌/手牌/弃牌生命周期。
- 污染必须造成可口述的机会成本：少桥、少终结、少防御、少抽牌或多花资源。
- 后续至少出现一次回应入口：清理、稀释、保留规避或用升级牌压过污染。

核心体验打分影响：

- 牌区生命周期与状态污染：`+0.8`。污染开始有抽牌质量体感。
- 敌意图、血量与失败压力：`+0.4`。污染造成可预判压力，而非随机惩罚。
- 奖励构筑回应：`+0.2`。如果后续奖励回应污染，可小幅加分。

### 镜头 6：缺桥时，玩家看到链路断在中段

玩家画面：

- 玩家有 0 费启动和 2 费 payoff，但缺少 1 费桥牌或授权桥。
- UI 不把这表现成“你不能出牌”的硬阻塞，而是让玩家看到链路断点：`缺 1 费桥，payoff 无法放大`。
- 奖励或路线提供补桥方向。

玩家应理解：

```text
我的问题不是伤害不够，而是中间缺桥，导致爆点接不上。
```

失败信号：

- 玩家只感觉手牌差，不知道差在哪里。
- 缺桥和缺终结都被描述成同一种“缺输出”。
- 补桥奖励没有被系统识别为回应问题。

可验收条件：

- 至少一次出现“能启动但接不上”的局面。
- 断点反馈用玩家语言说明缺的是桥，而不是 raw combo state。
- 后续奖励、路线或升级至少一个明确回应 `补桥`。

核心体验打分影响：

- 战斗决策与升序爆点：`+0.7`。玩家能理解链路结构，而非只看费用亮灭。
- 奖励构筑回应：`+0.3`。补桥成为可识别的构筑需求。
- 玩家反馈与操作手感：不因显示提示加分；提示不清楚则不能验收。

### 镜头 7：缺终结时，玩家感到“铺好了但收不掉”

玩家画面：

- 玩家成功启动并接桥，倍率、授权或资源已经到位。
- 但手里没有终结牌，敌人仍残血或下一回合会反打。
- 战后问题标签明确写成 `缺终结`，并在奖励里给出至少一个终结回应。

玩家应理解：

```text
我不是不会打链路；我这套现在缺的是把优势转成击杀的终结牌。
```

失败信号：

- 铺链成功和终结失败没有区分，玩家只看到敌人没死。
- 奖励继续给桥或清污，但没有回应终结缺口。
- 终结牌只是高伤害，没有和前面铺垫产生放大关系。

可验收条件：

- 至少一次出现 `链路已成但未收束` 的局面。
- 战后奖励原因能标记 `找终结` 或等价表达。
- 玩家能回答：“这一战你是缺桥、缺终结，还是缺清污？”

核心体验打分影响：

- 战斗决策与升序爆点：`+0.6`。爆点目标更清楚。
- 奖励构筑回应：`+0.5`。奖励能回应终结缺口。
- 短 run / 地牢推进：`+0.2`。终结缺口成为下一节点计划。

### 镜头 8：污染、缺桥、缺终结三者被区分，而不是混成失败

玩家画面：

- 一次短 run 中，系统至少能区分三类体感：
  - `污染`：抽牌质量变差。
  - `缺桥`：链路中段接不上。
  - `缺终结`：铺垫完成但收不掉。
- 奖励、路线或升级候选分别对应这些问题，而不是都给泛用强牌。

玩家应理解：

```text
我知道自己这局到底脏在哪里、断在哪里、收不在哪里。
```

失败信号：

- 所有问题都显示为 `low damage` 或 `bad draw`。
- 同一组奖励同时被解释成能解决所有问题。
- 玩家无法把下一步计划归类到清污、补桥、找终结之一。

可验收条件：

- 三类问题至少各有一次可见标签或等价短反馈。
- 每类问题至少有一个不同的回应类型。
- 玩家复盘时能把本局主要问题归入其中一类或多类。

核心体验打分影响：

- 奖励构筑回应：`+0.5`。奖励理由更精确。
- 牌区生命周期与状态污染：`+0.3`。污染不再和普通失败混淆。
- 战斗决策与升序爆点：`+0.3`。缺桥/缺终结的战斗语义被拆开。

### 镜头 9：路线后 plan 失败，玩家知道原因并获得修正入口

玩家画面：

- 玩家选择高压或污染风险路线后，下一战 plan 没有完全成功。
- 失败不是黑盒扣血，而是明确指向原因：`高压路线多 1 敌意图，污染占手导致缺桥`。
- 战后给出修正入口：清污、补桥、降压、换终结或撤离。

玩家应理解：

```text
我不是被随机惩罚了；我刚才路线的风险兑现了，现在要修正计划。
```

失败信号：

- 路线失败只表现为血少了，玩家不知道和路线有关。
- 风险兑现后没有任何修正选择，只能继续被滚雪球。
- 修正入口与失败原因无关。

可验收条件：

- 至少一条路线能让 plan 失败或半失败，并显示来源。
- 战后奖励/路线中至少一个选项回应这个失败原因。
- 玩家能回答：“这次 plan 为什么没成？下一步怎么修？”

核心体验打分影响：

- 敌意图、血量与失败压力：`+0.5`。风险开始有因果和修正空间。
- 短 run / 地牢推进：`+0.4`。失败成为下一节点计划，而不是中断体验。
- 奖励构筑回应：`+0.3`。奖励能回应路线风险兑现。

### 镜头 10：结算页回看 plan 是否被验证

玩家画面：

- 最终节点或阶段结算页不只列奖励和路线，而是按 plan 回看：
  - `原计划：补桥后找终结`
  - `路线修正：走高压，获得 +1 MP，但污染占手`
  - `验证结果：升级牌成功补桥；缺终结导致最终漏血`
  - `下局建议：优先找终结或走清污路线`
- 结算不把结果写成教程长文，只保留 3-4 条短句。

玩家应理解：

```text
这局不是一串随机事件；我的奖励、路线、升级和失败原因组成了一条可复盘的计划线。
```

失败信号：

- 结算只显示胜利/失败和分数。
- 结算只列物品，不解释 plan 是否成功。
- 下局建议变成局外永久成长或账号升级暗示。

可验收条件：

- 结算至少回看：原 plan、路线变化、升级验证、一个主要缺口。
- 玩家能回答：“这局最关键的选择是什么？下局先补什么？”
- 结算语言保持 run 内，不出现永久账号成长暗示。

核心体验打分影响：

- 短 run / 地牢推进：`+0.6`。短 run 有可复盘边界。
- 局内牌成长 / gem-like 成长：`+0.3`。升级是否成功被纳入结算。
- 奖励构筑回应：`+0.3`。奖励、路线、缺口形成闭环。

## 3. 最小可验收流程

第17轮最小合格玩家流程必须覆盖以下顺序：

```text
第一战
  -> 暴露至少一个问题：污染 / 缺桥 / 缺终结
奖励
  -> 奖励回应问题
  -> 选择后生成下一战 plan
路线
  -> 路线选择前比较两个 plan
  -> 路线选择后改写 plan 摘要
升级
  -> 升级一张本 run 内相关牌
第二战
  -> 首回合验证升级或路线改变
  -> 污染 / 缺桥 / 缺终结至少一个产生体感
结算或第三节点
  -> 回看 plan 是否成功
  -> 给出修正入口或最终复盘
```

如果只做到奖励后显示 plan，但路线选择不改变 plan，第17轮最多只能从 `80` 加到 `82`。  
如果路线改变了 plan，但升级没有在下一战验证，最多只能到 `83`。  
如果污染、缺桥、缺终结仍然无法被玩家区分，最多只能到 `84`。  
只有 10 个镜头中至少 8 个可被玩家口述验收，才建议记到 `86 / 100`。

## 4. 玩家口述验收问题

| 镜头 | 只问一句 | 合格回答 |
| ---: | --- | --- |
| 1 | “你刚拿这张奖励是为了下一战解决什么？” | 能指向上一战问题和下一战计划。 |
| 2 | “两条路线会让你的计划哪里不同？” | 能比较稳妥修复与高压回报，或等价差异。 |
| 3 | “选完路线后，你的计划变成什么了？” | 能说出资源、风险、奖励或目标优先级变化。 |
| 4 | “升级牌刚刚验证了吗？” | 能指出升级牌出现、打出或改变局势。 |
| 5 | “污染让你这一手难在哪里？” | 能指出占手、少桥、少终结、少防御或多花资源。 |
| 6 | “这一手是缺桥还是缺终结？” | 能说出链路断在中段。 |
| 7 | “你铺好了以后为什么没收掉？” | 能说出缺终结，而不是泛泛说伤害低。 |
| 8 | “这局主要问题是哪一类？” | 能在污染、缺桥、缺终结中归类。 |
| 9 | “这次计划为什么失败，下一步怎么修？” | 能关联路线风险和修正入口。 |
| 10 | “这局最关键的选择是什么，下局先补什么？” | 能复盘奖励、路线、升级或主要缺口。 |

玩家答不上来时，对应镜头不能计入核心体验分。QA 全绿只能说明没有技术阻塞，不代表玩家理解成立。

## 5. 一票否决项

- 奖励后没有任何下一战 plan，玩家仍只是在点继续。
- 路线选择前后 plan 摘要没有变化，路线只像菜单选项。
- 升级牌下一战无法被看到、抽到、打出或验证。
- 污染只在日志或计数里，没有占手、抽牌质量或资源成本体感。
- 缺桥和缺终结被混成同一种失败，玩家无法区分。
- 高压路线只有收益没有风险，或风险不可预判、不可修正。
- 结算只列胜负，不回看 plan 是否成功。
- QA 报告、截图数量、脚本通过被写成核心体验加分理由。

## 6. 交付给后续 worker 的优先级

P0：

1. 奖励选择后生成 1 句下一战 plan。
2. 路线选择前后都显示 plan，并让路线选择实质改写 plan。
3. 升级牌必须在下一战有可见验证机会。
4. 污染、缺桥、缺终结必须有不同体感和不同回应标签。

P1：

1. 高压路线加入可预判风险和战后修正入口。
2. 结算页按 `原 plan / 路线修正 / 验证结果 / 下局补点` 做短复盘。
3. 奖励原因同时引用上一战问题和当前 route plan。

P2：

1. 为不同 build plan 增加 2-3 个短句模板，避免所有复盘长得一样。
2. 让第三节点候选根据 plan 成败调整排序，但先保证可读，不急着扩内容量。
