## 目标

奖励态下，如果快 run 路线层提供 `pendingNodeChoices`，HUD 需要在奖励卡之后显示 2 个路线按钮，玩家可点击选择路线。按钮必须用短 token 表达下一战影响：

- 下一战 modifier。
- 奖励倾向。

本轮不改 sim 核心；route 数据暂按 HUD-only adapter 防御式读取。

## 实现

### HUD route adapter

新增 `hudRouteChoicesState(snapshot)`：

- 从 `snapshot.route`、`snapshot.shortRunRoute`、`snapshot.run.route`、`snapshot.run.shortRunRoute`、`snapshot.run.pendingNodeChoices` 防御式读取路线候选。
- 最多显示 2 个候选，匹配当前短 run 设计。
- 将长字段压成 UI token：
  - `rewardRerollPlusOne` -> `复核+1`
  - `maxEnergyThisRunPlusOne` -> `MP+1`
  - `repair-resource` -> `偏修补`
  - `payoff` -> `偏终结`
  - `route-bridge` -> `偏路线`

`hudRunLayerState` 在有路线候选时显示：

- `路线候选 2`
- `下一战 / 选路线`
- `复核+1/偏修补`

### 奖励态路线按钮

`Reward` 面板在奖励卡组之后追加 `.route-choices`：

- 每个 `.route-choice` 是真实 button。
- `data-route-choice-id` 保留候选 id。
- 点击后发出 `{ type: 'select-route', routeChoiceId, traceId }`。
- 这里使用局部 cast 适配现有 `Intent` 类型；runtime 未接入前不会改 sim 核心。

### 溢出控制

新增 `.route-choice` 样式并复用 reward-card 的稳定 grid：

- 两列桌面、单列窄屏。
- `span/strong/small` 单行省略。
- `em` 两行 clamp。
- 所有子元素 `min-width: 0`，避免长中文说明撑出卡片。

## 测试

新增 UI 测试覆盖：

- 能从 run route pending choices 读出 2 个路线候选。
- modifier/奖励倾向短 token 正确。
- run layer 在 route pending 时切到 `选路线`。
- reward panel 渲染 2 个 `.route-choice`。
- 点击 route button 会发出 `select-route` 意图对象。
