# Redline Progression Layer Research 01

日期：2026-05-18
身份：Roguelike Progression Layer 专家
范围：只做研究文档；不改代码；不触碰 `prototype-web`。

## 一句话结论

Redline 当前 demo 不应该先做完整局外成长，也不应该把所有成长都塞进单次战斗。最适合的层级是：

```text
局外成长：只留概念和未来入口，不在当前 demo 兑现
一次 run：做成 3-5 场遭遇的短路线框架，可先用脚本表达
一场战斗：作为当前 demo 的主要体验容器，承载敌意图、奖励、失败压力
一回合手牌循环：作为最小爽点，承载 0 -> 1 -> 2、断链、修补、payoff
```

玩家第一轮必须记住的不是“我升级了多少属性”，而是：

```text
我看懂了本回合后果
我按正确费用顺序打出链
我用 Wild/draw/mana 修回了坏手牌
我靠这次链路救回一场快崩的战斗
奖励回应了我的打法
```

本轮使用公开资料做轻量核对，不捏造具体数值：

## 1. 局外成长属性

### 定义

局外成长发生在 run 之外。它的作用不是让某一场战斗立刻更热闹，而是给玩家一个跨失败的长期承诺：失败不是完全归零，部分资源、解锁、角色、基础能力或难度选项会留下。

Hades 的 Mirror of Night 是典型例子：玩家把 run 中带回的资源投入到基地能力里，改善后续尝试。Vampire Survivors 也有局外金币、角色/武器/关卡解锁和永久强化。它们共同点是：run 会重置，但账户/基地层保留。

### 时间尺度

- 单次反馈：一次失败后 10-60 秒内看到“可购买/可解锁”。
- 中期目标：几次 run 后形成明显能力差异。
- 长期目标：角色、难度、成就、构筑流派逐步打开。

### 玩家记忆点

- “我死了，但带回了资源。”
- “我解锁了新角色/新起手/新卡池。”
- “下次开局比这次多一个可控点。”
- “我可以主动选择更难规则换更多收益。”

### 常见属性

- 最大生命、复活次数、基础伤害、护甲/减伤。
- 起始资源：初始金币、初始卡、初始武器、初始能量。
- 系统权限：重抽次数、奖励刷新、路线预知、商店折扣。
- 解锁项：新角色、新卡、新遗物、新地图、新难度。
- 账户型收藏：图鉴、成就、挑战模式、皮肤。

### 常见奖励

- 永久货币。
- 新角色或角色变体。
- 新卡/新遗物进入池子。
- 起手祝福或局外天赋。
- 难度层级和挑战规则。

### 失败 / 重置边界

失败时通常重置本 run 的临时 build：当前牌组、临时 buff、路线进度、战斗内资源、未消费的部分临时货币。

失败后通常保留：永久货币、解锁进度、图鉴、成就、部分角色熟练度、局外天赋。

### Redline 当前取舍

当前 demo 不适合实现完整局外成长。原因：

1. 玩家还没稳定理解一回合牌序和一场战斗的救场爽点，过早给永久属性会把注意力带偏。
2. 完整局外成长会引入存档、解锁表、成长曲线、难度缩放、平衡债务。
3. 当前最需要证明的是“卡牌链路本身好玩”，不是“数值变大后能赢”。

建议只在设计上预留三个未来入口：

| 未来局外属性 | 作用 | 为什么现在不做 |
| --- | --- | --- |
| `Starting Permit` | 每 run 开局给一次终局授权或起手稳定性 | 会掩盖当前 3 MP 下链路是否成立 |
| `Audit License` | 增加奖励刷新/重抽次数 | 需要先有足够奖励池 |
| `Debt Insurance` | 失败后保留一张已解锁卡或资源 | 需要 run 层先成立 |

当前文案中可以提“局外成长会存在”，但不要把它做成可点系统。

## 2. 单次冒险 / 一次 run

### 定义

一次 run 是从开局选择到胜利/失败结算之间的完整冒险。它负责把多场战斗和非战斗事件串起来，让玩家感觉“这一局的构筑路线”有命运感。

Slay the Spire 的 run 由地图路线、战斗、精英、事件、商店、Boss、战后奖励组成。Monster Train 的 run 由多场战斗、战斗后的卡牌/神器/升级选择和路线强化组成。它们都不是只看单场战斗，而是让每个奖励影响之后的遭遇。

### 时间尺度

- 完整商业游戏：通常 20-60 分钟一局，取决于类型和玩家熟练度。
- 卡牌 roguelike demo：可压缩为 3-5 场遭遇，5-10 分钟。
- Redline 当前技术切片：可先压缩为 1 场战斗 + 1 次奖励，或 3-5 回合模拟一段 run 感。

### 玩家记忆点

- “这局我拿的是修链路线。”
- “我本来缺 2 费展开，后来奖励补上了。”
- “我为了打精英拿了高风险奖励。”
- “Boss 前我知道自己的 build 还缺稳定性。”

### 常见属性

- 当前 HP / 最大 HP。
- 当前 deck / 卡组厚度 / 抽牌一致性。
- 当前遗物 / 被动规则。
- 当前货币 / 商店资源。
- 当前路线位置 / 下一个房间类型。
- 当前 build 标签：毒、力量、召唤、护甲、连击、抽牌、返费。

### 常见奖励

- 战后加卡、删卡、升级卡。
- 遗物/神器/被动规则。
- 金币、药水、一次性消耗品。
- 角色成长、职业分支、专精。
- 商店购买、路线事件、代价型奖励。

### 失败 / 重置边界

失败时重置：

- 本 run 的牌组修改。
- 本 run 的遗物/神器。
- 本 run 的路线、地图、战斗状态。
- 当前货币、临时 buff、事件结果。

可能保留：

- 局外货币。
- 解锁项。
- 图鉴和挑战进度。

### Redline 当前取舍

Redline 当前不必做完整随机 run，但需要让玩家感到“这不是孤立按钮测试”。建议 P1 做一个极短 run 框架：

```text
遭遇 1：教学链 0 -> 1 -> 2
奖励 1：repair / extension / payoff 三选一
遭遇 2：用奖励修坏手牌或打更长链
奖励 2：根据表现给更偏向的选项
遭遇 3：高意图压力，验证 build 能否救场
```

如果当前只做一场战斗，也应该在战斗内模拟 run 感：

- 第 1-2 回合像“开局基础 deck”。
- 第 3 回合给一次奖励或临时补给。
- 第 4-5 回合让奖励改变手牌循环。
- 战斗胜利后显示“下一场会带走什么”。

不要在当前阶段做随机地图、商店经济、完整事件池。它们会扩大 scope，但不会直接回答 Redline 的核心问题：费用链是否足够爽。

## 3. 单局 / 一场战斗

### 定义

一场战斗是 run 内最小的胜负容器。它把当前 build 转化为即时压力：敌人要做什么、玩家手里有什么、本回合能不能避免伤害、是否值得贪更长链。

Slay the Spire 的战斗强调敌意图、能量、手牌、弃牌和战后奖励。Monster Train 的战斗强调多层空间压力、单位部署、法术、神器和防守目标。Vampire Survivors 虽然不是回合制，但每局战斗/关卡内也用经验升级、武器选择和时间压力制造“场内成长”。

### 时间尺度

- 卡牌 roguelike 普通战斗：几十秒到数分钟。
- 精英/Boss 战：数分钟以上。
- Redline demo：建议 3-5 回合、60-180 秒内看完一次完整救场。

### 玩家记忆点

- “结束回合会吃多少伤害，我知道。”
- “前排哪个敌人必须先处理。”
- “我这一手不够完美，所以选择修链而不是贪大招。”
- “我靠 payoff 取消了敌人意图。”
- “这场赢了以后奖励回应了我刚才的打法。”

### 常见属性

- 玩家 HP、护甲/格挡、能量/MP。
- 当前手牌、抽牌堆、弃牌堆、消耗区。
- 敌方 HP、意图、护甲、状态、位置。
- 当前回合数、阶段、行动次数。
- 当前 chain / combo / multiplier。
- 本战斗临时 buff/debuff。

### 常见奖励

- 战后卡牌三选一。
- 金币、药水、宝箱、遗物。
- 战斗内升级：经验升级、临时武器、被动增强。
- 直接恢复：回血、护盾、清除负面。
- Redline 可用的即时奖励：Wild、draw、mana stitch、payoff 授权、下一回合起手保证。

### 失败 / 重置边界

战斗失败常见结果：

- run 结束，进入结算。
- 或损失 HP/资源后继续，但付出长期代价。

战斗胜利常见结果：

- 清空战斗内临时状态。
- 保留 run 层变化：新卡、遗物、金币、HP 损失、事件后果。
- 进入奖励或下一节点。

### Redline 当前取舍

当前 Redline P0 应把“一场战斗”做成体验主容器，而不是做成完整 roguelike。

最低成立标准：

1. 战斗开始 3 秒内看到手牌、MP、敌意图。
2. 第一回合能打出一条明确 `0 -> 1 -> 2`。
3. 乱序出牌收益明显低于正确链，但不必完全禁止。
4. 至少一次坏手牌或断链，让玩家产生“需要修补”的需求。
5. 至少一次 Wild/draw/mana 修补成功。
6. 至少一次 payoff 因 chain 成立而明显取消敌意图。
7. 战后奖励能改变下一回合或下一场的手牌循环。

Redline 战斗层不要先追求：

- 大量敌人种类。
- 大量卡池。
- 复杂 status stacking。
- 完整金币商店。
- 多角色职业树。

当前更重要的是“战斗因果可复盘”：我为什么受伤、为什么救回来、为什么这张奖励能补我的缺口。

## 4. 单次循环发牌 / 一回合手牌循环

### 定义

一回合手牌循环是卡牌 roguelike 的最小决策单位。它通常包括：

```text
回合开始
  -> 抽牌 / 补手牌
  -> 获得本回合资源
  -> 玩家按顺序出牌
  -> 牌进入弃牌/消耗/保留
  -> 结束回合
  -> 敌人行动
  -> 下一回合重新循环
```

Slay the Spire 的基础战斗循环就是抽牌、能量、出牌、弃牌、敌人回合、抽空后洗弃牌堆。Redline 当前最该继承的是这个“手牌循环是压力核心”的原则，而不是把压力交给后台实时计时。

### 时间尺度

- 熟练玩家：几秒到十几秒打完一手。
- 新玩家：可能 30-60 秒思考。
- Redline 的 hyper-turn 目标：允许慢慢想，但一旦看懂路线，可以很快打完。

### 玩家记忆点

- “这手牌费用顺序对不对？”
- “下一张期望费用是什么？”
- “我缺哪一段链？”
- “这张 Wild 能不能补上？”
- “如果现在结束回合，会被打多少？”
- “payoff 现在是 armed 还是 unarmed？”

### 常见属性

- 手牌数量、抽牌堆数量、弃牌堆数量。
- 当前 MP / 最大 MP / 临时授权。
- 本回合已打费用序列。
- 下一张期望费用。
- 连击倍率、断链状态。
- 保留牌、消耗牌、生成牌。
- 本回合临时 buff：返费、抽牌、复制、降费。

### 常见奖励

手牌循环里的奖励通常不是永久奖励，而是即时节奏奖励：

- 抽 1 / 抽多张。
- 返还 MP / 临时 MP / 降低下一张费用。
- 复制下一张牌。
- 保留一张牌到下回合。
- Wild 补费用段。
- 击杀后追加行动。
- chain 达成后武装 payoff。

### 失败 / 重置边界

一回合结束时通常重置：

- 当前手牌中未保留的牌进入弃牌。
- 当前临时能量清空。
- 本回合 chain 状态清空或转化为 run/战斗层资源。
- 敌人按意图结算。

一回合内失败可能表现为：

- 断链导致倍率低。
- 没有资源打关键牌。
- 没有处理敌意图，结束回合受伤。
- 贪长链失败，被迫空过或吃伤害。

### Redline 当前取舍

这是当前 demo 的 P0 核心。

Redline 应优先把一回合做成“四段记忆”：

```text
识别：我看到 0 -> 1 -> 2 的路线
执行：我按顺序打出，倍率/反馈上涨
修补：缺口出现时，我用 Wild/draw/mana 接回
兑现：payoff armed，取消敌意图或清前排
```

最小规则建议：

| 规则点 | 当前 demo 建议 | 原因 |
| --- | --- | --- |
| 基础 MP | 保持 3 | 足够支撑 `0 + 1 + 2`，不先引入成长表 |
| 3 费 payoff | 不默认强；需要 armed | 避免玩家乱放大招也赢 |
| 临时授权 | 完成链或修链后给，只支付 payoff | 解释为什么短 demo 能打出终结牌 |
| 抽牌循环 | 小牌堆 + 可复盘 draw/discard | 让玩家记住“抽到/没抽到/修回来” |
| 奖励入牌 | 尽快进入下一手或下一次 draw | 奖励必须马上改变体验 |
| 断链 | 不禁止，但收益低、HUD 说明原因 | 保留速度感，不把游戏变成报错系统 |

## 层级之间的状态归属

| 状态/属性 | 应归属层级 | 是否带到下一层 | Redline 当前建议 |
| --- | --- | --- | --- |
| 局外永久货币 | 局外成长 | 跨 run 保留 | 只做文档预留 |
| 新角色/新卡池解锁 | 局外成长 | 跨 run 保留 | 当前不做 |
| 当前 deck | Run | 战斗间保留，失败后重置 | P1 轻量实现或脚本表达 |
| 遗物/被动规则 | Run | 战斗间保留，失败后重置 | P1 后再做 |
| HP 损失 | Run + 战斗 | 战斗后可保留到下一战 | P1 可做 |
| 战后奖励选择 | Run | 影响后续战斗 | P0 可做一次 |
| 敌人 HP/意图 | 战斗 | 战斗结束清空 | P0 必做 |
| 当前手牌/抽弃牌 | 战斗 + 回合 | 战斗内循环，战斗后通常重置或进入 deck | P0 必做 |
| 当前 MP | 回合 | 回合末清空 | P0 必做 |
| 临时授权 MP | 回合 | 本回合过期 | P0 推荐 |
| Chain multiplier | 回合 | 本回合结算后清空 | P0 必做 |
| Payoff armed | 回合或战斗内短状态 | 通常本回合有效 | P0 必做 |

## 适合 Redline 当前 demo 的层级裁决

### 应该做

1. 把“一回合手牌循环”定义成 Redline 的最小爽点。
2. 把“一场战斗”定义成当前 demo 的主要体验闭环。
3. 在战斗内放一次奖励，让玩家看到 build 正在形成。
4. 用 `repair / extension / payoff` 三类奖励解释成长，而不是先做属性海。
5. 用临时授权、返费、抽牌、Wild 修补解释长链，而不是直接把最大 MP 加大。

### 可以做但降级

1. 一次 run 可以先做成 3 场脚本式遭遇，不需要随机地图。
2. 战斗胜利后的奖励可以先固定三选一，不需要完整卡池权重。
3. 局外成长可以只在结算界面占位，写“未来可用清算额度/审计许可解锁”。

### 不应该做

1. 不要先做完整局外天赋树。
2. 不要先做永久属性数值堆叠。
3. 不要让 `Max MP +1` 成为当前 demo 的默认成长答案。
4. 不要把 run 层扩成商店、事件、地图、精英、Boss 全套系统。
5. 不要把一回合循环做成固定 6 张全量手牌，否则抽牌和构筑记忆会消失。

## Redline 推荐演示结构

如果只做当前 demo，建议用下面这个层级压缩：

```text
局外层：不进入主体验，只在结算/文档中预留

Run 层：显示“当前清算路线 1/3”，但不做复杂地图

战斗层：
  回合 1：稳定 0 -> 1 -> 2，玩家学会正确链
  回合 2：坏手牌或缺口，玩家看到断链风险
  回合 3：出现 Wild/draw/mana，修回链路
  回合 4：完成链，获得临时授权，armed payoff 救场
  回合 5：战后奖励回应打法，进入下一场或结算

回合层：
  每回合都明确展示 HP、MP、CHAIN、Next Cost、敌意图
```

这个结构的好处是：它能在不做完整 roguelike 的情况下，让玩家感到 roguelike progression 的雏形已经存在。

## 关键设计判断

### 1. 成长不等于永久属性

在 Redline 当前阶段，“成长”应优先表现为：

- 我多看到一张能修链的牌；
- 我多获得一次临时授权；
- 我能把坏手牌修成完整链；
- 我能把上一场奖励带进下一场；
- 我能在战斗内把 payoff 从 unarmed 变成 armed。

这些比 `+5% damage`、`+1 max MP` 更贴合当前核心。

### 2. Run 层服务记忆，不服务系统完整度

玩家只需要记住“这局我是怎么变强的”。当前不需要完整地图，只需要奖励选择能在下一段战斗里兑现。

### 3. 战斗层必须有失败压力

没有敌意图和受伤边界，牌序就只是算数。Redline 的战斗层必须让玩家知道：

```text
如果我这一手没处理，结束回合会发生什么
```

### 4. 回合层必须有修补空间

如果每手牌都刚好有 `0 -> 1 -> 2`，玩家记住的是固定脚本；如果每手牌都缺关键牌，玩家记住的是随机惩罚。最好的 demo 是：

```text
有一次完美链
有一次断链
有一次靠修补救回
有一次 payoff 兑现
```

## 后续验收问题

下一轮如果实现或评审，应按这几个问题验收，而不是只看功能是否存在：

1. 玩家能不能说出 Redline 的最小循环是“发牌 -> 识别链 -> 修链 -> payoff -> 敌意图结算”？
2. 玩家能不能区分“这场战斗内的临时收益”和“下一场还会保留的 run 奖励”？
3. 玩家能不能理解局外成长还没进入当前 demo，而不是误以为已经缺系统？
4. 奖励是否在 1 个回合或下一场内改变体验？
5. 失败时玩家能不能复盘“我断链/没处理意图/贪 payoff”而不是觉得系统随机杀我？

## 最终建议

Redline 当前应该把 progression layer 做成“由内向外”的验证：

```text
先让一回合手牌循环成立
再让一场战斗有救场记忆
再让一次 run 有奖励延续
最后才做局外永久成长
```

当前 demo 的最小可交付目标不是完整 roguelite，而是让玩家在 3-5 回合内完成一次清晰叙事：

```text
我拿到手牌
我看懂敌人意图
我打出 0 -> 1 -> 2
我遇到断链
我用奖励/修补接回
我释放 armed payoff
我赢下这场并选择下一步成长
```

只要这条成立，后续局外成长、完整 run 地图、商店、事件、角色解锁才有必要进入制作。
