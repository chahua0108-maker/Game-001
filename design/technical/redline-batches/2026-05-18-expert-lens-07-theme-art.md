## 19. 世界观 / 叙事设计师

### 当前判断

“红线清算局”是能解释卡牌链路和清算的题材，而且比纯吸血鬼、纯霓虹地牢更适合当前 hyper-turn card pressure：费用升序可以被解释为“立案 -> 标记 -> 执行 -> 清算终结”的程序升级，敌人意图可以被解释为“逾期债务/脉冲异常正在反噬”，payoff 可以被解释为“清算令生效”。问题是现在叙事只散落在 `Debt Hook`、`Verdict Mark`、`Clearance Order`、`Red Ledger Burst` 这些词里，HUD 和敌人还没有把玩家身份钉死为“清算员”。

### 10 个镜头观察

| # | 镜头观察 | 判断 |
|---:|---|---|
| 1 | 修改方案把核心从实时心跳改成 `0 -> 1 -> 2 -> 3` 卡牌链，这是“清算流程”天然能解释的结构。 | 题材和机制匹配。 |
| 2 | `Debt Hook` / `Blood Reclaim` 作为 0 费启动牌，像“抓取债务线索 / 回收血债”，但还没有明确“立案”感。 | 起链语义可再收束。 |
| 3 | `Redline Cut` / `Heartbeat Spark` 作为 1 费承接，动作感强，但“清算局流程”感弱。 | 现在像招式，不像执法步骤。 |
| 4 | `Verdict Mark` / `Clearance Order` 已经接近审计与清算语言，是目前最能支撑世界观的卡牌。 | 应作为命名母体。 |
| 5 | 3 费 `Severance Burst` / `Red Ledger Burst` 能表达终结，但 `Burst` 会把玩家联想到旧的 60 秒脚本爆发。 | payoff 语义需要避开旧方向残影。 |
| 6 | 敌人 `Debt Wisp`、`Redline Brute`、`Pulse Collector` 有“债 / 红线 / 脉冲”三类材料，但不像同一个机构案件里的对象。 | 怪物生态还未成体系。 |
| 7 | HUD 的主语已经有“本回合链路 / 下一张期望 / 敌人意图 / Payoff 预览”，但仍混有 `FSM`、`Deck`、`Draw`、`Discard`。 | 玩家身份会被 debug/系统词拉出戏。 |
| 8 | “敌人意图：将受 X 伤害”非常适合改成“未清算风险 / 追缴反噬 / 违约冲击”。 | 回合后果可直接叙事化。 |
| 9 | 走廊的 5x3 槽位可以解释为“清算通道 / 案件队列 / 债务排队入闸”，比实时推进更适合回合制补位。 | 空间逻辑可变成世界观资产。 |
| 10 | 目前没有一个首屏句子或 HUD 标签明确玩家是“红线清算局执行员”。 | 世界观入口缺失。 |

### 最大风险

最大风险是主题停留在名词贴皮：卡牌叫“清算”，但玩家实际感知仍是 MP、CHAIN、Payoff、FSM、Deck 这些系统词。这样会导致“红线清算局”无法解释玩家为什么要按费用升序出牌，也无法把每次断链、补链、清场变成叙事事件。

### 下一轮最小改动

只做一层文案语义映射，不改规则：

- 把链路解释为“清算程序升级”：`立案 MP0 -> 定性 MP1 -> 执行 MP2 -> 终结 MP3`。
- 把断链解释为“程序中断 / 证据链断裂”。
- 把 Wild / draw / mana 解释为“补证 / 调账 / 临时授权”。
- 把敌人意图解释为“未清算反噬伤害”。
- 在 HUD 或首回合提示中明确玩家身份：`红线清算局 · 第 7 执行员`。

### 验收方式

做一次不看设计文档的 60 秒观测：新玩家只看首屏、手牌和 End Turn 区域，必须能说出三件事：

1. 我扮演的是清算/执行角色，不是普通冒险者。
2. 费用升序不是抽象 combo，而是一条“清算程序”。
3. 结束回合前的伤害是“未完成清算的后果”。

## 20. 命名 / 文案设计师

### 当前判断

当前命名已经有可用的红线、债、账本、裁决、清算素材，但记忆点还分散：有的牌是英文动作名，有的是审计/判决词，有的是血/心跳词；HUD 则中英混杂，且 `Burst`、`FSM`、`Deck` 等词会稀释“红线清算局”的独特性。下一轮不需要扩卡，只需要建立一套命名语法，让玩家看到名字就知道它在链路中的角色。

### 10 个镜头观察

| # | 镜头观察 | 判断 |
|---:|---|---|
| 1 | `Debt Hook` 作为 0 费启动很清楚，但更像工具，不像正式案件动作。 | 可改成“立案钩索 / 债线钩取”语义。 |
| 2 | `Blood Reclaim` 语气强，但血债与清算局的关系未解释。 | 需要统一为债务、账本或血税体系。 |
| 3 | `Blood Tithe` 是抽牌启动，文案说“抽 1 张”，机制清晰，风味更偏献祭宗教。 | 与审计清算略偏。 |
| 4 | `Spark Tap` / `Heartbeat Spark` 更像电系技能，和 `Pulse Collector` 有联系，但和红线清算局关系弱。 | 脉冲线应定义成机构科技。 |
| 5 | `Redline Cut` 是最强记忆点之一，短、狠、和主题一致。 | 可以保留为核心攻击牌。 |
| 6 | `Verdict Mark` 直接把“判定/标记”放进卡牌，适合 1 费承接。 | 值得围绕它扩展词库。 |
| 7 | `Clearance Order` 非常适合 2 费展开，像一张正式文件，也能解释打前排。 | 是当前最佳文案方向。 |
| 8 | `Paper Route` 在代码 key 是 `paper_shatter`，显示名是 Paper Route，语义像路线，不像重排/整备。 | 命名与功能不够贴。 |
| 9 | `Severance Burst` / `Red Ledger Burst` 都用 `Burst`，但新方案明确不要固定 burst 伪装 build 爆发。 | 建议把 Burst 改成 Verdict / Warrant / Closure 类终结词。 |
| 10 | 敌人缩写 `WSP / BRU / COL` 在视觉上可读，但不形成玩家记忆点。 | 缩写应转成案件等级或债务类型。 |

### 最大风险

最大风险是“机制可读”和“记忆点”互相拉扯：为了让玩家懂规则，HUD 使用 MP、CHAIN、Payoff；为了风味，卡牌使用 Debt、Blood、Pulse、Ledger。两套语言没有合并时，玩家会记住系统，不会记住世界观。

### 下一轮最小改动

建立一张小型命名表，先服务首手和第一轮奖励，不扩内容量：

| 链路角色 | 当前示例 | 建议命名语法 |
|---|---|---|
| 0 费启动 | Debt Hook / Blood Reclaim | 立案、钩债、取证、回收 |
| 1 费承接 | Redline Cut / Verdict Mark | 红线、裁定、标记、定性 |
| 2 费展开 | Row Cleave / Clearance Order | 执行令、清算令、横向追缴 |
| 3 费终结 | Severance Burst / Red Ledger Burst | 结案、封账、红账裁决 |
| Wild 修补 | Wild Mana Stitch / Wild Gap Key | 补证、临时授权、缺口钥 |
| Draw / Reorder | Pulse Draw / Paper Route | 调档、复核、重排卷宗 |

HUD 文案建议把系统词包一层主题词：

- `CHAIN` -> `清算链`
- `Payoff 预览` -> `终局清算`
- `未解决 X 伤害` -> `未清算反噬 X`
- `结束回合` -> `提交回合`
- `Debug Trace` 继续折叠，不进入主体验。

### 验收方式

抽取首手 4 张牌和 3 张奖励牌做“遮住描述只看名称”的测试：玩家应能大致判断哪张是起手、哪张是承接、哪张是展开、哪张是终结、哪张是修补。若必须读长描述才能懂链路角色，命名还不合格。

## 21. Art Direction Lead

### 当前判断

当前视觉方向是“低模几何敌人 + 暗红走廊 + 青色轨道/槽位 + 红线斩击 + 白色 payoff 闪光”，已经有统一基础，适合快速原型和移动端可读性。它的优势是清楚、低成本、和卡牌链反馈兼容；问题是“审计清算”还没有进入形体语言，敌人看起来像霓虹几何怪，而不是债务、脉冲、红线案件的实体。

### 10 个镜头观察

| # | 镜头观察 | 判断 |
|---:|---|---|
| 1 | `corridorRenderer.ts` 的背景是深色低饱和，红色为主要危险色，青色为槽位/轨道色。 | 调色盘方向成立。 |
| 2 | 敌人三类用几何体区分：Debt Wisp 近似 icosahedron，Brute 是 dodecahedron，Collector 是 octahedron。 | 低模识别基础可用。 |
| 3 | Brute 使用洋红/暗红，Collector 使用青色，Wisp 使用红色。 | 类型区分有，但和“债务/清算”关系弱。 |
| 4 | 敌人标签显示缩写、名称和 HP，适合调试与可读性。 | 视觉效率高，但风味弱。 |
| 5 | 走廊地面横线、槽位圆环、车道线让 5x3 敌阵非常清楚。 | 这是当前最应该保留的视觉资产。 |
| 6 | `CardPlayed`、`DamageApplied`、`EnemyKilled`、`ClearBurstRequested` 已经映射到斩击、清场波、击杀膨胀。 | 卡牌触发视觉方向正确。 |
| 7 | 渲染层明确忽略 `AutoAttack`、`EnemyAdvanced`、`EnemyPressure`、`EnemyAttacked`，避免旧实时心跳视觉复活。 | 和新方案一致。 |
| 8 | payoff 使用白色/淡色斩击，普通 hit 用红色，all-enemy/burst 倾向青色。 | 反馈层次有雏形。 |
| 9 | 当前没有账本、印章、红线封条、票据、审计网格、裁决章等清算局视觉符号。 | 主题识别不足。 |
| 10 | 敌人和场景都偏抽象低模，若缺少 HUD 主题词，会被玩家理解为普通赛博地牢。 | 视觉需要文案一起锁定。 |

### 最大风险

最大风险是视觉统一在“低模霓虹”上，但没有统一在“红线清算局”上。低模霓虹是风格，不是题材；如果敌人、UI、卡牌反馈都只有红/青发光，玩家会觉得这是通用原型皮肤，而不是一个可记住的世界。

### 下一轮最小改动

不要重做模型，不生成新图，先在现有低模系统上加三类低成本符号：

- 场景：把地面横线解释成“审计闸线”，在前排槽位强化红线封锁感。
- 敌人：标签从 `WSP / BRU / COL` 增加案件类型感，例如 `DEBT / BREACH / PULSE` 或中文短章。
- VFX：普通接链用红线切割，修补用青色缝合线，payoff 用白色裁决章/清场波语义。

最小视觉目标不是更华丽，而是让每次卡牌链成立时看起来像“程序推进”，每次 payoff 像“结案盖章”。

### 验收方式

截三张静帧即可验收：

1. 战斗首屏：不读说明也能看出“红线 / 清算 / 审计通道”而不是普通霓虹地牢。
2. 接链瞬间：红线切割或轨道反馈能说明程序正在推进。
3. payoff 清场瞬间：视觉上明显强于普通攻击，并有“封账 / 结案 / 裁决”感。

## 优先级建议

1. 先把 HUD 主语从 `CHAIN / Payoff / 未解决伤害` 包装成“清算链 / 终局清算 / 未清算反噬”，这是最小成本、最大主题收益。
2. 统一首批卡牌命名语法：0 费立案、1 费裁定、2 费执行、3 费封账，避免继续扩卡。
3. 把 `Burst` 词从玩家可见终结牌里降级或替换，避免和上一轮失败的 60 秒 burst 混淆。
4. 敌人先按案件类型重命名或加标签，让 `Debt Wisp / Redline Brute / Pulse Collector` 进入同一套债务反噬生态。
5. 保留当前低模霓虹走廊，但补“审计闸线 / 红线封锁 / 裁决章”三种符号，不要急着换美术风格。
